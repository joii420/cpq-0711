垂直切片法：6 阶段实施路线图

每个阶段交付可运行的端到端功能闭环，依赖关系自底向上

P0 — 项目骨架 + 认证基础 ~2 周
后端
  • Quarkus 项目初始化 + 分层架构
  • Flyway 数据库迁移框架
  • User/Region/Department 表 + CRUD API
  • Session 认证 + 角色权限中间件
  • 首次登录改密 + 密码重置
  • 统一错误处理 + API 规范
前端
  • React + Ant Design 项目初始化
  • 路由 + 状态管理 + 布局骨架
  • 登录页 + 改密页
  • 用户管理页（CRUD）
  • 区域/部门字典管理页
✓ 交付标准：能登录、能管理用户/区域/部门、权限中间件拦截生效
P1 — 主数据管理 + 数据源 ~4 周
模块
  • 客户管理：CRUD + 多联系人 + 等级筛选
  • 产品管理：CRUD + Excel 导入 + 品类
  • 工序管理：6 大类种子数据 + API
  • 数据源管理：SQL/API 配置 + 测试 + 权限隔离
关键决策
  • 数据源提前到 P1 而非 PRD 的 M2b
  • 原因：组件管理依赖数据源（DATA_SOURCE 字段类型）
  • 只读数据库连接在此阶段配置
  • AES 加密存储在此阶段实现
✓ 交付标准：客户/产品/工序/数据源 完整 CRUD，数据源可测试执行
P2 — 配置中心（自底向上） ~5 周
构建顺序（严格依赖链）：
① 组件管理 → ② 模板配置 → ③ 工序选型 → ④ 产品-模板关联 → ⑤ 版本比对
  • 组件管理（~2周）：目录树 + 字段配置 + 公式拖拽构建器 + DATA_SOURCE 字段绑定弹窗 — 最复杂的前端交互
  • 模板配置（~1.5周）：拖拽组装 + 产品属性 + 小计公式 + 发布/快照/版本
  • 工序选型 + 模板关联（~1周）：产品绑定工序 + 工序组合↔模板绑定
  • 版本比对（~0.5周）：快照数据比对 + 差异展示
✓ 交付标准：能创建组件 → 组装模板 → 发布快照 → 绑定产品+工序 → 版本比对
P3 — 计算引擎（技术风险隔离） ~2.5 周
⚠ 技术风险集中阶段 — Drools POC 决定后续路径
  • Drools POC：验证 7.74.x + Quarkus 3.23.3 + Java 17 动态 DRL 加载
  • 折扣计算引擎：Drools 实现 + KieBase 缓存 + salience 映射
  • 审批路由引擎：FIXED/DYNAMIC 规则 + 兜底机制
  • JEXL 公式引擎：后端校验 + 循环引用检测
  • 前端 JS 公式引擎：与 JEXL 等价实现 + 容差校验
  • 定价策略模块：策略 CRUD + 规则管理（引擎消费方）
降级方案：POC 失败 → 纯 Java Service 实现，接口不变，对上层零影响
✓ 交付标准：PRD 验收标准全 7 条通过（单元测试 100%、集成测试、一致性验证、@Blocking、性能基准、降级开关）
P4 — 报价全流程 ~6 周
最大阶段 — 系统核心价值所在
报价生成器（5 步向导）
  • Step1: 客户选择 + 联系人 + 元数据
  • Step2: 产品配置（三步弹窗 + 页签组件 + 数据录入 + DATA_SOURCE 交互）
  • Step3: 价格策略应用（Drools 调用）
  • Step4: 商务条款
  • Step5: 提交审批（快照 + 路由）
  • 草稿自动保存机制
审批 + 输出 + 管理
  • 审批流程（通过/退回/通知）
  • 审批规则配置（系统管理子模块）
  • 报价单详情页 + 状态机操作
  • 报价单管理列表页
  • 报价单复制功能
  • 站内通知模块
  • 操作日志模块
✓ 交付标准：完整报价闭环可走通 — 创建→填写→折扣→提交→审批→状态流转
P5 — 输出 + 收尾打磨 ~3 周
  • PDF 导出（Quarkus Qute 模板 + 中文字体）
  • Excel 导出（Apache POI）
  • 打印样式优化
  • 邮件发送（报价单 + 通知统一）
  • 延期 / 客户反馈标记（ACCEPTED/REJECTED）
  • 定时任务（5 个 Scheduler 任务）
  • 端到端测试 + Bug 修复
✓ 交付标准：V1 功能完整，可进入 UAT
总估算：~22.5 周（单人全栈 + Claude 辅助）
