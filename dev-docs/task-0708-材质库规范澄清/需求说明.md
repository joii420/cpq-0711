## 2. 需求背景与目标 ⭐

- **要解决什么问题 / 痛点**：
     1.项目在之前已经做了材质的功能,当前要求材质库进行规范存储,查看当前数据结构是否满足存储要求
2. 材质库的数据文件已经放于dev-docs\tesk-0708-材质库规范澄清\材质库.xlsx,其中sheet(材质对应料号)可以忽略
  3. 规范材质库存储结构
  4. 根据需求判断当前系统中是否存在元素表,是否需要维护元素数据
- **本次要达成的目标**：
  1. 可以按照材质库.xlsx的文档格式进行导入,导入1000行 < 3s
2. 正确存储材质数据,材质料号数据
  3. 重复导入覆盖就的材质内容
  4. 主数据维护-> 材质页签 ,引用的是材质库的数据, 在材质页签增加导入材质库的功能,导入模板就是这份材质库.xlsx文件
  5. 编辑材质的抽屉中, 关联料号页签在本次任务中隐藏
  6. 将化学式-> 材质名称, 默认类型均为标准锁定, 在主数据维护-材质 中,增加创建时间和修改时间列, 排序顺序 (启用/禁用 > 修改时间 > 创建时间 )
- **不做什么（明确边界）**：
   非设计模块功能不做变更
