## 四、终验收清单（对齐需求第 8 节 + 8 项锁定决策）

上线前我逐条实测，全绿才算交付：

- [ ] **导入正确**：主数据维护→材质 上传 `材质库.xlsx`，254 种材质入库，报告列出跳过明细
- [ ] **只读两 sheet**：`材质对应料号` 与隐藏草稿表未被读入（无脏材质）
- [ ] **含量 ×100**：抽查某材质 `Σdefault_pct=100`（如 AgC3：Ag97+C3）
- [ ] **Upsert 语义**：二次导入同文件无重复、Σ稳定；手工新建材质不被导入清掉
- [ ] **删 seed**：12 条 demo 材质已清
- [ ] **元素主表**：`element` 表有数据、符号接得上定价键、中文名不被符号覆盖
- [ ] **CRUD**：新建/编辑/停用可用；编辑时材质编号只读
- [ ] **搜索**：材质编号 / 材质名称 / 元素（符号+中文）均可过滤
- [ ] **列表**：有创建/修改时间列；排序=启用优先→改时倒序→建时倒序
- [ ] **编辑抽屉**：化学式已改「材质名称」；关联料号 tab 已隐藏；无 name/配比
- [ ] **模板下载**：得到干净两 sheet 模板（非 12-sheet 脏文件）
- [ ] **性能**：1000 行导入 <3s（B6 用例 + 真文件实测）
- [ ] **无回归**：选配材质 Tab / `ConfigureSearchResource` 正常（name 走 COALESCE 不报错）

## 五、风险与雷区（验收时重点盯）

| 风险 | 后果 | 验收盯点 |
|------|------|----------|
| 含量未 ×100 | 触发"和=100"校验被拒 / 流入定价 100 倍错价 | 抽查 default_pct 是 97 不是 0.97 |
| element_code 存了数字编号非符号 | 接不上 `costing_element_price`，定价断链 | `element_code='Ag'` 不是 `'10001'` |
| 导入用了 truncate 而非 Upsert | 手工/存量材质被误删 | 造一条手工材质→导入后仍在 |
| 删了 name/spec_label DB 列 | 下游 `COALESCE(mr.name…)` 报错 | 列仍在，仅 UI 隐藏 |
| 导入嵌套循环连库 | 1000 行超 3s | 看 durationMs + 代码无 per-row DB 调用 |
