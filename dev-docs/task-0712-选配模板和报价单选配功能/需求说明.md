## 2. 需求背景与目标 ⭐

- **要解决什么问题 / 痛点**：

  - 报价单中产品的生成方式现在只有"从基础数据导入"，需增加**从已有产品选择**和**选配新增**两种方式
  - 可以给**材质**和**销售料号**配置 3D 文件源
  - 选配时需要显示产品的 3D 图片（随用户选择实时带出）

- **本次要达成的目标**：

  - 先了解业务流程后，将几个页面的**原型图（可交互 HTML）**设计出来，原型定稿后再推进前后端开发
  - 原型覆盖：①选配模板管理整组页面（列表 + 新建/编辑）②报价单「从已有产品添加」抽屉 ③报价单「选配添加」抽屉 ④配置中心「3D 模型配置」维护页
  - 用户可配置选配模板；在报价单中可通过「已有产品添加」和「选配添加」加入产品；加入后在报价单各视图按当前模板正常渲染
  - 销售料号和材质库都增加 3D 模型能力，选配时按用户选择实时自动带出 3D

- **不做什么（明确边界）**：

  - 报价单的其他功能、非本模块功能**禁止变动**
  - **报价卡片各视图 / Excel / PDF 导出不展示 3D**（3D 仅出现在两个"添加产品"抽屉内）
  - 不做 v0.4 独立全屏 3D 选配器（`configurator` / `product_config_*`）的改造，本次与其并存

## 4. 功能详述 ⭐

### 4.1 用户场景 / 操作流程

**选配模板管理（配置中心）**
- 新建选配模板 → 选择归属行业 → 勾选启用的选配参数（材质 / 元素含量 / 工序）并限定每个参数的可选值 → 创建
- 编辑选配模板 → 修改归属行业外的配置项（一行业一套，行业码建后不可改）→ 更新
- 停用选配模板（启用/停用两态）

**报价单 · 从已有产品添加**
- 草稿状态报价单 → 添加产品 ▾ → 从已有产品添加 → 弹出**当前客户**的产品列表（`material_customer_map`），支持 客户产品编号 / 销售料号 / 品名 / 规格 过滤 → 选中行展示该**销售料号的 3D 图** → 勾选（可多选）加入报价单 → 卡片按当前客户报价模板渲染

**报价单 · 选配添加（明细表格式，一产品可由多材质料号组合）**
- 草稿状态报价单（需已绑报价模板）→ 添加产品 ▾ → 选配添加 → 抽屉内是**空白选配明细表**（列：材质 / 元素含量 / 工序 / 数量 / 操作）
- 点【新增】→ 弹出子选择框：**材质（带过滤/搜索）→ 元素含量微调 → 工序** → 确认 → **回填成表格一行**，行内可编辑**数量**（一行 = 一个材质料号）
- 可重复新增多行；选材质时实时显示**该材质的 3D 图**
- 当表格**数量合计 ≥ 2**（≥2 件需组装）→ 提示/开启**组合工艺**选择（组合工艺来自工序库 `process_master` 中 `process_category='ASSEMBLY'` 的工序，标识用 `process_no`）
- **汇总**：展示明细表 + 组合工艺（如有）+ 指纹匹配；若整份配置**指纹命中已有销售料号**，自动带出该销售料号内容 + 其 3D 图 → 确认后加入报价单渲染卡片
- 若客户行业无模板且默认模板 `__DEFAULT__` 也不存在 → 抽屉内提示"缺少选配模板"

### 4.2 界面与交互（前端相关）

- 统一抽屉（Drawer）交互；列表页遵循 `SelectableTable` + 工具栏动作规范
- 3D 呈现形态：**上传 .glb 模型 + 预览图**；抽屉内默认显示预览图，可点开交互查看（交互查看为增强项）
- 3D 数据在**配置中心「3D 模型配置」新页**维护，按 材质库 / 销售料号 分区

### 4.3 业务规则与计算逻辑（后端相关）

- 已有产品列表来自 `material_customer_map`，按当前报价单客户过滤；过滤字段：`customer_product_no` / `material_no`(销售料号) / `customer_material_name`(品名) / 规格
- 选配可配置内容 = 材质（取自材质库 `material_recipe` / `MATERIAL_RECIPE` handler）+ 工序（取自工序库 `V6_PROCESS_MASTER` handler）+ 元素含量微调（`ELEMENT`，adjust 型）
- 有效模板解析：`EffectiveTemplateService.getEffective(customerNo)` —— 客户行业模板 → `__DEFAULT__` → `hasTemplate=false`（现成）
- 3D 模型按「对象类型 + 对象键」绑定：SALES_PART（=`material_no`）/ MATERIAL（=材质配方码）
- **后端不得存在 N+1**：产品列表、候选值、3D 预览、指纹匹配一律批量/JOIN

### 4.5 3D 模型「维护」与「选配带出」的操作流程关系 ⭐

> 本节明确：用户在配置中心**维护/上传**的 3D 模型，与**选配运行时**如何被查出并展示，二者靠 `subject_type + subject_key` 契约打通。

**A. 维护端（配置中心 · 3D 模型配置）—— 把 3D 绑到对象上**
- 管理员/产品经理在「3D 模型配置」页，按对象类型上传：
  - **材质模型**：`subject_type=MATERIAL`，`subject_key=`**材质配方码**；上传 `.glb` + 预览图
  - **销售料号模型**：`subject_type=SALES_PART`，`subject_key=`**销售料号(`material_no`)**；上传 `.glb` + 预览图
- 支持多版本，每个对象**仅一条 `is_current=true`**（"设为当前版本"可切换）；未维护 = 该对象无 `model_config` 记录。

**B. 运行端（报价单添加产品）—— 按选择实时查出并带出**

| 场景 | 触发点 | 查询键（取 `is_current`） | 展示 |
|---|---|---|---|
| 从已有产品添加 | 选中某销售料号行 | `SALES_PART` + 该 `material_no` | 该料号预览图；无则"未配置 3D 模型"占位 |
| 选配添加 · 选材质 | 新增子框里选定材质 | `MATERIAL` + 该材质配方码 | 该材质预览图；无则占位 |
| 选配添加 · 汇总指纹命中 | 整份配置**指纹命中**已有销售料号 | `SALES_PART` + 命中的料号 | 预览由"材质 3D"**切换为"料号 3D"** |

**C. 端到端关系图**
```
[维护端] 3D模型配置页
   材质模型  ──写──▶ model_config(subject_type=MATERIAL,  subject_key=配方码,   is_current)
   料号模型  ──写──▶ model_config(subject_type=SALES_PART, subject_key=销售料号, is_current)
                                   │
                                   │ 同 subject_key 同源同码
                                   ▼
[运行端] 报价单添加产品抽屉
   选材质         ──查 MATERIAL+配方码──▶ 材质3D 预览（实时）
   选/命中销售料号 ──查 SALES_PART+料号──▶ 料号3D 预览
```

**D. 关键约定（契约 + 兜底）**
- **匹配键同源**：维护端 `subject_key` 必须与运行端的选择值**同源同码**（材质=配方码、料号=`material_no`），否则带不出（对齐 AP-52「绑定语义/契约对齐」教训，见 R2）。
- **只取当前版本**：运行端只读 `is_current` 版本；历史版本仅供维护端回溯/回滚。
- **缺失不阻断**：查不到 `is_current` 记录 → 显示"未配置 3D 模型"占位，选配流程照常进行。
- **维护即生效**：维护端"设为当前版本/更新文件"后，下次打开选配即带出新版本（不得缓存陈旧，必要时显式 invalidate）。
- **展示边界**：3D 仅在「从已有产品添加」「选配添加」两个抽屉内呈现，**不进报价卡片各视图 / Excel / PDF**（D9）。

### 4.6 选配落库约定（= 等价导入的完整落库 + 指纹匹配 + 发号）⭐

> **原则**：选配确认发号后，必须做到"**一次等价于正常导入的完整落库（含头表）**"，严格遵循权威方案 **`docs/table/报价系统Excel导入落库方案.md`（V3.4）**，使发号后的选配料号数据与"导入的料号"等价、投标等下游可用；**不是**现有"只写 `*_item` 子表 + 自指行"的渲染取巧法。

**A. 涉及的 4 个 Sheet 落库逻辑（严格照落库方案对应章节）**

| 选配内容 | 对应 Sheet（方案章节） | 目标表 | price_type / cost_type |
|---|---|---|---|
| 材质 / 物料BOM | 物料BOM（§3） | `material_bom`(**头**) + `material_bom_item`(子) + `material_master` | — |
| 元素 | 物料与元素BOM（§4） | `element_bom`(**头**) + `element_bom_item`(子) | — |
| 工序 | 自制加工费（§10） | `unit_price` | `PROCESS` / `自制加工费` |
| 组合工艺 | 组装加工费（§14） | `capacity` | —（组装加工费语义，§14 列：seq_no=项次 等）|

- 自制成品的**工序**（unit_price）+ **材质身份**（material_master/material_bom）+ **元素**（element_bom）三者同以该自制成品料号（`finished_material_no`/`material_no`）为轴关联，都归于这个"自制成品"记录之下。
- **不涉及**的 sheet（本次选配不落）：元素单价 / 客户料号关系 / 元素回收折扣 / 来料各费 / 成品其他费用 / 组成件BOM / 组成件其他费用 / 组装加工费年降 / 电镀费用等。

**B. 选配专属的两个附加功能（导入没有、选配特有）**
1. **指纹匹配（判重用）**：`SalesFingerprintCalculator` 算**客户维度指纹**（token **按 paramTypeCode 字母序**排列 ELEMENT/MATERIAL/PROCESS → SIMPLE：`v1|CUST=..|ELE=..|MAT=..|PRC=..`；COMPOSITE：`v1|CUST=..|COMBO=子件:数量(排序)|CPROC=组合工艺process_no(排序)`，集合排序、去尾零、分隔符 `| = , : ∅` 守卫）→ 查 `sel_part_signature`(`UNIQUE(customer_no,structure_version,config_fingerprint)`)：命中 → **复用同一报价料号、幂等不重复落库**（对应原型"汇总指纹命中带出料号"）。
2. **发号（铸号）**：未命中 → `quoteAllocator.mintAndRegister(customerNo, yyMm)` 铸新报价料号（客户码+年月+序，`quote_material_no_seq`/`quote_customer_code`）+ 登记 `sel_part_signature`。

**C. 事务不变量**：指纹登记 + 上述完整落库必须**同一事务（REQUIRED）**，保证"签名可见 ⇔ 数据可见"，防并发败者复用先赢号时数据未提交 → Tab 空。

**D. 现状缺口（本次后端核心改造项）**：现有 `ConfigureProductService` 只写 `element_bom_item`/`material_bom_item` 子表（自指行 + `characteristic='2000'/NULL` 取巧）、`material_master.config_fingerprint` 传 null，**未写 `element_bom`/`material_bom` 头表、未做完整 BOM 语义** → 仅够镜像视图渲染。本次需改造为上表的完整落库。**逐表逐列规格在 `backtask.md` 阶段对齐落库方案 §3/§4/§10/§14 给出。**

### 9.2 关键决策

- **D1 — 整体路线**：在现有 `sel_template` + `ConfigureProductDrawer` + `AddProductModal` 基础上**补完**；v0.4 configurator 并存不动。
- **D2 — 3D 呈现形态**：上传 .glb **模型 + 预览图**；抽屉默认看预览图，可点开交互（交互为增强项）。
- **D3 — 3D 带出逻辑**：
  - 选配添加时，选材质 → 实时显示**材质 3D**；选配收尾指纹命中销售料号 → 自动带出该料号内容 + **料号 3D**。
  - 从已有产品添加时，可选项即已有销售料号 → 显示**销售料号 3D**。
- **D4 — 3D 数据模型**：`mat_part_model` 命名已过时，弃用为主存储。新建 **`model_config`**（含 `subject_type` SALES_PART/MATERIAL + `subject_key`）+ **`model_config_file`**（多文件 GLB/THUMBNAIL）。材质与料号都是"模型配置"对象，用字段区分，表名不局限料号。
- **D5 — 3D 维护入口**：配置中心开**专门新页**「3D 模型配置」，按 材质库 / 销售料号 分区，与 v0.4 part-model 页分开。
- **D6 — 模板兜底（default fallback）**：客户行业无选配模板 → 用默认模板 `__DEFAULT__`；默认也无 → 提示"缺少选配模板"。（`EffectiveTemplateService` 已内建）
- **D7 — 选配模板约束**：沿用"一行业一套"（`sel_template.industry_code` UNIQUE），行业码建后不可改；启用/停用两态，不做发布审批流。
- **D8 — 从已有产品添加语义**：直接加成品销售料号（可多选批量），套报价单已绑客户报价模板渲染，不再走材质/工序配置。

- **D10 — 渲染管线**：选配/已有产品加入的产品，沿用现有客户报价模板渲染机制，本次不改渲染管线、不动 `field_type`。
