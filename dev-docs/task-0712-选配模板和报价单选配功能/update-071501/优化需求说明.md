## 9. 需求澄清备案（技术总监梳理，2026-07-15 / update-071501）

> 本节逐条记录技术总监与需求方的澄清决策，是本次 update-071501 前端/后端/接口文档（`fronttask.md` / `backtask.md` / `api.md`）与开发的**唯一权威依据**。凡与 §1~§8 骨架冲突处，以本节为准。

### 9.0 一句话概括

> **把"产品分类"做成客户身上的一个绑定，之后报价/核价/选配三类模板全部以"客户所属产品分类"为唯一轴自动匹配，用户不再手选分类。** 本次只做"产品分类轴"这一件事，task-0712 已交付的其余功能一律不碰。

### 9.1 现状盘点（brownfield，本次是"换轴"非"从零"）

| 现有实现 | 位置 | 与本次关系 |
|---|---|---|
| 选配模板一行业一套 | `sel_template.industry_code` UNIQUE；`SelTemplateService.getByIndustry/upsert` | **换轴对象**：`industry_code → product_category_id` |
| 有效选配模板解析 | `EffectiveTemplateService.getEffective(customerNo)`：`customer.industryCode → 行业模板 → __DEFAULT__ → 无` | **换轴对象**：改读 `customer.productCategoryId` |
| 报价模板按分类+客户匹配 | `TemplateService.matchCustomerQuoteTemplate(customerId, categoryId)`（客户专属→通用兜底→NONE）已实现 | **复用**：`categoryId` 来源从"手选"改"客户带出" |
| 核价模板按分类+客户过滤 | `QuotationCreateForm` 调 `templateService.list({categoryId, templateKind:'COSTING'})` 已实现 | **复用**：同上，来源改客户带出 |
| 报价单创建表单手选产品分类 | `QuotationCreateForm.tsx`（默认预选"默认分类"，用户可改） | **改造**：改为客户带出、只读锁定 |
| 产品分类主数据 | `product_category`（含 `parentId` 列但业务不当树用）；管理页 `ProductCategoryManagement.tsx`；已有一条 `name='默认分类'` | **复用**：作为客户绑定与三模板匹配的轴 |
| 客户实体 | `customer`（有 `industry_code`，用于行业维度） | **新增字段**：加 `product_category_id` |
| 选配落库/候选过滤 | `ConfigureProductService.effectiveEnabledTypes` 调 `getEffective` | **不碰但需回归**：换轴后候选/落库不得回归 |

### 9.2 关键决策（逐条）

- **D1 — 统一轴**：核心 = 给客户绑定一个产品分类，报价/核价/选配三类模板全部以"客户所属产品分类"为唯一轴自动匹配，用户不再手选分类。
- **D2 — 客户新增绑定字段**：`customer` 表**新增 `product_category_id`（UUID）**；`industry_code` **保留不动**（行业概念另有用途，见 D8）。存储用 UUID（与 `template.category_id` 同源，带出后可直接喂 `matchCustomerQuoteTemplate`，免 code↔id 转换）。
- **D3 — 客户产品分类必填 + 新建默认"默认分类"**：客户管理新建/编辑页加"产品分类"单选（带过滤），**必填、不可为空**；**新建时默认预选 `name='默认分类'`**。迁移给所有存量客户 backfill 成"默认分类"，杜绝空值（属数据初始化，不算改业务存量）。
- **D4 — 客户产品分类可编辑**：客户建后**允许改绑**产品分类；**已生成的报价单不受影响**（报价单锁的是模板 id，且表头不持久化 `category_id`）；改绑只影响此后新建的报价单。
- **D5 — 客户列表不展示、不筛选**：客户列表页**不加**"产品分类"列，**不加**按产品分类的筛选（YAGNI）。产品分类仅出现在客户新建/编辑表单。
- **D6 — 选配模板换轴**：`sel_template.industry_code → product_category_id`（UNIQUE，**一产品分类一套**，沿用原一行业一套的唯一约束语义，仅换轴）；`SelTemplateService.getByIndustry/upsert`、`SelTemplateUpsertRequest`、`SelTemplateDTO`、`EffectiveTemplateService`、前端 `SelTemplateManagement.tsx` / `selTemplateService.ts` 全部换轴改名到产品分类；旧 `industry_code`/`industryCode` 命名一律不保留。
- **D7 — 产品分类扁平、精确匹配**：`product_category` 虽有 `parentId` 列，但业务上**不当树用（扁平）**；三模板解析一律**按产品分类精确匹配、不上溯父级**。
- **D8 — 行业概念保留**：`industry` 表、`customer.industry_code`、报价单 `snapshot_customer_industry` **原样保留不动**，只是"选配模板归属"不再用行业。
- **D9 — 报价单创建产品分类只读锁定**：报价单创建表单（`QuotationCreateForm`）的"产品分类"由**客户绑定唯一决定**，改为**只读展示、不可改**（与已有的"报价模板/核价模板自动结果"同性质）；移除"默认预选默认分类"那段手选逻辑，改为读客户带出值。报价/核价模板的匹配逻辑**保持不变**，仅 `categoryId` 来源从"手选"换成"客户带出"。
- **D10 — 选配模板未匹配 = 报错提醒（逻辑不变）**：选配解析在"客户产品分类下没有选配模板"时**报错提醒缺模板**，此终态行为不变。保留原有兜底链条结构：客户产品分类模板 →（"默认分类"选配模板兜底）→ 都没有则 `hasTemplate=false` 报错。即把原 `__DEFAULT__` 哨兵替换为真实的 `name='默认分类'` 产品分类。
- **D11 — 三类模板配置粒度不对称保持现状**：报价/核价模板是"一分类下可多套（客户专属 + `customer_id IS NULL` 通用兜底）"，选配模板是"一分类一套"。本次**不统一、不改**这个粒度差异（各自沿用现有匹配逻辑）。
- **D12 — 存量 `sel_template` 清空重配**：多条行业模板若都绑到"默认分类"会撞唯一约束；迁移时**清空 `sel_template` + `sel_template_item` + `sel_template_item_value`**（均为测试数据），换轴后重新配置。
- **D13 — 范围边界（只做产品分类轴）**：本次**只动** 4 块 —— ①客户绑定产品分类；②报价单创建产品分类改客户带出、只读；③选配模板换轴；④相关迁移。**明确不碰**（task-0712 已交付、本次禁改）：3D 模型配置 / 从已有产品添加抽屉 / 选配落库=等价导入完整落库 / 指纹匹配 + 发号 / 选配明细表交互（材质·元素·工序·数量·组合工艺）本身。

### 9.3 数据字段（本次新增/改动）

| 表 | 字段 | 变更 | 类型 | 必填 | 说明 |
|---|---|---|---|---|---|
| `customer` | `product_category_id` | **新增** | UUID | 业务必填 / DB 可空 | 引用 `product_category.id`；软引用（不加物理 FK）；存量 backfill "默认分类"；**DB 不加 NOT NULL（方案B，见 §9.7）**，业务非空由前端必填 + create 兜底 + backfill 三重保证 |
| `sel_template` | `industry_code` | **删除** | — | — | 换轴移除 |
| `sel_template` | `product_category_id` | **新增** | UUID | 是 | UNIQUE（一产品分类一套）；引用 `product_category.id` |
| `product_category` | `name='默认分类'` | **保障存在** | — | — | 迁移前置：不存在则 seed 一条（`code` 约定，如 `DEFAULT`），作为客户默认归属 + 选配兜底 |

### 9.4 接口影响（详见 `api.md`）

- 选配模板 CRUD：`POST /api/cpq/sel-templates`、`GET /api/cpq/sel-templates`、`GET/{id}`、`DELETE/{id}` —— 请求/响应体 `industryCode → productCategoryId`（+ 前端映射分类名）。
- 选配有效模板解析：`GET /api/cpq/sel-templates/effective?customerNo=` —— **URL/入参不变**，内部换解析源；响应 DTO `resolvedIndustryCode → resolvedCategoryId`（`usedDefault` 保留表示走了"默认分类"兜底）。
- 客户 CRUD：`CustomerDTO` / `CreateCustomerRequest` 增 `productCategoryId`；create 必填（后端兜底"默认分类"），update 可改。
- 报价模板匹配：`GET /api/cpq/templates/match-customer-quote?customerId&categoryId` —— **签名不变**，`categoryId` 由前端从客户带出。
- 报价单创建 commit：`categoryId` 以 **`customer.product_category_id` 为权威**（后端不信前端传值，用客户绑定值匹配模板），防前端漏改/篡改。

### 9.5 验收标准（终版）

1. **客户绑定**：客户新建/编辑可选产品分类；新建默认"默认分类"；必填非空；建后可改；存量客户已 backfill"默认分类"（无空值）。
2. **报价/核价模板匹配**：报价单创建时"产品分类"由客户只读带出、不可改；报价模板（客户专属→通用兜底）与核价模板据此自动匹配，行为与改造前一致。
3. **选配模板换轴**：选配模板管理页按"产品分类"配置（一产品分类一套）；选配抽屉按客户产品分类带出正确模板；客户产品分类无选配模板时报错提示（逻辑不变）。
4. **换轴回归**：选配抽屉候选值过滤 + 选配落库（`ConfigureProductService`）不回归；报价单渲染、3D、从已有产品添加、指纹发号等未触及功能零回归。
5. **工程质量**：后端无 N+1（客户/选配/模板匹配一律批量或单条查）；前端 `tsc` 0 错误 + 改动文件 Vite 200；Flyway 迁移 success=t。

### 9.6 已知风险

- **R1 — 共享 Flyway 历史（移动靶）**：当前最大版本 V336（V327→V330 有跳号）。工程师进场**先探当前最大版本号再定 V 号**，不写死；不改已应用迁移的号/名。
- **R2 — 换轴回归面**：`getEffective` 被 `ConfigureProductService.effectiveEnabledTypes` 复用；DTO 字段 `resolvedIndustryCode → resolvedCategoryId` 改名需 grep 全工程改净，避免选配落库静默失败。

### 9.7 开发期决策订正（2026-07-16，实现 + spec 评审阶段）

> 实现与评审中发现两处与初稿假设不符，技术总监裁决订正如下（已同步 `api.md` / `backtask.md`）：

- **订正1 — B5 commit 是"审计"非"覆盖"**：实测 `ImportSessionService.commit()` 里 `categoryId` 是死字段（`customerTemplateId`/`costingTemplateId` 由前端在调 commit 前用 `match-customer-quote`/核价 list 预匹配好透传，commit 本身不匹配模板）。故 B5 定为**防御性一致性审计**（查 Customer + 不一致 `LOG.warn`，**不覆盖模板/不改持久化/不重匹配**）；"以客户为准"的真实落点在前端只读锁定（D9）。否决"commit 内重匹配覆盖模板"的强版本（越出 D13 边界 + 破坏 MIXED 多模板下用户手选语义）。详见 `api.md §5`。
- **订正2 — 客户产品分类 DB 层可空（方案B）**：D3"不能空"是**业务语义**，由「前端必填 + `CustomerService.create` 兜底默认分类 + 存量 backfill」三重应用层保证，外加 `EffectiveTemplateService` 对 `null` 的默认分类兜底。V337 **不加 DB `NOT NULL`**——否则全后端约 31 个与本功能无关测试的 customer 夹具（未带此列）会在迁移后 setup 违反约束。DB 硬约束的边际收益只挡"绕过业务代码的裸 SQL 写入"，本系统仅测试夹具如此。换轴自身的 2 个 seltemplate 测试（Tier1：`EffectiveTemplateServiceTest`/`SelTemplateResourceTest`）已同步重写到产品分类模型。
