## 1. 为什么前端不用改

后端本次把导入语义从「逐 sheet 独立事务、可能 PARTIAL」改为「两阶段全量校验 + 整单回滚」，但**接口契约与返回结构对前端保持兼容**（详见 api.md）：

- 端点不变：`POST /api/cpq/basic-data-import/v6/quote`（multipart：customerId + file），仍异步返回 `PROCESSING`。
- 轮询不变：`GET /api/cpq/basic-data-import/v6/{recordId}`，仍返回 `status` + `metadata`。
- 进度条不变：`metadata.progress = {done,total,current}` 结构保留（Phase 2 写入阶段仍逐步上报）。
- 错误清单不变：失败时 `metadata.sheetResults[].errors[]` 结构保留，前端现有渲染直接复用。
- 状态取值变化仅：**不再出现 `PARTIAL`**（只会是 `PROCESSING` / `SUCCESS` / `FAILED`）—— 前端若对 PARTIAL 有特殊分支，退化为按 FAILED/SUCCESS 处理即可，**通常无需改动**（现有 UI 一般已把非 SUCCESS 视作失败态展示错误清单）。
