### 2026-07-09 · 第一轮澄清

**C1 · 任务目录归属**
- 本需求文档与全部任务文档统一存放在 `dev-docs/tesk-0709-导入报价数据和导入核价数据的版本升级与版本维护/`。
- （立项指令中曾误提 `task-0709-元素主表管理`，该目录是另一件已交付任务，与本次无关。）

**C2 · 现状认定：系统已存在两套版本化机制，不是从零做**
- `VersionedV6Writer`（现役 V6 路径，支持零 N+1 批量）——白名单 7 张表：`unit_price / capacity / plating_scheme / element_bom(_item) / material_bom(_item)`。其入参模型已经就是需求要的"轴/值"：
  - `groupKeyColumns` = **轴**（唯一定位一组当前生效行）
  - `contentColumns` = **值**（参与"内容是否变化"比对）
  - `versionTriggerColumns`（⊆值，可选）= **真正触发升版的值子集**（未指定则=全部值列，任意值变化即升版）
- `VersionedWriter`（旧客户级路径，写 change_log）——`mat_process / mat_fee / plating_fee / mat_plating_fee`，属 V44 已废弃表（AP-53），**本次不动**。

**C3 · 版本化范围（需求方拍板）**
- **分层一（已在白名单的 7 张业务表）+ 分层二（有 is_current+版本列但导入未接版本化写入器的业务表：`labor_rate / production_energy / auxiliary_energy / production_consumable / packaging_consumable / exchange_rate_v6`，以及分层三无版本列的 `tooling_cost`）都要纳入版本化**，统一约定"轴 + 比对值"。
- 主档/映射表（`material_master / material_customer_map`，本身无 is_current 列）维持 upsert，不版本化。
- 全局登记表（`material_version_mgmt` 等）暂不纳入，后续如需再议。

**C4 · 交付分步**
- 第一步：先为**当前已实现版本化的 sheet** 出一份《版本升级规则文档》（轴/值/触发列/落库升版逻辑，全部源自现有代码），供需求方审校调整。
- 第二步：调整定稿后，再对**需要新增版本化的 sheet**（分层二/三 + 核价侧 10 个未版本化的 unit_price 费用 sheet）补充规则。
- 规则文档：`版本升级规则文档.md`（与本文件同目录）。

### 2026-07-09 · 第二轮澄清（核价侧升版口径）

**C5 · 核价升版模型 = 甲·版本全系统自增**
- 所有核价表**忽略 Excel 自带的版本列**（元素价格版本 / 材料价格版本 / 汇率版本 / 取用的计算版本 / 版本编号），一律由 `VersionedV6Writer` 按内容指纹比对系统自增（首版 `2000`、内容变 `max+1`、旧组 `is_current=false` 翻转）。
- 与报价侧口径统一；覆盖核价侧原"Excel 业务版本直存"的 P01/P02/P03/P09/P10/P11/P12/P13/P14/P22 等所有表。

**C6 · 核价取数口径 = A·永远取 is_current=true 最新版（现行业务即如此）**
- 核价永远使用当前生效（`is_current=true`）的最新价，**不按版本号复现历史核价**。
- 直接后果：「核价版本」sheet（P04 → `material_version_mgmt`）不再承担"用元素/材料/汇率版本号钉住历史价格"的职责，**划出本期范围**（与 C3"全局登记表暂不纳入"一致）；P01/P02/P03 纯系统自增无引用断裂问题。
- 由此《版本升级规则文档》§四"差异④/⑤"里关于 Excel 版本号的顾虑对核价侧不再适用。

### 2026-07-09 · 第三轮澄清（料号口径纠偏 + 升版粒度模型）

**C7 · 料号语义以 task-0708/V315（master 已交付）为准，`核价落库方案.md` §四/§五(sales_part_no)是过时未提交 WIP，作废**
- `material_no` = **销售料号**（系统主料号，宏丰料号改名而来）；`production_no` = 生产料号（**描述列，不进唯一键、不进升版轴、不参与升版比对**）；`material_part_no` = 材质料号（仅 `element_bom(_item)` 进唯一键）。
- 全库已无 `sales_part_no` 列（V315 反做删除）。
- **升版轴（groupKey）以 `material_no`（销售料号）为锚**。`unit_price` 表无 `material_no` 列，销售料号按 sheet 结构落在 `code`（自制/成品/电镀/外协类）或 `finished_material_no`（来料类），轴锚定其承载列。

**C8 · 升版粒度模型（核价侧本次统一采用）**
- **销售料号 = 版本总轴**；**sheet = 分类**，**每个 sheet 独立管理自己的版本线**（不是所有 sheet 合成一个统一版本）。
- **版本组 = (sheet, 销售料号)**：某 sheet 内、某销售料号的**整批行**为一个版本组（**粗粒度、多行一组**）；子维度（来料料号/要素/工序/项次等）当作**值(content)**，不作轴。
- **顺序无关**：用 multiset 比对，**不因导入行的先后顺序不同而升版**。
- 任一值列变化 → **该 (sheet, 销售料号) 组整体升一版**（旧组 `is_current=false`、新组 `is_current=true`）；同一销售料号的其他 sheet 不受影响。
- 与现役 `VersionedV6Writer`「一个 material_no 一组、组内整批 multiset 比对升版」同构，改动面可控。

**C9 · 全局 5 表排除"销售料号轴"，各按自身轴独立升版**
- `元素核价价格表(P01)`（轴=元素代码 code）、`材料核价价格表(P02)`（轴=材料料号 code）、`汇率(P03)`（轴=币种对）、`电镀方案(P21)`（轴=scheme_no，已版本化）、`核价版本(P04)`（版本包，C6 已退化出范围）。
- 这 5 张不挂销售料号，不能并入销售料号版本组；仍走系统自增（甲）、按各自轴独立升版。

### 2026-07-10 · 第四轮澄清（unit_price 锚规则 + production_energy 重构 + 触发列）

**C10 · unit_price 版本组规则**
- 版本组 = **(system_type, price_type, 销售料号锚)**；`cost_type / code(来料料号) / operation_no / seq_no / 要素` 一律入**值(content)**，整组 multiset 比对、任一变则整组升版。
- **销售料号锚按 price_type 识别**：来料类（`INCOMING_PROCESS`/`INCOMING_OTHER`，P15/16/17）→ `finished_material_no`；其余（`CONSUMABLE`/`PACKAGING`/`SELF_PROCESS`/`FINISHED_OTHER`/`PLATING`/`OUTSOURCE_PROCESS`，P13/14/18/19/20/22/23）→ `code`；全局（`ELEMENT`/`MATERIAL_PRICE`，P01/02）无销售料号，按 code(元素/材料料号) 独立成轴。
- 共用 price_type 的两 sheet **合成一个版本组**：P16+P17→一个 `INCOMING_OTHER` 组；P19+P20→一个 `FINISHED_OTHER` 组。

**C11 · production_energy 表结构优化（本次新增任务项）**
- 根因：建表未加类型区分，折旧(`depreciation_unit_price`)与能耗(`energy_unit_price`)塞两列、**共用行** → 无法独立版本。
- 改法：`production_energy` **增 `price_type`（ENERGY=能源 / DEPRECIATION=折旧）** + 把两个单价列**合并为单列 `unit_price`**；P09 折旧写 `price_type=DEPRECIATION`、P10 能耗写 `price_type=ENERGY`，**各写各行、不共用**。
- 升版轴 = `(material_no 销售料号, price_type)`；值 = `process_no, unit_price, currency, unit`；工序整批一组、系统自增（甲）。两 sheet 独立版本。
- 迁移：加列 + 合并列 + 存量行按非空单价拆成对应 `price_type` 行（数据量小，或清空重导）；`核价落库方案.md` §9/§10 同步改。
- 命名注意：`production_energy.unit_price` 列与同名表 `unit_price` 仅名字相近、不同命名空间，语义=“单价（按 price_type 区分类型）”。

**C12 · 触发列口径（③采纳）**
- 核价 `capacity` + `labor_rate` **去掉触发列**，对齐甲：任一值（含金额）变化即升版。
- 报价 `Q14 组装加工费` 同款触发列**不在本期范围**，留档暂不动（短期内产能"金额是否升版"报价/核价不一致，待报价侧统一时拉平）。
