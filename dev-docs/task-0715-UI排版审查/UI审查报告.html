Executive Summary

三件最痛的事

从 ~190 条明细里归纳，下面三条是最刺眼、最伤「专业观感」的系统级事实。它们彼此独立、各自可单独整改。

规范失守 · P0

两条自家写进 CLAUDE.md 的强制规范大面积失守

Systemic Themes · 11

系统性问题（按修复杠杆排序）

这一节是报告核心。下面 11 个主题是从明细里归纳出的「共因」——修一个主题 = 一次性拉齐几十个页面。建议优先按主题批量整改，而非逐页打补丁。

按严重度筛选：
01

弹窗规范失守：Modal 承载表单/向导（应为 Drawer）

P0

CLAUDE.md 明文："新建/编辑表单、详情、多步骤向导、批量导入一律用右侧抽屉，不再用居中 Modal"。实际 36 个文件含 Modal，其中承载表单/向导的违规点至少 15 处：DataSourceModal 三步向导、ProductTemplateBinding 两步向导、ConfiguratorPage 三步提交、PricingStrategy、ProductManagement 导入、CustomerLeadList 审核、ApprovalRule/Department/Region……

整改 → 抽 FormDrawer / WizardDrawer / ImportDrawer 三个共享壳批量迁移；Modal 仅保留 Modal.confirm 危险二次确认与 SelectableTable 批量确认（规范许可）。
02

列表规范失守：裸 Table + 行内动作按钮

P0

「行内不放动作按钮，变更/危险动作上提工具栏；统一 SelectableTable」是强制规范。实际大量列表仍是裸 <Table> + 行内 编辑/停用/删除/解绑/重置密码：客户线索、模板绑定（行内 Switch+删除）、公式表、配置器全列表、系统各管理页……全站仅 ApprovalRuleManagement 一个页面遵守。

整改 → 迁 SelectableTable，动作上提工具栏并声明 enabledWhen(rows)；危险动作走列出所选项的二次确认。
03

色彩体系失控：主色形同虚设 + 多套外来色板混入

P1

主色 #667eea（紫）只活在 Login 渐变与少数卡片；真正的 CTA/选中态/金额强调却散落 AntD 蓝 #1677ff（39 文件 63 处）。混入 Element-Plus 灰阶（#909399/#606266/#303133）、Material（#1976d2/#4caf50）、Chakra（#e53e3e/#48bb78）三套色板。组件簇同页并存三套蓝、模板簇四套蓝、危险色三写法。「金额合计」5 屏 5 色。

整改 → 定义语义 token（--cpq-primary:#667eea、colorSuccess/Error、单一「金额强调色」），全站 grep 替换硬编码 hex。覆盖面最广、性价比最高的一项。
04

emoji 当结构性图标

P1

约 40 个文件用 emoji 充当导航/按钮/状态图标，与 Ant Design 线性 SVG 割裂、跨 OS 字形不稳、读屏无语义：侧栏菜单 🛒 3D 选配 · 📋 · 🎯 · 🔗 · 📩、主 CTA 💾 保存模板、CostingPartData 图标 + emoji 双图标叠加、客户线索 📩⏳✅🚫、报价视图切换 Segmented 📝📊📈。

整改 → 结构性图标全部换 @ant-design/icons；emoji 仅作正文装饰性辅助。
05

内联样式 + 魔数泛滥，无 design token 层

P1

全站 2457 处内联 style={'{{}}'}、1272 处硬编码 hex、大量非阶梯字号（fontSize:10.5/11.5/12.5，低于 12px 可读下限且亚像素模糊）、脆弱的负边距与视口硬算（margin:-24px、calc(100vh-112px)——头部高度一变即错位，横跨组件/模板/定价多簇）。

整改 → 抽公共色/字号/间距 token 与 CSS class，内联仅保留一次性布局；字号归整到 12/13/14 阶梯。
06

缺「我在哪」导航：0 面包屑 / 0 PageHeader / 标题层级乱

P1

全站 Breadcrumb=0、PageHeader=0；176 页仅 32 页有标题（144 页无标题）。有标题的层级也不一：主标题散落 level={4}（20 处）与 level={5}（21 处），还有裸 <h2>，无统一规则、无 h1。Header 顶栏左侧完全空置。Hub 页双标题冗余（同屏两个「产品管理」）。

整改 → 约定统一页头组件（标题 level={4} + 副标题 + 面包屑）；Hub/Tab 子页去重复标题与外层 padding。
07

数字/金额列不右对齐、不等宽 + 精度口径不一

P1

数值列（单价/损耗率/年用量/折扣率/含量%）在核价、报价、主数据、定价多簇普遍左对齐、非 tabular-nums，无法按小数点纵向扫读。ElementPriceCenterPage 是全站唯一做对的样板。金额精度打架：toLocaleString() 无小数 vs formatCurrency 2 位；货币码 RMB 与 CNY 混用。

整改 → 数值列统一 align:right + tabular-nums；对外金额统一 formatCurrency（2 位）；货币码统一 CNY。
08

抽屉宽度不成体系

P2

规范建议 480 / 720 / 960 / 1200 四档，实际散落 420/520/560/580/600/640/680/760/780/840/900/920/1000/1080/1100 共 17+ 种。近半数抽屉宽度是随手拍的非标值，同类抽屉宽窄不一。

整改 → 归并到四档标准宽度。
09

无障碍普遍薄弱

P2

icon-only 按钮缺 aria-label/title 遍布（排序 ↑↓、删除、拖拽柄）。原生控件绕过设计系统：<input type=checkbox>、原生 <select>、原生 <table>、自绘勾选框 div.cmm-chk——均无焦点环/键盘/读屏语义。纯颜色/纯 emoji 传达状态；低对比灰 #bfbfbf/#ccc（≈1.6:1）；拖拽加组件无键盘替代。

整改 → 补 aria/Tooltip；原生控件换 AntD 组件；低对比灰提到 ≥#8c8c8c；状态补图标/文字。合规诉求下可上升 P1。
10

加载/空态不统一、工程细节泄漏

P2

裸 <Spin/>（无居中缩左上）反复出现；<Spin tip> 在 AntD v5 独立使用失效（4 处）；纯文字「加载中...」无骨架；CostingPartDataPage 出现「顶部大 Empty + 下方空 Tab」双重空态。工程细节泄漏给用户：Alert 暴露 VITE_USE_MOCK_MASTER_DATA、message 里字面显示 <br/>、原样渲染反引号。

整改 → 统一居中 Spin / Skeleton / <Empty>；修 v5 Spin tip；技术细节仅进日志。
11

新老两代代码并存（原型直转 vs 现行规范）

P2

现行规范基线（可作标杆）：system / system-monitor / system-config / change-log 四组、核价 master-data 主流程。原型直转重灾区：configurator（3D 选配）+ feature-library（emoji + 魔数 + #1890ff/#52c41a + 主 CTA 一律染绿 + mock「后端待实现」控件暴露给用户）、ModelConfig（Element-Plus 色板）、报价 qt-* 卡片（Material/Chakra）。死代码污染一致性：ComponentTree/HeaderPreview/CrossTabRefDrawer/ImportExcelModal 均 0 引用。

整改 → 以 system 簇为基线做一致性清扫，并清理死代码；AddProductModal 实现已是 Drawer，重命名消除误导。

核价模块

组件管理模块

P0 1P1 6P2 11

  • P1字段配置表 13 列无横向滚动，窄屏挤压换行

模板管理模块

P0 2P1 8P2 12

问题最密集的模块。弹窗与列表两条强制规范成片违反；主色失控最刺眼（品牌紫 + 三种蓝各占一角）；内联样式与 magic number 遍布。

  • P1四套蓝并存（#667eea/#1976d2/#1890ff/#1677ff）
  • P1主 CTA 💾 保存模板 是 emoji + 裸 button，与「发布」争焦

主数据 / 基础数据 / 导入

P0 0P1 8P2 27

Remediation Roadmap

三阶段修复路线（均不改功能）

「先补强制规范硬伤 → 再做 token 化收敛 → 最后打磨」。每阶段独立可交付、独立可回归。

1

清偿规范硬伤

P0 · PR 评审强制项

  1. Modal → Drawer：抽三个共享壳，迁约 15 处表单/向导
  2. 裸 Table → SelectableTable：迁约 8+ 处列表，动作上提 + enabledWhen
  3. 删死代码：ComponentTree / HeaderPreview / CrossTabRefDrawer / ImportExcelModal
  4. 重做首页：Dashboard 换成真正的工作台
⚠ 触及协议级文件时跑 Playwright E2E 复测
2

Design token 收敛

P1 · 覆盖面最广 · 性价比最高

  1. 主色 token：--cpq-primary:#667eea，grep 替换 63 处蓝 + 三套外来色板
  2. 金额：单一强调色 + 数值列右对齐 tabular-nums + formatCurrency 2 位
  3. emoji → AntD 图标（~40 文件）
  4. 抽屉宽度归四档（480/720/960/1200）
  5. 统一页头 + 面包屑 + 标题层级（主标题 level=4）
  6. 加载/空态统一（居中 Spin / Skeleton / Empty）
3

无障碍与响应式打磨

P2 · 收尾

  1. icon-only 补 aria/Tooltip；原生控件换 AntD 组件
  2. 低对比灰提到 ≥#8c8c8c；纯色/纯 emoji 状态补图标
  3. 字号魔数归 12/13/14；脆弱布局改 flex
  4. 全屏配置器补断点；宽表补 scroll={'{{x}}'}
  5. 侧栏加 collapsible；Hub 子路由高亮修正
