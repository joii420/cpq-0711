# 接口文档 · 核价导入数据版本升级与版本维护

> 结论：**对外接口契约不变（请求/响应/URL/字段均不变）**，仅**内部落库与升版行为**变化。前端无需改动。

## 1. 涉及接口（沿用，不改契约）

| 用途 | 方法 & 路径 | 变化 |
|---|---|---|
| 核价数据导入 | `POST /api/cpq/basic-data-import/v6/pricing`（沿用现有导入端点） | 契约不变；内部落库改为"按销售料号系统升版" |
| 报价数据导入（回归对照） | `POST /api/cpq/basic-data-import/v6/quote` | **不改**（本期报价侧不动） |
| 导入结果/进度查询 | 沿用现有查询端点 | 不变 |

> 实际端点路径以现有 `PricingImportService` / 导入 Resource 为准（实现时 grep 确认；本任务不新增/不改路由与 DTO）。

## 2. 行为变更说明（请求方无感，仅结果语义）

1. **忽略 Excel 版本列**：导入核价数据时，Excel 自带的"价格版本/计算版本/汇率版本/版本编号"**不再直存**为版本号；系统按内容自动生成版本（首版 `2000`、内容变 `max+1`）。
2. **按销售料号升版**：每个 sheet 对每个销售料号独立管理版本；(sheet,销售料号) 整组数据发生**值变化**才升版，旧版本 `is_current=false`、新版本 `is_current=true`。
3. **顺序无关**：同一份数据仅调整行顺序重导，**不产生新版本**。
4. **取数口径**：下游一律取 `is_current=true` 的当前版本（不变）。
5. `production_energy` 内部由 `energy_unit_price`/`depreciation_unit_price` 两列合并为 `unit_price`+`price_type`（`ENERGY`/`DEPRECIATION`），**仅影响后端表与视图**，不改导入接口出入参。

## 3. 响应结构（不变）

- 沿用现有导入响应：`sheets[]`（每 sheet 成功/失败行数）、`failedRows`、错误明细等，字段与结构不变。
- 版本号、is_current 等为**落库内部字段**，不在导入响应里新增暴露（如后续需要"本次各销售料号升到第几版"的回显，另行提需求）。

## 4. 兼容与错误

- 无新增错误码；沿用既有"销售料号必填/未匹配"等校验与 `recordError` 明细。
- 清空重导安全（幂等）：同数据重导不升版、不重复；值变才升版。
