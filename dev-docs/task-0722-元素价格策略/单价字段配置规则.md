# 单价字段配置规则 —— 接入客户元素价格策略

> **适用对象**：负责组件字段配置 / SQL 视图配置 / Excel 模板配置的 agent 与人工配置员。
> **目标**：让某个「单价」字段自动按**客户价格策略**取价，而不是让销售手填。
> **配套**：`需求说明.md` §11（澄清结论）、`原型图.html`（页面形态）。
> **状态**：2026-07-22 定稿口径。实现细节以 `backtask.md` 为准，若两者冲突以 `backtask.md` 为准并回头修订本文。

---

## 0. 一句话规则

> **在驱动视图里 `LEFT JOIN` 价格表函数，把它的 `unit_price` 列以中文别名输出，再把「单价」字段的取数路径指向该别名。字段类型一个字都不要改。**

---

## 1. 前置契约：价格表函数

价格计算逻辑**单点**落在一个 PG 表函数里，所有驱动视图共用。配置方**不要**自己写窗口聚合 SQL。

### 1.1 签名

```sql
f_customer_element_price(p_customer_no TEXT, p_base_date DATE)
RETURNS TABLE (...)
```

### 1.2 返回列（配置时可用的列）

| 列名 | 类型 | 含义 | 备注 |
|------|------|------|------|
| `element_code` | VARCHAR | 元素符号（Ag / Cu / 301…） | **JOIN 键**，对齐 `element.element_code` |
| — | — | *入参 `p_customer_no` 接受两类值*：真实客户编码（如 `CUST-1269`，报价侧用）或字面量 `_GLOBAL_`（核价侧用） | 策略表客户维度存 `customer_no VARCHAR`，故两者同构 |
| `unit_price` | NUMERIC(18,4) | **最终单价**（已含 × 系数 + 加价） | 无价时该元素**不出现在结果里**（不是返回 0） |
| `currency` | VARCHAR | 货币 | 来自价格记录 |
| `price_unit` | VARCHAR | 计价单位 | 来自价格记录，**不做换算**（§11.10） |

> **重要**：`unit_price` 已经是"可直接用的最终价"，配置方**不要**再乘系数或加价——那是策略表的职责，重复计算会导致翻倍。

### 1.3 行数量级

一次调用返回"该客户 × 该基准日"下**全部有价元素**，约几十行。是一次性求值的表函数，不是逐行调用，**不存在 N+1 性能问题**。

## 2. 三步配置法

### Step 1 —— 驱动视图 SQL：JOIN + 输出中文别名

在元素类驱动视图（如 `ys_view`）的 FROM 子句里加一段。**报价侧与核价侧传的客户参数不同**：

```sql
-- 报价侧（system_type='QUOTE'）：传运行时真实客户
LEFT JOIN f_customer_element_price(:customerCode, :priceBaseDate) cep
       ON cep.element_code = ebi.component_no

-- 核价侧（system_type='PRICING'）：传字面量 '_GLOBAL_'
LEFT JOIN f_customer_element_price('_GLOBAL_', :priceBaseDate) cep
       ON cep.element_code = ebi.component_no
```

> **为什么核价侧传 `_GLOBAL_`**：核价基础数据（BOM / 工序 / 费用）全部挂在 `customer_no='_GLOBAL_'` 下，
> 是**全局成本口径、不分客户**（实测：`element_bom_item` 的 PRICING 行 26/26 全为 `_GLOBAL_`）。
> 核价算的是成本，成本不因客户而异；报价算的是售价，两者差额即毛利。详见 `需求说明.md` §11.11。

SELECT 列表里加输出列（**中文别名必须与组件字段名逐字一致**）：

```sql
cep.unit_price  AS 单价,
cep.currency    AS 货币,
cep.price_unit  AS 计价单位,
```

### Step 2 —— 声明所需变量

`component_sql_view.required_variables` 数组按侧声明：

| 侧 | required_variables |
|----|--------------------|
| 报价侧（QUOTE） | `["customerCode", "priceBaseDate"]` |
| 核价侧（PRICING） | `["priceBaseDate"]` —— **不含 customerCode**，因为传的是字面量 `'_GLOBAL_'` |

> `priceBaseDate` 是本期新增的注入参数，**不声明会导致占位符落空、整表取不到价**。

### Step 3 —— 组件字段取数路径

**字段类型保持原样，一个字都不要改**（改了会触发 AP-44 的 17 点联动协议，代价极大）：

| 字段名 | 报价侧配置 | 核价侧配置 |
|--------|-----------|-----------|
| 单价 | `field_type` 维持 `INPUT_NUMBER`；<br>`default_source = {"type":"BASIC_DATA","path":"$ys_view.单价"}` | `field_type` 维持 `BASIC_DATA`；<br>`basic_data_path = "$ys_view.单价"` |
| 货币 | `field_type` 维持 `INPUT_TEXT`；<br>`default_source = {"type":"BASIC_DATA","path":"$ys_view.货币"}` | 同左，用 `basic_data_path` |
| 计价单位 | 同上，指向 `$ys_view.计价单位` | 同上 |

**报价侧 vs 核价侧的唯一区别**：报价侧字段是 `INPUT_*`，用 `default_source`（自动带出**默认值**，销售仍可改）；核价侧字段是 `BASIC_DATA`，用 `basic_data_path`（纯取数，只读）。这与两侧现状一致，**不要统一它们**。

## 3. 硬约束（8 条，违反即出 bug）

| # | 约束 | 违反后果 |
|---|------|---------|
| 1 | **必须 `LEFT JOIN`，禁止 `INNER JOIN`** | 无价元素的整行会消失，BOM 行数变少，页签小计错 |
| 2 | JOIN 键用**元素符号**（`element_bom_item.component_no`），不是元素编号（`element_no`） | 恒不命中，全表无价 |
| 3 | **禁止** `COALESCE(cep.unit_price, 0)` 之类兜底 | 无价必须是 NULL 才能触发"留空标黄让销售手填"；填 0 会静默算出 0 元成本 |
| 4 | **禁止**在视图里再乘系数 / 加价 | `unit_price` 已含，重复计算翻倍 |
| 5 | 中文别名与组件字段名**逐字一致**（含空格、全半角） | 路径解析不到，渲染空白 |
| 6 | 核价侧**传字面量 `'_GLOBAL_'`，禁止传 `:customerCode`**；核价视图 WHERE 里**不要**加 `customer_no = :customerCode` | 核价数据的 `customer_no` 恒为 `_GLOBAL_`，加客户过滤会**一行都查不到**（`_GLOBAL_ ≠ CUST-xxxx`）。⚠️ 本条曾在 v1 规则中写反，已于 2026-07-22 更正 |
| 7 | `required_variables` 必须同时含 `customerCode` + `priceBaseDate` | 占位符未绑定，视图执行报错或返 0 行 |
| 8 | 不要为"源名 / 命中日期 / 取值方式"等依据信息加输出列 | 本期已砍角标与 tooltip（§11.14D），加了也没人消费，徒增视图复杂度 |

## 4. 完整示例

### 4.1 报价侧（`ys_view`，`system_type='QUOTE'`）

```sql
SELECT
  child.hf_part_no,
  ebi.material_no   子件,
  ebi.seq_no        序号,
  ebi.component_no  元素,
  ebi.content       含量,
  ebi.issue_unit    单位,
  cep.unit_price    单价,        -- ← 新增
  cep.currency      货币,        -- ← 新增
  cep.price_unit    计价单位,     -- ← 新增
  mm.unit_weight    单重
FROM element_bom_item ebi
  INNER JOIN ( ... ) child ON child.子料号 = ebi.material_no
  LEFT JOIN material_master mm ON mm.material_no = ebi.material_no
  LEFT JOIN f_customer_element_price(:customerCode, :priceBaseDate) cep   -- ← 新增
         ON cep.element_code = ebi.component_no
WHERE ebi.system_type = 'QUOTE'
  AND ebi.is_current = true
  AND ebi.customer_no = :customerCode
  ...
```

### 4.2 核价侧（`ys_view` @ COMP-0040，`system_type='PRICING'`）

```sql
SELECT
  ebi.material_no      material_no,
  ebi.characteristic   AS view_version,
  ebi.seq_no           序号,
  ebi.component_no     元素,
  ebi.content          含量,
  cep.unit_price       单价,       -- ← 由原 gvv.value_number 改来
  ebi.issue_unit       单位
FROM element_bom_item ebi
  LEFT JOIN material_master mm ON mm.material_no = ebi.material_no
  LEFT JOIN f_customer_element_price('_GLOBAL_', :priceBaseDate) cep   -- ← 新增，注意传字面量
         ON cep.element_code = ebi.component_no
WHERE ebi.system_type = 'PRICING'
  AND :versionFilter(ebi.is_current, ebi.characteristic, ebi.material_no)
  -- 注意：这里【没有】customer_no 条件，且不要加 —— PRICING 数据恒为 _GLOBAL_
```

> **核价侧只有一处改动**：单价取数从全局变量 `COST_ELEMENT`（`global_variable_value`）改为 `cep.unit_price`。
> **不要**加客户过滤条件（约束 #6）。核价侧 `required_variables` 因此**只需 `priceBaseDate`**，不需要 `customerCode`。

## 5. 配置自检清单

配完逐条打勾：

- [ ] 视图 SQL 用的是 `LEFT JOIN`，不是 `INNER JOIN`
- [ ] JOIN 键是元素符号列（`component_no`），不是元素编号
- [ ] 没有对 `unit_price` 做 COALESCE / 乘系数 / 加价
- [ ] 中文别名与组件字段名逐字一致
- [ ] `required_variables` 按侧声明正确（报价侧两个 / 核价侧只有 `priceBaseDate`）
- [ ] 核价侧传的是字面量 `'_GLOBAL_'`，且视图 WHERE 里**没有** `customer_no` 条件
- [ ] 核价侧实测：换一个不同客户的报价单，核价元素单价**不变**（证明未受客户影响）
- [ ] 组件字段的 `field_type` **未改动**（报价 `INPUT_NUMBER` / 核价 `BASIC_DATA`）
- [ ] 报价侧用 `default_source`、核价侧用 `basic_data_path`，没有配反
- [ ] 用 expand-driver 端点实测：有价元素出数、无价元素返 NULL（不是 0、不是整行消失）
- [ ] 换一个**没配价格策略**的客户实测：全部元素单价为 NULL，且 BOM 行数与改动前一致

## 6. 适用范围

- 本期**只接通** `COMP-0029`（报价标杆）与 `COMP-0040`（核价标杆）两个组件（§11.19）。
- 其余元素类组件副本（`COMP-0029__imp1` / `COMP-0020__imp1` / `COMP-0020__imp1__imp1`）**由业务按本文自行配置**，配置方式完全相同，只需把示例里的视图名换成该组件自己的视图名。
- 本规则也适用于将来任何"想按客户策略取元素价"的新组件——照 §2 三步走即可，无需改任何代码。
