6策略试算（抽屉 720px）

定价管理 → 定价策略 → 选中客户 → 元素价格策略 Tab → 右上角「🧮 策略试算」
入口位置（本轮确定）：挂在「元素价格策略」Tab 内容区右上角，与"取值优先级"说明同一行。
  • 不挂在「客户级默认策略」卡片头上 —— 试算算的是默认 + 例外合并后的最终结果（结果表里"命中规则"列会标明每个元素走的是例外还是默认），挂在默认卡片里会让人误以为只试算那一条。
  • 试算按当前页面上的配置算（含尚未保存的改动），可以"先试算、满意了再保存"。
  • 试算只读、不落库，纯计算预览。
为什么要这个：策略是"窗口 + 取值方式 + 系数加价"的组合，光看配置看不出最终多少钱。试算让维护人在保存前后都能用指定基准日当场验算，也是验收时最直接的核对工具（对应验收要点 §11.21#5）。

8核价单取价效果（元素页签）

核价单 / 核价工作台 → 元素 页签
核价走 _GLOBAL_ 全局策略，与报价侧口径不同（§11.11，第 5 轮评审改写）。
  • 实测事实：核价侧数据全部挂在 customer_no='_GLOBAL_' 下（element_bom_item 的 PRICING 行 26/26）， 是全局成本口径、不分客户。故现有核价视图没有客户过滤不是遗漏、是设计。
  • 业务含义：成本归成本、售价归售价，两者差额即毛利。同一料号不因客户不同而核出不同成本。
  • 因此报价侧与核价侧的元素单价可以不同 —— 这是预期特性，不是 bug。
  • 未配 _GLOBAL_ 策略时，核价元素单价全空（不回退到报价单客户的策略）。
  • 这是一处替换：核价元素单价原取自全局变量 COST_ELEMENT（不分客户、不分日期的固定值）， 现改接价格表函数并传字面量 '_GLOBAL_'。旧全局变量数据保留不删，作回滚退路。
