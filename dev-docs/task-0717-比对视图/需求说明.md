## 2. 需求背景与目标 ⭐

- **要解决什么问题 / 痛点**：
   报价单中,用户在比对视图中,可以查看同个销售料号的报价侧和核价侧各个页签之间的比较, 并且默认比较各个销售料号的产品卡片总计

- **本次要达成的目标**：
  - 在比对视图,默认比较各个产品卡片的产品总计
- 在比对视图中,增加配置功能, 可以读取报价侧的所有页签和核价侧的所有页签, 然后用户可以选择一个报价侧的页签和核价侧的页签进行比对,将比对结果增加到比对列表的列中
  - 在配置功能中,可以设定差异阈值, 比对结果差异值小于阈值的在比对列表中的值显示为橙色, 将差异值<0的 标为红色
  - 一次配置增加一列,在比对列表中可以删除比对列,并且报价单的比对列和核价单打开的比对列单独保存管理,分别对应了销售和财务两种角色各自的自定义列
  - 一个销售料号分别有五行,上面是报价侧是标题行和数据行,下面的核价侧的标题行和数据行,加一行差异值(报价值-核价值)
  - 比对视图含有分页功能/销售料号的过滤功能/超过阈值的料号前置展示选项("差异料号")
  
- **不做什么（明确边界）**：
  其余功能模块不动

## 11. 需求澄清纪要（2026-07-17 · 技术总监定稿）

> 本节为需求讨论后的**最终共识**，与前文若有冲突以本节为准（已逐条与需求方确认）。

### 11.0 关键背景：本功能是「改造现有实现」而非全新开发
现系统报价单页面已存在 `报价单 | 核价单 | 比对视图` 三个 Tab，**「比对视图」Tab 已有一套实现**：
- 现有列 = 报价/核价两侧 Excel 模板**共有的 `comparison_tag`**（系统固定、用户不可配）；
- 现有行 = 料号并集，每料号 **2 行**（报价/核价），值取 Excel 视图列值；
- 差异仅单布尔高亮（`valuesDiffer` → 粉色）；有"仅看有差异"开关 + 导出。
- 相关文件：`comparisonModel.ts`(`buildComparisonModel`) / `comparisonTable.tsx`(`ComparisonTable`) / `ComparisonView.tsx`(编辑态,`useLinkedExcelRows`) / `ReadonlyComparison.tsx`(详情只读,冻结快照) / 后端 `CostingSheetService.buildComparison` + `GET /quotations/{id}/comparison` + `POST /comparison/export`。

**本次改造 = 把上述「列模型」整体换成「用户配置的页签对比列」**，并新增差异行、阈值双色、按角色持久化配置、分页/过滤/差异料号前置。

### 11.A 载体与入口
- 复用现有「比对视图」Tab，**不新建页面/路由**。
- 报价单**编辑页**（销售入口）与核价单页面（财务入口）：**可配置**（有"新增比对"按钮）。
- 报价单/核价单**详情页**：**只读 + 过滤**，**无**"新增比对"按钮。

### 11.B 列模型
| 列类型 | 取值 | 可删 | 阈值 |
|--------|------|------|------|
| 默认列「产品卡片总计」 | 各料号**产品卡片总计**（SUBTOTAL 组件的独立公式值，**不是**各页签小计的加总） | ❌ 永远在、不可删 | ✅ 可改，默认 0 |
| 用户列 | 报价侧（页签 → 字段小计 / 页签合计）与核价侧（页签 → 字段小计 / 页签合计）各选一，按添加顺序追加末尾 | ✅ 可删 | ✅ 每列可配，默认 0 |

- **"值"的两级粒度**：选定页签后，可选该页签内某个**字段小计**（`is_subtotal` 列）或该页签的**页签合计**。
- **页签目录来源**：卡片结构快照 `quoteCardStructure.tabs` / `costingCardStructure.tabs`；每页签可选项 = 该页签所有 `is_subtotal` 列 + 一个"页签合计"。
- **模板一致性**：一张报价单**报价侧一套模板、核价侧一套模板**，所有销售料号页签集统一（无料号级异构页签）。

### 11.C 行与布局（每销售料号 3 行块）
> 📌 2026-07-17 原型评审变更：由 5 行改为 **3 行**——去掉块内"报价·标题行 / 核价·标题行"，页签·小计名统一放在**列头**展示（列头分两行显示"报价：页签·小计 / 核价：页签·小计"），块内不再重复。
```
列头(每列)：报价：投料·材料小计
           核价：投料成本·材料费小计     ← 标题在列头
──────────────────────────────────────────────────────
料号 XXXX          列1「产品卡片总计」   列2（用户配）…
报价(数据行) →        15500                8000
核价(数据行) →        14500                7500
差异(差异行) →       +1000 🟠            +500
```
- **列头**分两行显示该列的报价侧与核价侧所选"页签·小计"名（默认列显示"产品卡片总计"）。
- 每料号块 3 行：报价数据行 / 核价数据行 / 差异行。
- **差异行** = 报价数据 − 核价数据，按 11.D 着色。
- **单边料号**（一侧有、另一侧无该销售料号）：缺失侧的数据行**整行变灰**，差异行显示"—"；且**计为差异料号**、前置优先级最高。对称处理（报价缺→报价数据行变灰，核价缺→核价数据行变灰）。

### 11.D 着色规则（**整格填色**标在差异格；差异值 = 报价值 − 核价值）
> 📌 2026-07-17 原型评审变更：着色由"仅字体色"改为**整个差异单元格填色**（红底白字 / 橙底白字），更醒目。
> ⚠️ 解决原文冲突：§2 写"<0 标红"、§8 写"<-1 标红" —— **最终取 `<0`**（§2），红线**全局固定**、不可配。
- 差异值 `< 0` → 🔴 **红**（固定红线，红优先）
- 否则 差异值 `< 阈值`（每列可配，默认 0）→ 🟠 **橙**
- 否则 无色
- 语义：红线抓"报价低于成本"的亏损；阈值抓"利润低于目标"的预警。默认阈值 0 时两档重叠 → 只显红；阈值调正数才出现橙区。

### 11.E 配置保存与生命周期
1. **保存粒度 =（报价单 × 入口桶）**：每张报价单各存一份，**非跨单全局模板**。
2. **桶（bucket）= 销售 / 财务，按入口页面区分**（报价单编辑页→销售桶，核价单页面→财务桶），**不看**登录用户真实角色。
3. **共享粒度 = 按桶共享**（非按具体用户）：同一张单、同一入口进来的人看到同一份配置。
4. 配置持久化；**只存列定义**（选了哪个页签/小计 + 阈值 + 顺序），**不存值**；值在打开视图时实时取。
5. **详情页**只读展示**同侧桶**已保存配置（报价单详情→销售桶，核价单详情→财务桶），不可增删/改阈值。

### 11.F 辅助功能
- **过滤框**：输入销售料号做**子串模糊匹配**（仅匹配销售料号本身）。
- **差异料号**：勾选开关后，把差异料号（任一列差异格标橙/标红 或 单边料号）**排序前置**（其余料号仍在下方可见，非过滤隐藏）。
- **分页**：按销售料号块分页（一个料号 5 行不切断），默认每页 10 个料号，**页大小用户可调**。
- **导出**：⛔ 本期**不做**。去掉比对视图上的导出按钮；后端旧 `POST /comparison/export` 端点保留不动、不再被调用。

### 11.G 架构决策（技术总监定，供备案）
- **端点三分**（详见 `api.md`）：
  - `GET …/comparison-view/meta` — 两侧「页签 → 可比对值(字段小计/页签合计)」目录，供连线抽屉。
  - `GET …/comparison-view/data[?frozen=]` — 逐销售料号、两侧、**逐页签全部小计** + 产品卡片总计的取值矩阵；前端据此映射任意列，**新增/删列不重拉**。
  - `GET/PUT …/comparison-view/config?bucket=` — 按（报价单×桶）读写列配置（JSONB 全量覆盖）。
- **单一数据源纪律**：`data` 的取值必须复用**渲染报价单/核价单 Tab 的同一后端服务**（`ComponentDataEffectiveRows`/`CardSnapshotService`/`FormulaCalculator`/`LineDiscountService`），严禁旁路取数（防 AP-50 双源漂移）。
- **数据源口径**：编辑态 `frozen=false`（当前有效值）、详情/已提交核价单 `frozen=true`（冻结快照）；两种口径都要与对应场景 Tab 展示值逐值一致。
- **前端职责**：组装列、算差异、着色、排序、过滤、分页、连线抽屉；不新增取值逻辑。
- **持久化表** `quotation_comparison_config`（一行 = 一个「报价单 × 桶」，`columns` JSONB）。
- 详见 `api.md` / `backtask.md` / `fronttask.md`；UI 以定稿原型 `prototype-比对视图.html` 为 1:1 基准。

### 11.H 原型定稿决策（2026-07-18）
- **新增比对抽屉 = 连线配置版**（否决初版下拉版）：左列报价单页签树、右列核价单页签树，**点击式连线**配对；每个页签展开「字段小计 + 页签合计」为可连接节点。
  - 三色层级：页签分组头=蓝带、字段小计=绿点、页签合计=橙点橙字。
  - 核价侧连接点置于**内侧（左缘）**、标签右对齐 —— 连线只走中间空隙、不穿标签。
  - **一对多连线：允许**（一个节点可连多条）。
  - 页签折叠：本期**不做**，默认全展开。
- **比对值显示格式**：末尾"小计/合计"前加间隔点（`加工费小计`→`加工费·小计`、`页签合计`→`页签·合计`），统一用于连线节点/已配对清单/主表列头。
- **交付形态**：连线版为唯一定稿形态（不保留下拉版双份维护）。
- 定稿原型文件：`prototype-比对视图.html`（连线配置版）。
