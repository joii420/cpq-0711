## 2. 需求背景与目标 ⭐

- **要解决什么问题 / 痛点**：
  用户已通过导入把核价基础资料落库（tesk-0709 已完成"导入即版本化"）。现在需要一个可视化维护入口，让用户按销售料号查询核价基础数据、修改内容并触发资料版本迭代、回看历史版本。

- **本次要达成的目标**：
  在"主数据维护"（`/master-data-hub`）中新增第 5 个页签 **料号核价**：
  1. 展示"有核价数据的销售料号"列表，支持搜索，每行显示品名/规格/尺寸 + 已配置 sheet 数汇总（N/16）+ 最近更新时间；
  2. 点击行开抽屉，抽屉内固定 16 个 tab（每个 tab = 一个"版本组"，即升版/编辑的最小单元），展示该料号该 sheet 当前版本（`is_current=true`）的数据列表；
  3. 每个 tab 顶部可切换历史版本号（纯只读查看历史）；
  4. 具编辑权限的用户可对当前版做**全表格自由编辑**（改值 / 增行 / 删行）并保存，保存时按内容指纹比对，内容真变才触发升版；
  5. 编码列（料号/工序号/元素码等）旁展示只读"名称"列（主表关联带出）。

- **不做什么（明确边界，见 C1/C7/C9 澄清）**：
  - 全局 4 表（P01 元素核价价格 / P02 材料核价价格 / P03 汇率 / P21 电镀方案）**不挂销售料号、本期不纳入**（C1）。
  - 历史版本**只做只读查看，不做回滚**（C7）。
  - 整组下线（删空一个版本组）走专门 API，**不在本期**（C5）。
  - ⚠️ **边界修订（C9）**：原始需求写"只维护已导入结果"，澄清后**扩展为**"固定 16 tab + 空 tab 允许从零新建该 sheet 第一组数据（生成 2000 版）"，即含少量"新建"能力，不再局限于"仅编辑已导入行"。

### 4.2 界面与交互（前端相关）

- **宿主**：`MasterDataHubPage.tsx` 新增第 5 个 Tab「料号核价」。
- **列表**（C3/C4）：数据源 = 至少在一张 PRICING 版本化表有 `is_current=true` 数据的销售料号；列 = 品名/规格/尺寸 + 「已配置 N/16」汇总 + 最近更新时间；支持按料号/品名搜索。按 `docs/列表操作规范` 用 `SelectableTable`（本页以"点行进抽屉"为主入口）。
- **抽屉**（右侧 Drawer，宽度 960/1200）：顶部料号信息；固定 16 个 tab（C2/C9）。
- **每个 tab**：
  - 顶部：版本号切换下拉（列当前版 + 历史版，含操作人/时间/来源，C11）；当前版可编辑，历史版只读（C7）。
  - 表体：该版本组行表格；编码列旁只读名称列（C8）；编辑态支持改值/增行/删行（C5），编码列走主表下拉选择（C12）。
  - 空 tab（无数据）：允许"从零新建"，渲染空表单 + 按列结构元数据新增行（C9）。
  - 保存：整组提交 → 后端指纹比对 → 变则升版、不变不产生新版本（C6）。

### 4.3 业务规则与计算逻辑（后端相关）

- **取数**：永远取 `is_current=true` 最新版；历史版按版本号过滤该组 `is_current=false` 行。
- **保存/升版**（C6）：复用 `VersionedV6Writer`，整组 multiset 指纹比对：内容未变→复用旧版本号不写库；任一值变→旧组 `is_current=false`、新组 `is_current=true`、版本号 = 该轴 `MAX(数字版本)+1`；首组 = `2000`。核价侧无触发列，无"金额原地更新不升版"分支。
- **列结构元数据**：后端为 16 个版本组各提供一份列定义（列名、类型、轴/子维度/值分类、是否编码列及其名称关联主表），供前端渲染 + 新建校验。
- **名称带出**（C8）：料号→`material_master.material_name`、工序号→`process_master`、元素码/组成件→`element`；只读、随编码联动、不进指纹比对。
- **护栏**：轴（料号/price_type 等）锁定不可改；至少留一行（整组清空拒绝）；并发同轴 `pg_advisory_xact_lock` 串行化 + 建议乐观版本校验（提交携带期望当前版本号，若已被他人升版则拒绝）。
- **接口纪律**：禁止 for 循环嵌套查库，一次查询内存计算（沿用 tesk-0709 约束）。

### 4.4.0 编码/枚举列下拉数据源映射（C12/C13 定稿）

**A. 有主表 → 下拉选择（选中带出名称，名称只读、不进指纹比对）**

| 编码列 | 出现在 | 下拉源 | 显示 |
|--------|--------|--------|------|
| 工序号 `operation_no`/`process_no` | P13/14/18/23、产能、工时单价、折旧、能耗、辅助能耗、模具工装、物料BOM | `process_master` | `process_no` + `process_name` |
| 元素 `component_no` | P07 物料与元素BOM | `element` | `element_code` + `element_name` |
| 来料料号 `code` | P15/16/17 | `material_master` | `material_no` + `material_name` |

**B. 枚举列 → 固定枚举下拉（无字典表，值取自现有导入/handler 或 CHECK 约束枚举）**
> ⚠️ **严格校验（2026-07-12 定稿，撤销原"未知可输入回退"）**：保存时后端校验每个 ENUM 列的值必须在该列 `options`（含 CHECK 约束枚举）内、BOOLEAN 列必须是布尔值，非法值返 400 并指明列；前端一律用固定 `options` 下拉、**不允许自定义输入**。
`currency`（CNY/USD/EUR…）、`unit`、`calc_type`（材料/元素）、`production_type`、使用特性 `component_usage_type`、`is_effective`（是/否）。

**C. 无主表 → 自由文本（C13）**
- 要素名称 `cost_type`（P16/17/19/20）：自由文本；名称尽力解析。
- P22 电镀费类型 `cost_type`：固定二选一（电镀加工费 / 电镀材料费）。
- 模具编号 `tooling_no`（P12）：自由文本。
- 物料BOM 组成件 `component_no`（P06，按 `calc_type` 可能是料号或元素）：本期自由文本（名称尽力解析），按 `calc_type` 动态下拉列入后续增强。

## 8. 验收标准 ⭐

1. 主数据维护出现「料号核价」第 5 tab；仅授权角色可见（PRICING_MANAGER 能进）。
2. 列表只显示有核价数据的销售料号，可搜索，正确显示品名/规格/尺寸 + N/16 + 最近更新时间。
3. 点行开抽屉，固定 16 tab；有数据 tab 正确显示当前版数据 + 编码列旁名称列。
4. 版本切换：能列出历史版本（含操作人/时间/来源），切历史版为只读，切回当前版可编辑。
5. 编辑保存：改值/增行/删行后保存 → 内容变→升版（旧版 `is_current=false`、新版 `is_current=true`、版本号 +1）；内容未变→不产生新版本。
6. 空 tab 从零新建：填入首组数据保存 → 生成 `2000` 版、`is_current=true`。
7. 编码列走主表下拉、选中带出名称；无编辑权角色进入为只读。
8. 后端接口零 for 循环嵌套查库；并发编辑不产生双 current / 重复版本号。
