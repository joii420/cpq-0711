### ⚠️ 重大变更声明（相对原 spec）
原 spec `2026-07-12-核价导入自动补工序主表-design.md` 采用**方案 A**：*核价导入时读 Excel「工序名称」列 → upsert `process_master`，仅工序*。
经与需求方（技术总监）逐条澄清，**整体重构为方案 B + (ii)**：
- **主数据先行、导入不写主表**（方案 B）：名称由「主数据维护 / 产品管理」各权威主表提供，维护页**纯 join 解析**；核价导入 handler **一律不写主表**。

## 1. 问题（symptom）

核价基础数据维护页（`PricingMaintenanceService.readRows`）对编码列靠 `LEFT JOIN` 主表带出名称。**导入落库的编码体系与主表不相交 → 名称列 null**：
- 工序：导入落 `Z002/Z008/Z053/Z490/Z611`，`process_master` 只有 `K*/MRO*` → 工序名 null（BL-0045 原始症状）。
- 材质料号：维护页当前 `subDimReadonly` **根本没做名称解析**（只显示编码）。
- 元素 / 来料料号：视主表覆盖情况可能 null。

## 2. 澄清决策（5 问逐条备案）

| # | 问题 | 结论 |
|---|------|------|
| Q1 | 同一工序编号跨 sheet 名称不一致时听谁的 | **(a)** 首个 sheet 胜出、不告警（insert 幂等，同码不同名属源数据问题，不在本期兜） |
| Q2 | 只补工序还是四码全补 | **(b)** 四码全补：工序 / 元素 / 材质 / 料号 |
| Q3 | 名称补齐机制方向 | **(B) 主数据先行**：导入 handler 不写主表；名称由权威主表提供，维护页纯 join 解析 |
| Q4 | 已导入码怎么进主表 | **(ii)** 给**工序**主数据建批量导入（对齐材质库）；**元素不新建导入**——材质库导入已 `syncElementMaster` 落 `element` 主表 |
| Q5 | 材质料号怎么带出材质名 | 从材质库 `material_recipe` 查；路径 **`material_part_no → material_master.material_recipe_id → material_recipe.name`（两跳）** |

**三个已确认的小默认**：
1. **工序导入写入语义 = upsert 覆盖**（`ON CONFLICT(process_no) DO UPDATE`，重导刷新名称；与材质库 `material_recipe` upsert 一致）。
2. **工序导入模板字段**：`工序编号`(必填、唯一) + `工序名称`(必填) 两列必需；其余 `process_category / is_outsource / standard_currency / standard_unit / default_defect_rate` 为**选填列**。
3. **未维护 / 未绑定码**：名称列显示灰字「未维护」/「未绑定」，**不阻断**保存/浏览。

## 4. 本期范围

**真开发 2 项**：
1. **工序主数据批量导入**（后端 service + 端点 + 模板；前端 Drawer + 入口）→ 灌入工序码+名，工序名 join 立即解析。
   - ✅ **无需 Flyway 迁移**：`uq_process_master_no ON process_master(process_no)` 唯一索引**已存在**（V218:142），`ON CONFLICT (process_no)` 直接可用。（原 spec C1 多余，撤销。）
2. **维护页材质名两跳 join**（后端 `readRows` 特判 + 前端渲染「材质名」列 + 未绑定灰字）。

**核对 2 项（不改代码，验收时确认）**：
3. 元素：核对导入 `component_no` 与 `element.element_code`（符号）键对齐——材质库导入已落 `element`，若键不齐需在验收记录中标注。
4. 料号：核对 `来料料号` 在 `material_master` 覆盖率——P05/P06/P24 已 upsert，抽查非 null。

## 5. 非目标（本期不做）

- ❌ **不改核价导入 10 个 P-handler**（守 B：导入不写 `process_master`/`element`/`material_recipe`/`material_master`）。
- ❌ 不建元素批量导入（材质库导入已覆盖）。
- ❌ 不做工序/元素/材质的编码归一或映射表。
- ❌ 材质料号未绑定 `material_recipe_id` 的**补绑**不在本期（属「材质管理→绑定料号」的数据维护动作）。
- ❌ `process_master` 的选填列（类别/外协/单位/币种/不良率）不强制填，导入模板留空即写 null。

## 6. 验收标准（AC）

| # | 标准 |
|---|------|
| AC-1 | 工序主数据导入：上传含「工序编号+工序名称」的 xlsx → `process_master` upsert（新增 + 覆盖名称），返回报告（总行/新增/更新/跳过明细/耗时）；**连续重导两次幂等**（无重复行）。 |
| AC-2 | 导入后，维护页各含工序码 tab 的「工序名」**非 null 且正确**（`Z002/Z008/Z053/Z490/Z611` → 铣割/成品清洗/…/无缝焊接）。 |
