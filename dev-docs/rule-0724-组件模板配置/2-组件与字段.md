## 2.1 组件模型

- `component`（组件=页签）：`fields` JSONB / `formulas` JSONB / `excel_columns` JSONB / `data_driver_path`($view) / `component_type`(NORMAL/SUBTOTAL/EXCEL) / `row_key_fields` / `tab_type`/`part_no_field`/`part_name_field`/`sort_field`。
- `component_directory`（目录=一套模板的组件容器）。
- `template` + `template_component`（模板引用组件，不复制结构）。

## 2.2 字段类型 field_type

| 类型 | 用途 | 判定 |
|---|---|---|
| `INPUT_TEXT` | 标识/名称/单位/货币/料号 | 文本 |
| `INPUT_NUMBER` | 量/率/金额/厚度/汇率 | 数字；金额加 `is_amount=true`(content="0"，汇率"1") |
| `LIST_FORMULA` | 列表公式 | （详见 §5 + AP-37） |
| `FORMULA` | 行内公式列 | `formula_name` 指向 formulas |

## 2.3 default_source 绑定

- 结构：`{"path":"$view._列","type":"BASIC_DATA"}`。
- 无源字段（销售手填）：不给 default_source，`content` 给默认。

## 2.4 rowKeyFields（行键）

- 值 = `fields[].name`（不带 `_`）；多行可编辑组件的行唯一键。
- 撞键 = editRows 塌缩（末值×行数）；唯一化加 `#序号`（见常见坑）。
- 一号多材质：材质字段必须进 rowKeyFields。

## 2.5 `_` 前缀别名铁律

- SQL 视图**所有业务列别名加 `_` 前缀**（避免业务列被隐式 JOIN 误当谓词注入）。
- **唯 `hf_part_no` 不加 `_`**（驱动键）。
- 字段 `name` 保持中文原名（不带 `_`），`default_source.path` = `$view._列`。
