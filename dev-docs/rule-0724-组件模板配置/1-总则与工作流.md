## 1.1 配置前必读

- 环境：DB `10.177.152.12:5432/cpq_db`；账号 `admin/Admin@2026`（SYSTEM_ADMIN）。
- 大前提：报价模板**统一** `INPUT_TEXT`/`INPUT_NUMBER` + `default_source` 绑 `$view`；**禁止在 DB 建物理视图**（用组件 SQL 视图 `$name`）。

## 1.2 🚨 每组件配置前必问用户 4 项（强制交互，禁止自己猜）

从客户 Excel 推出组件 + 字段后，**逐个组件**用 `AskUserQuestion` 问、让用户勾选：

1. **页签类型 `tabType`**：BOM / 材质元素 / 零件 / 外购件 / 主件 / 不设(空)
2. **料号列 `partNoField`**：从该组件字段名里勾一个（非 BOM/费用页签必填）
3. **料号名称列 `partNameField`**：从字段名里勾，可不设
4. **行键 `rowKeyFields`**：从字段名里**多选**

> 禁启发式猜（实证返工：`来料` 曾被猜零件实为 BOM；`自制加工费` 料号曾绑 `finished_material_no` 实为 `code`）。

## 1.3 🔑 料号列绑定值铁律

`partNoField` 绑的列**必须是该 `tabType` 语义层级的料号，禁绑成品/主件料号**。

| tabType | 料号列语义 | 应绑的列（V6） |
|---|---|---|
| 主件 | 成品/产品料号 | `material_master.material_no` |
| 材质元素 | 材质料号 | `element_bom_item.material_part_no` |
| 零件 | 组成件料号 | `material_bom_item.component_no`；`unit_price` 源取 **`code`**（非 `finished_material_no`=成品） |
| 外购件 | 外购组成件料号 | `material_bom_item.component_no`（OUTSOURCED） |

> 原因：`BomNodeTypeResolver` 把每个有类型页签 `partNoField` 取值收集为该类型候选料号；零件页签绑成成品料号 → 树把成品节点误判零件。
> `unit_price` 列语义：`code`=零件料号、`finished_material_no`=成品料号；常同值但语义不同（零件加工费上卷父成品时分叉）——按语义绑 `code`。

## 1.4 交付方式

- **API**（配置/调试）：`POST /auth/login` → `POST /component-directories` → `POST /components` → `POST /components/{cid}/sql-views` → `PUT /components/{cid}/driver-view`。
- **JSON 导入**（标准交付）：暂存目录建组件 → `GET /component-directories/{id}/export` → 用户建目录 → `POST /component-directories/{新id}/import/commit?conflictPolicy=RENAME`。
