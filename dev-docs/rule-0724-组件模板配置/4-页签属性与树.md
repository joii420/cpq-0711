## 4.1 tabType / partNoField / partNameField（task-0721）

- `tabType`：`BOM`/`材质元素`/`零件`/`外购件`/`主件`（5 类，非法值 400，可空）。
- `partNoField`：料号列字段名，`{材质元素,零件,外购件,主件}` 4 类必填（缺 400）。
- `partNameField`：料号名称列字段名，可空。
- `tabType→characteristic`：材质元素→RECIPE、零件→ASSEMBLY、外购件→OUTSOURCED；主件=成品/树根；BOM=树页签。
- `tabType='BOM'` 保存自动 `bomRecursiveExpand=true`；被 COSTING 引用会 400 → 报价树须新建专用组件。
- 类型判定严格按 `partNoField` 显式取值，禁按字段名启发式猜。

## 4.2 sort_field（行排序，task-0722）

- 多行页签按数字/序号正序显示 → 用 `sort_field`（= `fields[].name` 之一）。
- 快照组装层（ConfigureSnapshotService）按其**数字感知升序**（数字段数字序、文本段字典序）；树页签按树序不排。
- 原因：视图 ORDER BY 在 DRAFT pending 管线下失效（见 §3.6）。
- 存 `component.sort_field` 列（V353）；设后 refresh-snapshot 生效。

## 4.3 树契约（BOM 页签）

- 树页签视图输出 `material_no`(本节点子件号) + `parent_no`(父号)，按 `(parent_no, material_no)` 边键挂数据。
- 报价树：`:total_material_no` 契约（见 报价侧 §树）。
- 核价树：spine 全节点渲染，`costing_bom_tree_config` 递归 SQL（见 核价侧 §树）。

## 4.4 两侧树契约差异

| | 报价 | 核价 |
|---|---|---|
| 绑定变量 | `:total_material_no` | spine（`:production_part_nos` + versionFilter） |
| 渲染 | Pass2（仅 tabType=BOM）；数据页签走 Pass1 平铺 | 每组件出整棵 spine，缺数据补空行 |
| 配置 | `costing_bom_tree_config` usage=QUOTE | usage=COSTING |
