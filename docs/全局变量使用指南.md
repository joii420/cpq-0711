# 全局变量使用指南

> 2026-05-18 V190 重构后版本。配套阅读：[配置中心架构](./配置中心架构.md)。

## 一、什么是「全局变量」

「全局变量」= **轻量、业务无关的配置项**（默认值 / 阈值 / 系数 / 折扣率 / 默认成材率…）。

它**不是**：
- ❌ 业务主数据（如核价价格 — 那是核价模块的事）
- ❌ 用户操作记录（如订单状态 — 那是业务流转）
- ❌ 计算逻辑（如毛利计算公式 — 那是公式定义）

> 📐 全局变量的 schema 层定义（value_source_type / LOOKUP_TABLE / SCALAR / KV_TABLE 等字段语义）详见《配置中心架构.md》§三，本文侧重新建/维护/引用的操作流程。

## 二、形态分类

### LOOKUP_TABLE — 按 key 查表

例：工序默认成材率 — 按 process_code 查。

```sql
-- 注册
INSERT INTO global_variable_definition
  (code, name, var_type, key_columns, value_column, value_source_type, visibility, unit)
VALUES
  ('PROCESS_DEFAULT_YIELD', '工序默认成材率', 'LOOKUP_TABLE',
   '["process_code"]'::jsonb, 'value_number', 'KV_TABLE', 'PUBLIC', '%');

-- Seed 值
INSERT INTO global_variable_value (var_code, key_id, key_values, value_number) VALUES
  ('PROCESS_DEFAULT_YIELD', 'Z350', '{"process_code":"Z350"}'::jsonb, 15.0000),
  ('PROCESS_DEFAULT_YIELD', 'Z029', '{"process_code":"Z029"}'::jsonb, 100.0000);
```

### SCALAR — 单值

例：系统默认毛利率 = 20%。

```sql
INSERT INTO global_variable_definition
  (code, name, var_type, key_columns, value_column, value_source_type, visibility, unit)
VALUES
  ('SYSTEM_DEFAULT_MARGIN', '系统默认毛利率', 'SCALAR',
   '[]'::jsonb, 'value_number', 'KV_TABLE', 'PUBLIC', '%');

-- SCALAR 的 key_id 用 '_' 占位
INSERT INTO global_variable_value (var_code, key_id, key_values, value_number)
VALUES ('SYSTEM_DEFAULT_MARGIN', '_', '{}'::jsonb, 20.0000);
```

> SCALAR 当前后端 `upsertEntry` 抛 400，待补充支持。如临时需要可手工 SQL。

## 三、组件字段引用全局变量

### 场景 A：作为输入框默认值（INPUT_NUMBER + default_source）

```jsonc
{
  "name": "成材率",
  "field_type": "INPUT_NUMBER",
  "content": "100",                          // 静态兜底（DB 查不到时）
  "default_source": {
    "type": "GLOBAL_VARIABLE",
    "code": "PROCESS_DEFAULT_YIELD",
    "key_field_refs": {}                     // 留空 → 同名映射 (key 列 process_code → driver row.process_code)
  }
}
```

用户体验：
- 报价单打开时 placeholder 显示「默认 15 · PROCESS_DEFAULT_YIELD」
- 用户输入即覆盖（覆盖值只活在当前草稿）
- 公式取值用三级兜底链

### 场景 B：作为公式 token（直接计算）

```jsonc
{
  "name": "本工序成本",
  "field_type": "FORMULA",
  "formula_tokens": [
    {"type": "global_variable", "code": "PROCESS_DEFAULT_PRICE",
     "key_field_refs": {"process_code": "工序代码"}},
    {"type": "operator", "value": "*"},
    {"type": "field", "value": "数量"}
  ]
}
```

公式引擎 (`FormulaCalculationService.resolveGlobalVariable`) 会按 def.value_source_type 自动分发到 KV_TABLE 或 COSTING_VIEW。

### 场景 C：作为只读展示（BASIC_DATA + global_variable_code）

V109 旧形态，仍兼容。建议新场景用 default_source 或 FORMULA。

## 四、加变量的 SOP

> G1~G3 已落地 — UI「新建变量」按钮可用 (2026-05-18).

### 路径 A：纯 UI（推荐 90% 场景）

1. 打开「全局变量管理」页 → 点「新建变量」
2. 填写表单：
   - **code**: 大写英文+数字+下划线 (如 `SYSTEM_DEFAULT_MARGIN`)
   - **name**: 中文名 (如「系统默认毛利率」)
   - **变量类型**: `LOOKUP_TABLE` (按 key 查) 或 `SCALAR` (单一值)
   - **key 列名**: LOOKUP_TABLE 必填 (如 `process_code`), SCALAR 留空
   - **单位**: 选填 (如 `%`、`CNY/KG`)
3. 创建后行内点「维护数据」添加值
4. 组件管理给目标字段配 `default_source` (G3 编辑器)

### 路径 B：Flyway 注册（仅初始化 / 数据迁移场景）

1. **新建 Flyway 迁移**（V200+）注册定义 + Seed 值（参考第二节模板）
2. **修改 component.fields JSON** 添加 default_source 或 formula_tokens 引用
3. **同步 template.components_snapshot** 引用该组件的所有模板
4. **触发 Quarkus 重启**（touch java 文件）让 Flyway 跑
5. **强刷浏览器**验证 placeholder / 公式结果

### 自检 checklist

```
□ global_variable_definition 行已 INSERT (visibility=PUBLIC, value_source_type=KV_TABLE)
□ global_variable_value 至少 1 行 (按 distinct key 全 seed, 避免漏)
□ component.fields JSON 含 default_source 或 formula_tokens
□ template.components_snapshot 已同步 (避免老报价单看不到新配置)
□ 强刷后 placeholder 显示「默认 X · CODE」 (有后缀 = DB 命中; 无后缀 = content 兜底)
□ 「全局变量」页能看到新变量且可「维护数据」
```

## 五、维护数据

打开「配置中心 → 全局变量」：

1. 列表里找到变量（默认隐藏 COSTING_INTERNAL 核价 3 张，toggle 可查看）
2. 点「维护数据」抽屉
3. 表格列出所有 (key, value) 行
4. 「新增」/「编辑」/「删除」二次确认 modal
5. 保存后**立刻生效**（缓存自动失效闭环 `invalidateDependentCaches`）

### 核价 3 张（ELEM_PRICE / MAT_PRICE / EXCHANGE_RATE）

- 在「全局变量」页**默认隐藏**（toggle 显示），visibility=COSTING_INTERNAL
- 「维护数据」按钮**禁用**，tooltip 提示「请到核价模块页面维护」
- 在公式 Picker 里**仍可作为变量引用**（带「核价」紫色 tag）

## 六、常见坑

| 坑 | 现象 | 修法 |
|---|---|---|
| Seed 只覆盖样本 key | placeholder 显示 content 兜底（无 ` · CODE` 后缀） | `INSERT ... SELECT DISTINCT key FROM mat_xxx ON CONFLICT DO NOTHING` |
| 模板被拆过 | 只同步一个 snapshot，另一个仍持旧版 | DO $$ 循环 ARRAY[id1, id2, ...] 一次刷所有 |
| 写入后缓存未失效 | UI 改了但 placeholder 没变 | 走 `GlobalVariableService.upsertEntry`（已含 `invalidateDependentCaches`） |
| 白名单字段拷贝漏字段 | 前端拿不到 default_source 配置 | 检查 enrichComponentData / buildComponentDataFromTemplate / buildField 三处都拷 |
| key_field_refs 写组件字段名 | driver row 取值返 null | 留空 → 同名映射；或显式映射 key 列 → driver row 物理列名 |
| 历史 V184 散字段 (default_basic_data_path) | 新代码不识别 | V191 已自动迁移；新场景一律用 default_source |
