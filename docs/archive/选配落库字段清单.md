# 选配产品 · 落库与渲染全流程文档（独立产品 SIMPLE / 组合产品 COMPOSITE）

> 用途：完整说明"选配生成报价单产品"时，**每个字段以什么规则、落到哪张表的哪个字段**，含**料号指纹匹配 + 自动生成**，以及最后**报价单产品卡片如何通过模板/组件查询基础数据渲染选配结果**。
> 来源：全部取自代码 + 库实测（`ConfigureProductService.java` / `FingerprintCalculator.java` / `AutoAllocatePartNoProvider.java` / `component_sql_view` / 模板 `components_snapshot`）。

## 0. 三个阶段总览

```
[选配向导提交] ConfigureProductRequest
        │
   阶段A  料号解析与落库  configure() → resolvePart()（每配件）+ COMPOSITE 父级
        │   ├─ 指纹匹配(命中→复用旧料号) / 未命中→自动生成新料号
        │   └─ 写：料号身份 / 元素 / 材质 / 子配件(ASSEMBLY) / 工序 / 组合工艺
        │
   阶段B  报价行创建  buildLineItems() → insertLineItem()
        │   └─ 写 quotation_line_item（SIMPLE 1 行 / COMPOSITE 父 1 + 子 N）
        │       + per-quote 工序(quotation_line_process) + per-quote 组合工艺
        │
   阶段C  报价单卡片渲染（前端）
            模板 componentsSnapshot → enrichComponentData → useDriverExpansions
            → POST /components/batch-expand → 各组件 data_driver_path 视图($cz/$ys/$gx/$zpj)
            → 视图读 V6 基础表(material_bom_item/element_bom_item/unit_price)按 customer_no+料号过滤
            → ComponentCell 渲染每个字段
```

**核心规律**：卡片每个页签显示的 = 它的 `data_driver_path` 视图按 `(customer_no, 料号)` 查 V6 基础表的结果。**所以"选配是否把某字段落到该视图读的 V6 表"直接决定页签是否有数据。**

## 1. 料号身份：指纹匹配 + 自动生成

### 1.1 指纹算法（`FingerprintCalculator`，决定"是否复用已有料号"）

| 产品类型 | 指纹输入串 | 维度 |
|----------|-----------|------|
| 独立 SIMPLE | `sha256("v1\|SIMPLE\|{recipeCode}\|{元素排序 code=pct,...}")` | 材质配方 + 元素含量（**不含工序**，工序改 per-quote） |
| 组合 COMPOSITE | `sha256("v1\|COMBO\|{子件料号排序,...}")` | 子配件料号集合 |

- 元素/子件先排序再拼接 → 顺序无关；`BigDecimal.stripTrailingZeros` 防 `12` vs `12.0` 误判。
- ⚠️ 现状：`configure()` 调的是 **2 参 `simpleFingerprint(recipeCode, elements)`**（不含 processIds）。即"同材质同元素、不同工序" = **同一料号**。

### 1.2 匹配 / 复用（`lookupHfByFingerprint`）

```
查：SELECT part_no FROM mat_part WHERE config_fingerprint = :fp   （V44 mat_part 为权威）
命中 → 复用该 part_no（reused），不新建
未命中 → 自动生成新料号
```

### 1.3 自动生成新料号（`AutoAllocatePartNoProvider`）

| 类型 | 料号格式 | symbol 取值 | 流水来源 |
|------|----------|------------|----------|
| 独立 SIMPLE | `CFG-{symbol}-{6位}` | `recipe.symbol`（如 AgNi → `CFG-AgNi-000001`） | `part_no_sequence`（SELECT…FOR UPDATE 行锁 + `next_val`+1） |
| 组合 COMPOSITE 父 | `CFG-COMBO-{6位}` | 固定 `"COMBO"`（→ `CFG-COMBO-000001`） | 同上 |

> existing 模式（用户选已有料号）：不生成、不算指纹新建，直接用 `existingHfPartNo`，必要时 backfill V6/V44/工序（见 §4.6）。

## 2. 字段级落库矩阵（当前真实行为）

> 列含义：**请求字段** = 前端入参；**落库表.字段** = 当前 INSERT 写入；**渲染视图读** = 页签 driver 视图实际读的源；**状态** ✓对齐 / ✗缺口 / △组合父级问题。
> ⚠️ 选配现在是"V44 + V6 双写"过渡期：V44(`mat_*`) 为指纹/复用权威，V6(`material_*`/`element_*`) 为渲染源。

### 2.1 料号身份

| 请求字段 | 规则 | 落库表.字段（V44） | 落库表.字段（V6） | 渲染 | 状态 |
|----------|------|--------------------|--------------------|------|------|
| —（生成/复用的 hfPartNo） | 见 §1 | `mat_part.part_no` | `material_master.material_no` | 卡片头/快照 | ✓ |
| `productType` | SIMPLE/COMPOSITE | `mat_part.product_type` | `material_master.material_type`（COMPOSITE 父写 'COMPOSITE'） | — | ✓ |
| 指纹 | §1.1 | `mat_part.config_fingerprint`（ON CONFLICT 幂等） | `material_master.config_fingerprint` | — | ✓ |
| `unitWeightGrams` | 仅新建写 | `mat_part.unit_weight` | `material_master.unit_weight`（COMPOSITE 父=NULL） | 元素页·单重 | ✓ |
| `recipeCode`→recipe.id | custom 时 | `mat_part.material_recipe_id` | `material_master.material_recipe_id`（COMPOSITE 父=NULL） | 材质页·材质代码 | ✓ |

### 2.2 元素（custom 配件的元素含量）

| 请求字段 | 规则 | 落库表.字段（V44） | 落库表.字段（V6 — 渲染源） | 渲染列 | 状态 |
|----------|------|--------------------|----------------------------|--------|------|
| `elements[].elementCode` | 每元素 1 行，seq 递增 | `mat_bom(bom_type='ELEMENT').element_name` | `element_bom_item.component_no` | 元素页·元素 | ✓ |
| `elements[].pct` | 含量% | `mat_bom.composition_pct` | `element_bom_item.content` | 元素页·含量 | ✓ |
| （固定） | — | `mat_bom.part_version=2000` | `element_bom_item`: system_type='QUOTE', customer_no=客户code, hf_part_no=material_no=料号, characteristic='2000', seq_no | — | ✓ |
| 单位 issue_unit | — | — | **未写** → 元素页·单位 空 | 元素页·单位 | ✗(小) |

### 2.3 材质（自定义材质料号的"自指物料行"）

| 请求字段 | 规则 | 落库表.字段（V6 — 渲染源 `$cz_view`） | 渲染列 | 状态 |
|----------|------|----------------------------------------|--------|------|
| `recipeCode`→recipe.symbol | 幂等补一行"自指物料行"（WHERE NOT EXISTS） | `material_bom_item`: material_no=component_no=料号(自指), **characteristic=NULL**, customer_no=客户code, seq_no=1, **component_usage_type=recipe.symbol** | 材质页·材质名称 | ✓(SIMPLE) |
| recipe.code/symbol | join | `material_master.material_recipe_id`→`material_recipe.code/symbol` | 材质页·材质代码/化学符号 | ✓ |

> △ 组合父级 `CFG-COMBO-*` **不写** characteristic=NULL 自指行（父级只有 ASSEMBLY），故父级材质页空（见 §5）。

### 2.4 子配件（仅 COMPOSITE：父→子 ASSEMBLY 关系）

| 请求字段 | 规则 | 落库表.字段（V44） | 落库表.字段（V6 — 渲染源 `$zpj_view`） | 渲染列 | 状态 |
|----------|------|--------------------|----------------------------------------|--------|------|
| 子件料号（解析自 parts[]） | 每子件 1 行 | `mat_bom(bom_type='ASSEMBLY').child_part_no` | `material_bom_item.component_no` | 子配件·子料号 | ✓ |
| （固定） | — | `mat_bom`: seq_no, composition_pct=100, part_version=2000 | `material_bom_item`: material_no=父, **characteristic='ASSEMBLY'**, customer_no, seq_no, **composition_qty=1** | 子配件·序号/数量(=1) | ✓ |
| 子件名称 | join | — | `material_master.material_name`(子件) | 子配件·子料号名称 | ✓ |
| 单位 issue_unit | — | — | **未写** → 子配件·单位 空 | 子配件·单位 | ✗(小) |

### 2.5 工序（用户在工序步选的 processIds）★ 主缺口

| 请求字段 | 规则 | 落库表.字段（V44） | 落库表（per-quote） | 渲染视图 `$gx_view` 读的源 | 状态 |
|----------|------|--------------------|---------------------|----------------------------|------|
| `processIds[]`→process.code | 每工序 1 行，seq 递增 | `mat_process.process_code` | `quotation_line_process.process_id` | **`unit_price.operation_no`** | **✗** |
| `processIds[]`→process.name | 中文名 | `mat_process.assembly_process` | （视图侧 JOIN process_master） | `process_master.process_name` | **✗** |
| （固定/隔离） | — | `mat_process`: customer_id, hf_part_no, seq_no, version=1, is_current=true, part_version=2000, status='ACTIVE', **quotation_line_item_id**(按行隔离) | `quotation_line_process`: line_item_id × process_id | — | — |
| 子件 code / 序号 / 成材率 | — | — | — | `unit_price.code` / `unit_price.seq_no` / `global_variable_value(var_code='PROCESS_DEFAULT_YIELD', key_id=operation_no).value_number` | **✗** |

> **缺口本质**：工序落到了 `mat_process`(V44，已废弃，无视图读) + `quotation_line_process`(per-quote，**当前无 mirror 视图读**)；而工序页 `$gx_view` 只读 V6 `unit_price`（仅靠**导入**填）。→ 选配料号工序页恒空。

### 2.6 组合工艺（仅 COMPOSITE：compositeProcesses）

| 请求字段 | 规则 | 落库表.字段（V44） | 落库表（per-quote — 渲染源） | 渲染列 | 状态 |

### 2.7 报价行（quotation_line_item）

| 字段 | SIMPLE | COMPOSITE | 规则 |
|------|--------|-----------|------|
| `id` | tempId 或随机 | 父=tempId；子=各 PartRequest.quotationLineItemId 或随机 | 前端可预先生成 |

## 3. 各页签字段 → 视图列 → V6 源表（渲染口径，逐字段）

| 页签 | driver | V6 源表 + 过滤 | 字段→视图列 |
|------|--------|----------------|-------------|
| 材质 | `$cz_view` | `material_bom_item` WHERE characteristic **IS NULL** AND customer_no=:code，料号=material_no | 子件=child_hf_part_no · 材质代码=recipe_code · 化学符号=chemical_symbol · 材质名称=child_part_name(COALESCE component_usage_type…) · 规格标签/配方类型=空 |
| 子配件 | `$zpj_view` | `material_bom_item` WHERE characteristic **='ASSEMBLY'** AND customer_no | 序号=seq_no · 子料号=component_no · 子料号名称=material_master.material_name · 数量=composition_qty · 单位=issue_unit |
| 元素 | `$ys_view` | `element_bom_item`（经 material_bom_item ASSEMBLY 关联）AND customer_no | 子件=material_no · 序号=seq_no · 元素=component_no · 含量=content · 单位=issue_unit · 单重=material_master.unit_weight · 组成用量=ASSEMBLY.composition_qty · 单价=INPUT · 小计=FORMULA(含量×单价×单重×组成用量÷100) |
| 工序 | `$gx_view` | **`unit_price`** WHERE finished_material_no=料号 AND system_type='QUOTE' AND price_type='MATERIAL' AND cost_type='自制加工费' | 子件=code · 序号=seq_no · 工序代码=operation_no · 工序=process_master.process_name · 成材率=global_variable_value(PROCESS_DEFAULT_YIELD) · 单价=INPUT(默认120) · 小计=FORMULA(单价÷(成材率÷100)) |
| 组合工艺 | （无 SQL 视图） | `quotation_line_composite_process` WHERE line_item_id=本行 | 序号=seq_no · 工艺代码=def_code · 参与配件=participating_parts · 参数值=param_values · 工艺单价=INPUT |

## 4. 阶段C：报价单卡片"添加 + 渲染"链路（前端）

1. **添加卡片**：选配接口返回 `lineItems` DTO（含 id/料号/composite_type/父子关系/compositeProcesses）→ 前端 `QuotationWizard` 把这些行加入报价单，渲染产品卡片。
2. **持久化**：保存草稿(saveDraft)写 `quotation_line_item.template_id` + componentData 快照。
3. **取模板结构**：详情/编辑页 `enrichComponentData(templateId)` 从 `template.components_snapshot` 取每个组件的 fields/formulas/`data_driver_path`（后端 ComponentDataDTO 不持久化这些，必须从模板补）。
4. **驱动展开**：`useDriverExpansions` → `POST /api/cpq/components/batch-expand`，每个组件按 `data_driver_path`（`$cz_view`/`$ys_view`/`$gx_view`/`$zpj_view`）+ `(customerId, 料号, lineItemId, compositeType)` 跑视图，返回每行 `basicDataValues`。
5. **隐式过滤**：`ImplicitJoinRewriter` 给视图注入 `hf_part_no=料号` 谓词；视图自身 `WHERE customer_no=:customerCode`。
6. **渲染**：`ComponentCell` 按字段类型出值——BASIC_DATA 取 driver 行值、INPUT_NUMBER 取用户输入、FORMULA 用 `computeAllFormulas` 现算。
   - 工序/材质页空 = 第 4 步视图查 V6 基础表 0 行（因为选配没把数据落到该 V6 表）。

## 5. 组合产品（COMPOSITE）父级专项

父级 `CFG-COMBO-*` 落库：`material_master`(COMPOSITE) + `material_bom_item`(只有 ASSEMBLY 子件行) + 组合工艺(per-quote)。
故父级卡片各页签：

| 页签 | 父级是否有数据 | 原因 |
|------|----------------|------|
| 元素 | ✓ | `$ys_view` 经 ASSEMBLY 关联子件 element_bom_item |
| 子配件 | ✓ | `$zpj_view` 读 ASSEMBLY 行 |
| 材质 | ✗ | `$cz_view` 要 characteristic=NULL，父级只有 ASSEMBLY |
| 工序 | ✗ | `$gx_view` 读 unit_price，父级无（且选配整体不写 unit_price） |
| 组合工艺 | ✓ | 读 quotation_line_composite_process |

> 已存在聚合视图：`v_composite_child_materials` / `v_composite_child_processes` / `v_composite_child_elements` / `v_composite_child_weights`（"父级聚合子件"路径设计内）。
