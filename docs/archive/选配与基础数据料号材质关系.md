# 选配产品与基础数据料号 — 材质数据关系说明

> **用途**:厘清"报价单 → 添加产品 → 选配"流程中,料号、材质字典、BOM 三者的关系。说明导入料号与选配新建料号的"身份差异",以及材质数据在两种来源下的取数链路。
>
> **适用读者**:开发、业务管理员、PM、新加入的 Agent。
>
> **关联文档**:
> - `docs/PRD-v3.md` §3.2.2(报价单第二步)、§5.2(产品管理)、§5.4(组件管理)
> - `docs/反模式.md`(AP-25 endpoint 缺失类反模式)
> - `docs/配置方法论.md`(组件/模板/公式三层配置)
> - `docs/数据一致性方法论.md`
>
> **版本**:2026-05-16 创建(对应 V178 + V180 schema 基线)。

## 一、术语与边界

| 术语 | 定义 |
|---|---|
| **导入料号** | 通过「基础数据导入」(V6 staging 三步)从客户 Excel 写入 `mat_part` 的料号。占当前数据 132/136(~97%)。 |
| **选配料号** | 通过「报价单 → 添加产品 → 选配」流程,由 `ConfigureProductService.configure()` 落库的料号。`part_no` 形如 `CFG-<symbol>-<seq>`(由 `PartNoProvider` 生成)。当前 4 条。 |
| **材质字典** | `material_recipe` + `material_recipe_element` 两张表。SYSTEM_ADMIN 在「配置中心 → 材质配方管理」维护。代表可选目录 + 元素配比约束(默认值、可调范围、是否锁定)。 |
| **元素 BOM** | `mat_bom` 表 `bom_type='ELEMENT'` 类型行。任何料号都有(导入或选配落库时都会写一份)。代表每个料号实际的元素配比快照 — 是核价、报价取数的权威来源。 |
| **指纹** | `config_fingerprint`(对 `recipeCode + 元素配比 + 工序集合` 做 SHA256)。用于选配时的"已有相同配置料号复用"判定,**只有选配料号有**。 |

## 二、链路全景

### 2.1 选配抽屉 5 步(`ConfigureProductDrawer.tsx`)

```
Step0 选产品类型 (SIMPLE / COMPOSITE)
   ↓
Step1 搜料号 (Step1SearchPart.tsx)
   │   GET /api/cpq/quotations/configure/search-parts?q=<keyword>&size=50
   │   后端: ConfigureSearchResource.searchParts
   │   SQL : mat_part LEFT JOIN material_recipe ON mr.id = mp.material_recipe_id
   │         匹配 part_no/part_name/specification/size_info/mr.symbol/mr.name
   │
   ├─ 选中已存在料号 → partMode='existing', matLocked=true
   │                  (selectedHfPartNo 锁定 + 元素含量只读)
   │
   └─ 无匹配料号    → partMode='custom',  matLocked=false
                      (走自定义路径,后续从字典选材质 + 可调元素含量)
   ↓
Step2 材质选配 (Step2Material.tsx)
   ├─ matLocked=true  (锁定路径)
   │   GET /api/cpq/quotations/configure/existing-part/{hfPartNo}/material
   │   后端: 详见第六节
   │
   └─ matLocked=false (自定义路径)
       GET /api/cpq/material-recipes               (字典列表)
       GET /api/cpq/material-recipes/{id}          (字典详情 + elements)
   ↓
Step3 选工艺 (mat_process)
   ↓
Step4 (COMPOSITE 才有) 组合工序 (mat_composite_process)
   ↓
Step5 汇总 → POST /api/cpq/quotations/{quotationId}/configure-product
              ConfigureProductService.configure()
                ├─ resolvePart: 算指纹查 mat_part.config_fingerprint
                │     命中 → 复用,不动基础表
                │     未命中 → insertMatPart(写 material_recipe_id + fingerprint)
                ├─ insertElementBom: 元素配比落 mat_bom (bom_type='ELEMENT')
                └─ insertProcesses:  工序落 mat_process (customer_id NOT NULL)
```

## 三、数据库模型

### 3.1 字典系(选配的"目录")

```
material_recipe                              material_recipe_element
  ├─ id PK                          ┌──────►   ├─ id PK
  ├─ code UNIQUE (CuZn36, AgNi10)   │          ├─ recipe_id FK CASCADE ──┐
  ├─ symbol (化学式)                 │          ├─ element_code (Cu/Zn/Ag) │
  ├─ name                           │          ├─ element_name (中文)      │
  ├─ spec_label                     │          ├─ default_pct              │
  ├─ recipe_type ∈                  │          ├─ min_pct / max_pct        │
  │   {locked, editable, partial}   │          ├─ is_locked                │
  ├─ status ∈ {ACTIVE, INACTIVE}    │          └─ sort_order               │
  └─ sort_order                     │                                     │
                                    └─────────────────────────────────────┘
                       UNIQUE (recipe_id, element_code)
                       CHECK: is_locked=true → min/max 必须 NULL
                              is_locked=false → min/max 必填且 min<=max
```

**维护入口**:`/config/material-recipes`(`MaterialRecipeManagement.tsx`)— SYSTEM_ADMIN 配置。

### 3.2 料号 / BOM 系(数据落地)

```
mat_part
  ├─ part_no PK
  ├─ part_name / specification / size_info
  ├─ category_id → product_category.id
  ├─ unit_weight numeric(20,10)
  ├─ material_recipe_id → material_recipe.id  ON DELETE SET NULL
  │     • 导入填:NULL
  │     • 选配填:非 NULL
  ├─ product_type ∈ {SIMPLE, COMPOSITE}
  ├─ config_fingerprint UNIQUE (partial: WHERE NOT NULL)
  │     • 导入填:NULL
  │     • 选配填:非 NULL(SHA256 of recipeCode + elements)
  └─ status_code ∈ {Y, N}

mat_bom
  ├─ hf_part_no FK → mat_part.part_no
  ├─ bom_type ∈ {ELEMENT, INCOMING, RECYCLE, ...}
  ├─ seq_no
  ├─ element_name  ◀ ELEMENT 行存元素代码
  ├─ composition_pct numeric(12,8)  ◀ ELEMENT 行存元素百分比
  ├─ input_material_no / input_material_name
  ├─ gross_qty / net_qty / loss_rate / ...
  └─ part_version (V158)
       UNIQUE (bom_type, hf_part_no, seq_no,
               COALESCE(input_material_no,''),
               COALESCE(element_name,''),
               part_version)

mat_process
  ├─ customer_id NOT NULL   ◀ 与客户绑定
  ├─ hf_part_no
  ├─ seq_no / process_code
  ├─ part_version
  └─ is_current
```

## 四、料号身份对照

| 维度 | **导入料号** | **选配料号** |
|---|---|---|
| 落库流程 | V6 staging(`/import-session/.../commit`)| `ConfigureProductService.configure()` |
| `mat_part.material_recipe_id` | **NULL**(grep 0 命中导入流程任何写入)| 必填(`insertMatPart`:289-296) |
| `mat_part.config_fingerprint` | **NULL** | 必填(`FingerprintCalculator.simpleFingerprint`) |
| `mat_part.product_type` | `SIMPLE`(默认)| `SIMPLE` 或 `COMPOSITE` |
| `part_no` 形态 | 客户料号映射后的 HF 内部号(如 `3100210010`)| `CFG-<recipe.symbol>-<seq>`(如 `CFG-AgCu-000008`)|
| `mat_bom (bom_type='ELEMENT')` | ✓ 写一份(由 V6 staging 落)| ✓ 同步写一份(`insertElementBom`:300-315) |
| `mat_process` | ✓ V6 staging 落(`customer_id` 来自映射)| ✓ `insertProcesses` 落(`customer_id` 来自 quotation)|
| 选配 Step1 可被搜到 | ✓ 能搜到,但 recipe 字段 LEFT JOIN 命中 NULL,前端 `recipeCode/recipeSymbol/recipeName/recipeSpec/recipeType` 全空 | ✓ recipe 全字段完整 |
| 选配 Step2 锁定路径数据源 | **BOM 派**:`mat_bom WHERE bom_type='ELEMENT'`(只读)| **字典派**:`material_recipe_element`(只读)|
| 指纹复用可达性 | ❌ 永远不可被指纹命中复用(列恒 NULL)| ✓ 同样 (recipeCode + elements) 配置会复用此料号 |

## 五、材质数据取数决策树

> **2026-05-28 更新（AP-53 续 5 / V265）**：绑定列从 V44 `mat_part.material_recipe_id` 迁到
> V6 `material_master.material_recipe_id`。下方决策树的"查 mat_part.material_recipe_id"现读
> `material_master.material_recipe_id`；BOM 派数据源从 `mat_bom(bom_type='ELEMENT')` 改为
> `element_bom_item`（characteristic=MAX 口径）。字典本体 material_recipe/_element 不变。

```
[选配 Step2 — 渲染元素配比表]
   ↓
查 material_master.material_recipe_id (按 selectedHfPartNo)   ← V265 后（原 mat_part.material_recipe_id）
   ↓
   ├─ NOT NULL → 字典派 (recipeBound=true)
   │     从 material_recipe + material_recipe_element 取
   │     • recipeCode / Symbol / Name / Spec  ← material_recipe 字段
   │     • elements[]: elementCode / elementName / defaultPct
   │                   / minPct / maxPct / isLocked ← material_recipe_element 字段
   │     • UI: "料号已绑定该材质,元素含量锁定" (warning Alert)
   │
   └─ IS NULL → BOM 派 (recipeBound=false)
         从 mat_bom (bom_type='ELEMENT', part_version=latest) 取
         • recipeCode/Symbol/Name 全空(没绑字典)
         • elements[]: elementCode = element_name 原值
                       elementName = element_name 原值(无中文映射)
                       defaultPct  = composition_pct
                       minPct/maxPct = null (BOM 派不知道字典约束)
                       isLocked    = true (只读)
         • UI: "该料号未绑定材质字典,展示其导入 BOM 的元素配比,只读不可改" (warning Alert)
```

**关键约束**:
- BOM 派与字典派**返回相同 DTO 形态**(`ExistingPartMaterial`),前端 `toDetail()` 函数统一适配,无需关心数据源。
- BOM 派的 `elements[]` **只取最新 `part_version`**(`MAX(part_version) per hf_part_no`),避免旧版本数据混入。

## 六、选配新建料号的"双写"行为

`ConfigureProductService.configure()`(Step5 提交)的 PASS 1 中,对每个 `partMode='custom'` 且指纹**未命中**的配件:

```java
// resolvePart 内部 (ConfigureProductService.java:215-216)
insertMatPart(hfPartNo, "SIMPLE", fingerprint, unitWeight, recipe.id);
//                              ↑                            ↑
//                  config_fingerprint               material_recipe_id

insertElementBom(hfPartNo, elements);   // 同步写 mat_bom (bom_type='ELEMENT')
```

→ **字典是"绑定关系"凭证;`mat_bom` 是"实际配比"快照**。两边都写而不是只写字典,因为:
1. 核价(`CostingSummaryService.compute MATERIAL_COST`)从 `mat_bom`/`mat_bom JOIN element_price` 取数,**不读字典**;
2. 字典 `default_pct` 只是"模板默认值",用户在 Step2 可在 `min/max` 范围内调整,真实值快照在 `mat_bom`;
3. 字典后续 ARCHIVED / 删除,只会断 FK(`material_recipe_id` SET NULL),但料号的 `mat_bom` 不丢失。

**反推**:导入料号没绑字典(material_recipe_id NULL)但有 mat_bom — 完整支撑核价/取数;只是选配 Step2 看不到字典中的"配比可调范围"。

## 七、Bug 修复记录

### Bug #1(2026-05-16 已修复)— 锁定路径 endpoint 缺失

**症状**:用户在选配 Step1 选了已存在料号后,Step2 永远显示"该料号无材质数据"或长时间 loading。

**根因**:前端 `materialRecipeService.ts:94` 调 `GET /api/cpq/quotations/configure/existing-part/{hfPartNo}/material`,但后端 `ConfigureProductResource` / `MaterialRecipeResource` / `ConfigureSearchResource` 都没有此端点。HTTP 实测 404(`/q/health → 404`,其他 `/api/cpq/...` 端点正常 401)。

**修复**:`ConfigureSearchResource` 加 GET endpoint,业务逻辑放在 `MaterialRecipeService.getForExistingPart(hfPartNo)`,按上节决策树实现字典派 / BOM 派分支。

### Bug #2(待修,本次仅记录)— 指纹复用对导入料号失效

**现象**:用户用某 recipeCode + 完全相同的元素配比走"自定义路径"配置,**永远不会复用已存在的导入料号**,而是新建 `CFG-xxx-xxxxx` 料号 → 同一物理产品出现"导入版"和"选配版"双胞胎。

**根因**:`lookupHfByFingerprint`(`ConfigureProductService:108-114`)查 `config_fingerprint` 列,导入料号此列恒 NULL → 永远不命中。

**修复方向**(后续单独 ticket):导入流程在写 `mat_part` 时同步:
- 算指纹(若 Excel 给了 recipe.code 列)
- 反查 `material_recipe.code` 命中则绑定 `material_recipe_id`
- 计算 `config_fingerprint` 写入

### Bug #3(已知数据质量问题)— mat_bom ELEMENT 行 composition_pct 全 NULL

**现象**:数据库实测 `mat_bom WHERE bom_type='ELEMENT'` 所有行 `composition_pct IS NULL`,且 `element_name` 列被脏数据污染(同 seq_no 一行 "Cu"、一行 "25.85" — 把元素百分比串到了 element_name 列)。

**根因**:V6 导入流程解析客户 Excel 时,把"元素含量"这列错写到了 `element_name`,而 `composition_pct` 留空。Excel 模板的列映射有问题。

**影响**:
1. 即使 Bug #1 修了,导入料号在 Step2 BOM 派下显示的元素配比百分比都是空;
2. 核价 `MATERIAL_COST = Σ(material_bom × element_bom.composition_pct × element_price × ...)` 算出 NULL/0;
3. `v_q_element_merged` 视图 `composition_pct` 列也全空。

**修复方向**(后续单独 ticket):
- V6 staging 的 mapping template 校验 element 列与 pct 列分离
- 历史脏数据通过 Excel 重新导入 / SQL 修复

**当前 endpoint 选择**:按规范写,**不**做兜底解析(不把 element_name 里 looks-like-number 的当 pct)。导入数据修干净后 endpoint 自动正确。

## 十一、材质-料号统一管理(2026-05-16 Phase 1-4 上线)

### 11.1 模块定位

**材质** = 料号的归类属性(N:1),不是独立料号实体。`mat_part.material_recipe_id` FK 表达此关系。

「材质管理」页(`/config/material-recipes`)是**统一管理入口**,实现"打开材质看料号清单"。「料号管理」侧(InternalMaterialManagement)本期**不动**(它操作的是历史遗留的 `internal_material` 表,与 `mat_part` 主表分离)。

### 11.4 智能推断算法

```
对每个 material_recipe_id IS NULL 的 mat_part:
  ↓
  查 mat_bom WHERE hf_part_no=X AND bom_type='ELEMENT'
  ↓
  提取所有 element_name (DISTINCT, 跳过 NULL/空)
  ↓
  对每个 element_name e:
    ├─ 跳过 isPureNumber(e)        如 "25.85"
    ├─ 跳过 isPureElementSymbol(e) 如 "Cu"/"Ag"/"Ni"(单字大写开头长度≤2)
    └─ 反查字典:
        ├─ material_recipe.code = e          → EXACT_CODE   (置信度 3)
        ├─ material_recipe.symbol = e        → EXACT_SYMBOL (置信度 2)
        └─ TRAILING_DIGITS 剥离后 = symbol   → PREFIX_MATCH (置信度 1)
                                                  "AgCu3" → 前缀 "AgCu"
                                                  匹配 symbol=AgCu 的 AgCu85/AgCu90
  ↓
  候选去重(同 recipeId 保留最高置信度), 按置信度排序返回
```

**当前数据集预期命中**(基于 132 条未绑料号 + 12 条字典):
- EXACT_CODE: 0(字典 12 条无任何精确出现在 element_name 里)
- EXACT_SYMBOL: 0
- PREFIX_MATCH: 部分料号会命中(如 `AgCu3` → `AgCu85/AgCu90` 两个候选)

→ 实际多数料号会"无候选 + 人工选择",字典扩充后命中率提升。

### 11.5 关键工程约束

1. **boundPartsCount 用一条聚合 SQL**,内存 join 回 DTO,避免 N+1
2. **listParts 按需 setParameter**,无 keyword 时不设 :kw(JPA setParameter 对不存在参数会抛 IAE)
3. **bind-parts 允许"转移"语义**:已绑其他材质的料号可覆盖;前端在子 Drawer 内显示警示标签 "转移自 XXX",二次 confirm
4. **suggestBindings 不修改任何数据**,纯读;confirmBindings 才落库,且必须人工 review 后调用
5. **InternalMaterialManagement 暂不动**:它是历史遗留页;真正的料号主表是 mat_part,本模块全部直接操作 mat_part
