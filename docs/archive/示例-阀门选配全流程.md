# 示例 — 工业球阀 3D 选配全流程

> 端到端演示：从工程师上传 UG NX 设计 → PM 维护特征库 → PM 建选配模板 → 销售选配生成报价单 → 客户自助分享 → 销售审核线索 → 转客户。
>
> 配套 seed 脚本：`docs/sql-seed/valve-demo.sql`（手工执行入库一份完整阀门示例数据）

---

## 一、业务背景

某工业阀门厂希望让销售在客户面前直接选配置出报价（避免来回邮件 + Excel 配置表）。
产品族 **工业球阀**，5 个标准维度 + 1 个数值输入 + 复杂联动约束。

### 1.1 产品族结构

```
工业球阀 (基础料号 VALVE-BALL-BASE)
├── 公称通径 DN          [必选 · 9 个档位]   DN15 / DN20 / DN25 / DN32 / DN40 / DN50 / DN65 / DN80 / DN100
├── 压力等级 PN          [必选 · 4 个档位]   PN16 / PN25 / PN40 / PN64
├── 阀体材质             [必选 · 4 选 1]     铸钢 WCB / 不锈钢 304 / 不锈钢 316 / 黄铜
├── 连接方式             [必选 · 3 选 1]     法兰 FLANGE / 螺纹 THREAD / 焊接 WELD
├── 驱动方式             [可选 · 3 选 1]     手动 HANDLE / 气动 PNEUMATIC / 电动 ELECTRIC
└── 工作介质温度 (℃)     [必选 · 输入]       -20 ~ 200℃（MANUAL 输入）
```

### 1.2 子料号拼接规则

```
基础料号 + 各选项的 partno_prefix + 选定值 + partno_suffix
─────────────────────────────────────────────────────
VBV (基础) + -DN50 + -PN25 + -304 + -FL + -EL = VBV-DN50-PN25-304-FL-EL
                                                     ↑
                                    法兰 (FL) 连接 + 电动 (EL) 驱动
```

### 1.3 基础价 + 价格 delta

**基础配置**（DN25 + PN16 + WCB + FLANGE + HANDLE）= **¥1,800**

| 维度 | 取值 | priceDelta |
|---|---|---|
| **DN** | DN15 | -¥300 |
|  | DN20 | -¥200 |
|  | DN25 | 0（基线）|
|  | DN32 | +¥150 |
|  | DN40 | +¥320 |
|  | DN50 | +¥580 |
|  | DN65 | +¥980 |
|  | DN80 | +¥1,450 |
|  | DN100 | +¥2,200 |
| **PN** | PN16 | 0（基线）|
|  | PN25 | +¥180 |
|  | PN40 | +¥420 |
|  | PN64 | +¥980 |
| **材质** | WCB 铸钢 | 0（基线）|
|  | 304 不锈钢 | +¥250 |
|  | 316 不锈钢 | +¥580 |
|  | 黄铜 | +¥120 |
| **连接** | FLANGE 法兰 | 0（基线）|
|  | THREAD 螺纹 | -¥80 |
|  | WELD 焊接 | +¥150 |
| **驱动** | HANDLE 手动 | 0（基线）|
|  | PNEUMATIC 气动 | +¥2,800 |
|  | ELECTRIC 电动 | +¥3,600 |
| **温度** | （MANUAL 输入，不影响价格）| 0 |

**示例计算**：
```
DN50 + PN25 + 304 + FLANGE + ELECTRIC + 温度 80℃
= 1800 + 580 + 180 + 250 + 0 + 3600 + 0
= ¥6,410
```

### 1.4 约束规则（业务联动 — §3.4.1 算法未文档化，本示例仅展示场景）

| 序号 | 约束类型 | 触发条件 | 影响 | 业务原因 |
|---|---|---|---|---|
| C1 | EXCLUDES | DN ∈ {DN15, DN20} | PN ∈ {PN40, PN64} 禁选 | 小通径无法承受高压 |
| C2 | EXCLUDES | 连接 = WELD | 驱动 = PNEUMATIC 禁选 | 焊接后无法装气动执行器 |
| C3 | EXCLUDES | 材质 = 黄铜 | PN = PN64 禁选 | 黄铜强度不够 |
| C4 | NUMERIC_RANGE | 介质温度 > 150℃ | 材质 ∈ {黄铜, 304} 禁选；316 必选 | 高温须用 316 |
| C5 | IMPLIES | 驱动 = ELECTRIC | 自动加配「24V 电源线」附件（暂未建模） | 电动阀必须供电 |

## 二、端到端流程（7 步）

### 步骤 1️⃣：工程师上传 3D 模型（UG NX 工作流）

**角色**：CAD 工程师
**入口**：`🛒 3D 选配 / 📦 3D 源文件管理`

```
1. 工程师在 UG NX 设计阀门 base 模型（含所有可显隐 mesh：
   - mesh_body_wcb (默认显)
   - mesh_body_304 / mesh_body_316 / mesh_body_brass (默认隐)
   - mesh_flange / mesh_thread / mesh_weld (互斥)
   - mesh_handle / mesh_pneumatic / mesh_electric (互斥)
   - mesh_label_dn_size (可参数化文本)
   )
2. UG NX 内导出 STEP214 → VALVE-BALL-BASE-v1.stp
3. 同时保留 UG 源文件 VALVE-BALL-BASE-v1.prt
4. 打开 CPQ「📦 3D 源文件管理」→ 点「+ 上传新模型」
5. 4 步向导：
   ① 选模式 = "📐 UG NX 工作流（.prt + .stp 双文件）"
   ② 基本信息：part_no = "VALVE-BALL-BASE" / version = 1 / label = "球阀基础模型 v1"
   ③ 上传 .prt + .stp（后端 FreeCAD → Blender 流水线转 GLB，~30-60s）
   ④ 关联到模板（此时无模板可选，跳过；后续 PM 建模板时自选）
6. 完成 → mat_part_model 表入一条记录
```

**对应表写入**：
```sql
INSERT INTO mat_part_model (part_no, version, label, is_current, glb_url, mesh_count, vertices, size_kb, uploaded_at)
VALUES ('VALVE-BALL-BASE', 1, '球阀基础模型 v1', true,
        'cpq-3d-glb://VALVE-BALL-BASE/v1/model.glb', 18, 28450, 2680, NOW());

INSERT INTO mat_part_source_file (part_no, model_id, file_role, file_url, uploaded_at) VALUES
  ('VALVE-BALL-BASE', <model_id>, 'UGNX_SOURCE', 'cpq-ugnx-source://...', NOW()),
  ('VALVE-BALL-BASE', <model_id>, 'STP_NEUTRAL', 'cpq-stp-source://...', NOW()),
  ('VALVE-BALL-BASE', <model_id>, 'GLB_DRACO',   'cpq-3d-glb://...',     NOW());
```

### 步骤 4️⃣：销售选配生成报价单（核心使用场景）

**角色**：销售
**入口**：`🛒 3D 选配 / 🎯 开始选配`

```
1. 销售在「开始选配」看到 PUBLISHED 模板「工业球阀选配」卡片 → 点「🎯 进入选配」
2. 全屏选配页打开（base 模型加载到 Babylon 画布）：
   左侧 3D 画布：旋转/缩放球阀模型
   右侧选项面板：
   - DN: 默认 DN25 → 销售改成 DN50
   - PN: 默认 PN16 → 销售改成 PN25
   - 材质: 默认 WCB → 销售改成 304
   - 连接: 默认 FLANGE → 保留
   - 驱动: 默认 HANDLE → 销售改成 ELECTRIC
   - 温度: 默认 80 → 销售改成 120
   每次选择 → 调 POST /api/cpq/configurator/instances/evaluate-by-template/{templateId}
   后端返 priceBreakdown + totalPrice，前端实时更新底部价格栏
3. 3D 实时联动（TODO 切片：Babylon 集成）：
   - 选 304 → mesh_body_wcb 隐藏 + mesh_body_304 显示
   - 选 ELECTRIC → mesh_handle 隐藏 + mesh_electric 显示
   - DN 改变 → mesh_label_dn_size 文本参数化更新
4. 底部价格栏显示：
   基础价 ¥1,800 + DN50 +¥580 + PN25 +¥180 + 304 +¥250 + FLANGE ¥0 + ELECTRIC +¥3,600 + 温度 120 ¥0
   = ¥6,410
5. 点「🎯 提交配置」→ 两步式 Modal 打开：
   步骤 1 显示「✓ 配置已保存为选配实例 CI-202605-001」+ 三选一按钮
6. 销售选「🎯 直接生成新报价单」:
   后端 POST /api/cpq/configurator/instances/{id}/link-action
   Body: { action: "NEW_QUOTATION" }
   后端:
   - 生成 mat_part 子料号 VBV-DN50-PN25-304-FL-EL（按 partno_prefix/suffix/include 拼接）
   - 生成 quotation (TODO 集成 quotation 模块)
   - 生成 line_item (subtotal = ¥6,410)
   - UPDATE instance SET status='LINKED', linked_quotation_id=新ID,
                          generated_part_no='VBV-DN50-PN25-304-FL-EL',
                          generated_quotation_id=新ID,
                          linked_at=NOW()
7. 跳转报价单编辑页（标准报价单流程，不动）→ 销售完善商务条款 → 输出 PDF 给客户
```

**对应表写入（核心实例链路）**：
```sql
INSERT INTO product_config_instance (
    instance_code, template_id, template_version, name, customer_id, user_id,
    selected_values, config_fingerprint, computed_total_price, base_price, status, expires_at
) VALUES (
    'CI-202605-001', <template_id>, 1, '罗克韦尔Q3球阀订单', <customer_id>, <user_id>,
    '{"DN":"50","PN":"25","MATERIAL":"304","CONNECTION":"FLANGE","DRIVE":"ELECTRIC","TEMP_RANGE":"120"}',
    sha256(canonical), 6410.00, 1800.00, 'SUBMITTED', NOW() + INTERVAL '30 days'
);

-- 提交对话框选 NEW_QUOTATION 后
UPDATE product_config_instance SET
    status = 'LINKED',
    linked_quotation_id = <new_quotation_id>,
    linked_at = NOW(),
    generated_part_no = 'VBV-DN50-PN25-304-FL-EL',
    generated_quotation_id = <new_quotation_id>,
    generated_line_item_id = <new_line_item_id>
WHERE instance_code = 'CI-202605-001';
```

| 场景 | DN | PN | 材质 | 连接 | 驱动 | 温度 | 总价 | 子料号 |
