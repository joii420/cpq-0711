# 数据库表字段说明 — 报价渲染链路

> 范围:报价单 → 明细 → 组件/模板渲染 → V6 物料数据源 → 字典,共 17 张核心表。
> 不含 staging(`*_staging`)、审计日志、核价(`costing_*`)、3D 配置、安全/工具表。
> 生成日期:2026-05-29。字段说明结合 DB 注释 + 业务理解;ERP 透传字段(导入保留、渲染未用)按组略述。

## 0. 一图看懂渲染数据流

```
customer ──┐
           │ customer_id / customer_no(code)
template ──┤  (报价模板 customer_template_id;含 components_snapshot 冻结组件配置)
           │
quotation ─┴─< quotation_line_item >─┬─< quotation_line_process >── process ── process_master
   (单)         (产品行, 1行=1产品)    │      (本行工序, per-quote)   (工序字典)  (工序名主数据)
                  │ template_id        │
                  │ product_part_no    └─< quotation_line_component_data >
                  │ composite_type             (每行×每组件×页签: row_data 编辑层 + snapshot_rows 冻结层)
                  │                                    │ component_id
                  ▼                                    ▼
            渲染时:component_sql_view(mirror 视图)按 BNF 路径展开行,
            从 V6 物料表(material_bom_item / element_bom_item)按 customer_no + 料号取数据:
              material_master ──(material_recipe_id)── material_recipe ──< material_recipe_element
```

**两套数据层(AP-53)**:V6(`material_master / material_bom_item / element_bom_item / process_master`)为现行;V44(`mat_part / mat_bom / mat_process`)已废弃,本文不含。
**关键设计**:V6 物料表之间用**文本业务键**(`material_no` / `customer_no` / `hf_part_no`)关联,**不是 uuid 外键**;报价/组件/模板侧才是 uuid 外键。

### E.3 Mirror 视图 = 渲染 join 的真正逻辑(存于 component_sql_view.sql_template)

**composite_child_materials_mirror**(选配-材质):
```sql
FROM material_bom_item asy
LEFT JOIN material_master mm ON mm.material_no = asy.component_no
WHERE asy.system_type='QUOTE' AND asy.characteristic IS NULL AND asy.customer_no = :customerCode
  AND asy.material_no = <本行料号>
-- material_name = COALESCE(component_usage_type, mm.material_type, mm.material_name)
```

**composite_child_elements_mirror**(选配-元素含量):
```sql
FROM element_bom_item ebi
LEFT JOIN material_master mm ON mm.material_no = ebi.material_no
WHERE ebi.system_type='QUOTE' AND ebi.hf_part_no IS NOT NULL AND ebi.customer_no = :customerCode
  AND ebi.characteristic = (取该 customer 下 MAX(characteristic) 那组)
-- element_name = component_no, composition_pct = content
```

**composite_child_processes_mirror**(选配-工序列表,per-quote):
```sql
FROM quotation_line_process qlp
JOIN quotation_line_item qli ON qli.id = qlp.line_item_id
JOIN quotation q            ON q.id   = qli.quotation_id
JOIN process pr             ON pr.id  = qlp.process_id
LEFT JOIN process_master pm ON pm.process_no = pr.code
WHERE qlp.line_item_id = :lineItemId
-- assembly_process = COALESCE(pm.process_name, pr.name, pr.code)
```

> `:customerCode` / `:lineItemId` 等占位符由 RuntimeContext 在展开时注入(driver 行上下文)。
> 材质/元素 mirror 按 **客户 + 料号** 取数(跨客户复用须逐客户补数据);工序 mirror 按 **报价行** 取数(per-quote)。

## F. 渲染数据流(端到端)

1. **加产品**:configure/import 写 `quotation_line_item`(+ `quotation_line_process` 工序 + V6 物料数据),并把每组件整行解析值冻进 `quotation_line_component_data.snapshot_rows`。
2. **保存**:`saveDraft` 全量重建行(换新 UUID,行 template_id 兜底为单头模板),重算快照。
3. **渲染**:前端按 `template.components_snapshot` enrich 出每页签组件配置(含 `data_driver_path`);对有 driver 的组件调 batch-expand → 后端 `expandWithSnapshot` 命中 `snapshot_rows` 直返,否则走 mirror 视图实时展开(上面 E.3)→ 字段按 `basic_data_path`(`$view.column`)取值渲染。
