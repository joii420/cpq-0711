# 报价系统 Excel 导入落库 —— 代码 vs 方案 全量核查报告

> 核查日期：2026-05-30
> 基准文档：`docs/table/报价系统Excel导入落库方案.md`（V3.0，唯一正确基准）
> 核查链路：报价单管理 → 从基础数据导入 → 报价单 Excel 导入（QUOTE 链路）
> 核查方式：逐 Sheet 逐字段 + 固定写入值 + 特殊规则 + 目标表，全维度对照

---

## 〇、链路与入口确认

| 环节 | 实现 |
|------|------|
| 前端入口 | `QuotationList.tsx` →「从基础数据导入」按钮 → `QuoteBasicDataImportV6Drawer.tsx` |
| 前端服务 | `basicDataImportV6Service.importQuote(customerId, file)` |
| 后端入口 | `QuoteImportService.importExcel(...)`（`com.cpq.basicdata.v6.quote`） |
| 调度 | 19 个 `SheetHandler`（Q01~Q19），每个 Sheet 独立 `REQUIRES_NEW` 事务 |
| 列匹配 | `SheetRow.getStr()` 按"表头**包含**关键字"匹配（表头先去全部空白） |
| `unit_price` 写入 | 统一经 `UnitPriceWriter.upsert()`（native `INSERT ... ON CONFLICT DO UPDATE`） |

入口正确，确为 QUOTE 链路。`system_type=QUOTE`、`customer_no` 由系统从 `customerId` 反查注入，均在 `QuoteImportService` 集中设置。

**结论总览**：19 个 Sheet 中 **8 个完全符合**，**11 个存在偏差**（其中 🔴 严重 1 项、🟡 偏差/受 schema 限制 若干）。无致命阻断，但存在 1 处语义错（§3 料号表同步对象搞反）与一批"文档字段在 V6 表中不存在 → 代码折叠/丢弃"的结构性偏差。

|:-:|-------------|---------|:----:|

| 9 | 元素回收折扣为 UPDATE，按最新 characteristic | Q05 `UPDATE ... characteristic=(SELECT ... ORDER BY characteristic DESC LIMIT 1)` | ✅（匹配键见 §5） |

| 12 | 布尔「是→1，否→0」TINYINT | `SheetRow.getBool()` 返 `Boolean`，列类型为 **BOOLEAN**（非 1/0 数字） | 🟡 语义等价，表示形式不同 |
| 13 | 过滤空行 | `ExcelParserService` `allBlank` 跳过 | ✅ |
| 14 | 多表写入顺序（料号→关系→BOM主→子→单价→年降） | `orderedHandlers()` = q18,q02,q03,q04,q05,q12,q01,q06... | ✅ |

## 四、差异汇总（按严重度）

### 🔴 严重（语义/数据正确性）

**S1 · §3 物料BOM 料号表同步对象搞反**
- 方案 §3 字段表：同步「投入料号」(component) → `material_master.material_no`，并写其 `material_type`（**只写数字**）与 `material_name`（投入料号名称）。
- 代码：同步的是「宏丰料号」(主件)，`material_type` = 产出料号类型**原文**（未取数字），`material_name` = `null`，且真正的「投入料号」未落 master。
- 影响：`material_master` 料号集与 `material_type` 取值都与方案不符；下游按 material_type 数字判断（银点/非银点）会拿到「1.银点类」整串。
- 备注：文档 §3 字段表与 §3 末注**自相矛盾**，需你裁定基准。

### 🟡 偏差 —— 受 V6 表结构限制（文档字段在目标表中不存在）

**D1 · 固定金额/比例无法用 `pricing_price` 空值区分**
- 方案通用规则：固定费用写 `pricing_price`、比例费用写 `cost_ratio`，**以 pricing_price 是否为空判断类型**。
- 现实：`unit_price.pricing_price` 为 **NOT NULL**，代码把 null 一律回填 `0`（Q07/08/10/11/13/15 及 `UnitPriceWriter` 兜底）。
- 结果：比例型费用的 pricing_price=0（非空），**只能靠 `cost_ratio` 是否为空区分**。涉及 6 个 Sheet。

**D2 · 文档字段在目标表无对应列 → 折叠/丢弃**
| 位置 | 文档字段 | 目标列现状 | 代码处理 |
