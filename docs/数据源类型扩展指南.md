# 数据源类型扩展指南

> 2026-05-18 V190 起 DATA_SOURCE 字段支持多种数据源类型。本文说明如何加新类型。

## 一、现有类型清单

```typescript
datasource_binding: {
  type: 'DATABASE_QUERY' | 'GLOBAL_VARIABLE' | 'HTTP_API' | 'BNF_PATH',
  config: { ... 按 type 各自的配置 ... }
}
```

| type | 现状 | 用途 |
|---|---|---|
| `DATABASE_QUERY` | ✅ 已实现 | SQL/API 查询数据源（基于 DataSourceService） |
| `GLOBAL_VARIABLE` | ✅ 已实现 | 引用全局变量值（按 driver row 动态 key） |
| `BNF_PATH` | ✅ 已实现 | BNF 路径（DataLoader 调用） |
| `HTTP_API` | ✅ 已实现 (默认关闭) | 外部 API 调用 — 严格安全约束, 见 [HTTP_API_安全配置.md](./HTTP_API_安全配置.md) |

## 二、加新类型的标准动作

> Phase I2 (2026-05-18) 已落地 `DataSourceResolver` 抽象层 + CDI Instance 自动注册. 加新类型零 wiring 代码.

### Step 1：定义 `config` 结构

在 `cpq-frontend/src/pages/component/types.ts` 的 `DefaultSource` interface 加新 type：

```typescript
export interface DefaultSource {
  type: 'GLOBAL_VARIABLE' | 'BNF_PATH' | 'HTTP_API' | 'YOUR_NEW_TYPE';
  // ... 老字段保留
  your_config?: {
    field1: string;
    field2: number;
  };
}
```

### Step 2：后端 Resolver

新建 `cpq-backend/.../datasource/resolver/YourTypeResolver.java`：

```java
@ApplicationScoped
public class YourTypeResolver implements DataSourceResolver {
    @Override
    public String type() { return "YOUR_NEW_TYPE"; }

    @Override
    public Object resolve(Map<String, Object> config, Map<String, Object> driverRow) {
        // 你的解析逻辑
        return value;
    }
}
```

**零额外注册** — `DataSourceResolverRegistry` 启动时通过 `Instance<DataSourceResolver> resolvers` 自动收集所有 `@ApplicationScoped` 实现.

### Step 3：前端配置 UI

在「组件管理 → 字段编辑 → DATA_SOURCE 字段」加 type 选择 + 各类型的配置 form（Phase D follow-up）。

### Step 4：渲染层

在 `QuotationStep2.computeAllFormulas` + placeholder 渲染处加新类型分发分支。

### Step 5：文档

把新 type 加进本文件 + `docs/配置中心架构.md` 的字段类型矩阵。

## 三、HTTP_API 类型的设计草案（Phase D follow-up）

```jsonc
{
  "type": "HTTP_API",
  "api_config": {
    "method": "GET",
    "url_template": "https://api.example.com/price/{partNo}",
    "headers": {
      "Authorization": "Bearer ${env:API_TOKEN}"
    },
    "body_template": null,
    "response_path": "data.price",
    "cache_ttl_seconds": 300,
    "timeout_ms": 5000,
    "retry": { "max": 2, "backoff_ms": 1000 }
  }
}
```

### 待解决的设计问题

- 鉴权 secret 怎么存（环境变量 / 安全配置中心）
- 失败兜底（HTTP 失败 → 用 field.content / 0 / 报错？）
- 缓存策略（按 url 全 query 哈希 / 按 partNo / 全局）
- 安全免外名单（防 SSRF — 必须白名单 host）

## 四、禁止事项

❌ 不要把数据来源类型硬编码进 `field_type` — 增加类型用 `datasource_binding.type`  
❌ 不要把 type 列表写死在 Java 枚举 — 用 Resolver registry  
❌ 不要让单个 type 处理多种语义 — 一 type 一 resolver
