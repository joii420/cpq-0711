## 4. Plan 3b/3c 关键改造点速查（细节以集成设计为准）

**3b 改 `ConfigureProductService`**：
- 发号仅 2 处：`resolvePart:247-248`(SIMPLE) + `configure:796-797`(COMPOSITE) → `quoteAllocator.mintAndRegister(customerCode, yyMm)`。
- 复用判定：`resolvePart:240` 的 `lookupHfByFingerprint`（生产侧全局）→ 换 `SalesSignatureRepository.lookup(customerCode,"v1",销售指纹)`；命中 **在落库(:262)之前 return**（R3 防重复落库/累加，守 AP-51）。
- `:262` `insertMaterialMasterV6(..., fp)` → `fp=null`（R1）。
- 并发败者回读（`sel_part_signature` ON CONFLICT DO NOTHING 返 0 → 回读先赢者号、弃己号，孤儿号可接受）。
- `isCfg`（`MaterialBomMergeHandler:262`）**保持原样**（报价料号不再命中，属期望；别把报价料号加进拒绝集）。
- `yyMm` = `YearMonth.now()`（与现役 `MaterialBomMergeHandler:59` 同口径）。
- customerCode 空 → 选配强制报错（报价料号号内嵌客户码）。

**3c 改落库**（备选 A，switch 分发，非 SPI）：
- 按 `enabledParams` 的 paramTypeCode 决定调哪些落库方法（模板没启用工序就不落工序）。
- `insertElementBomV6`/`insertMaterialBomItemV6`/`insertProcessSimpleUnitPriceV6` + COMPOSITE 三方法**补 `sales_part_no = quotePartNo`**（显式填，别留 NULL）。
- material_master 建行 fp=NULL 防断链。
- 主键归属：选配语境 `material_no = sales_part_no = quotePartNo`（1:1 自洽），**不复用**既有生产料号行。

**依赖链**：3a(已合,产 EnabledParam 投影) → **3b** → **3c**（3c 依赖 3b 的 quotePartNo + resolvePart 骨架，须 3b T4 之后；可同分支连续做，也可 3b 先合再起 3c）。
