# 报价单 与 核价单 功能总结

> **目的**：从业务整合视角统一描述报价单（Quotation）与核价单（Costing Summary）两条主线的功能、流程、数据流、相互关系。本文是入门 + 速览级文档，详细需求请参考 `docs/PRD-v3.md`（核价系统部分尚未回写到 PRD，详细以 RECORD.md + 本文为准），操作步骤请参考 `docs/archive/操作说明.md`，Excel 视图配置请参考 `docs/Excel模板配置指南.md`。
>
> **适用读者**：业务方、PM、新加入的开发 / 测试 / Agent。
>
> **版本**：2026-06-18（与 V81 数据库 schema 对齐；§2.4 已补 Phase2 整行级快照脚注）。

---

## 一、定位与边界

### 报价单（Quotation）— 销售对外的"价格清单"

- **业务目的**：向客户出价；记录客户、产品、数量、单价、折扣、税费、币种、有效期，最终形成 PDF / Excel 输出
- **角色主线**：销售代表（SALES_REP） → 销售经理审批（SALES_MANAGER）
- **结果对外**：审批通过后给客户的报价 PDF / Excel
- **强调**：商务条款（折扣、付款条件、有效期、币种）

### 核价单（Costing Summary）— 内部的"成本计算"

- **业务目的**：算出某个料号的真实成本（材料 + 加工 + 模具 + 设计 + 商务加价）— 给报价提供"底价"
- **角色主线**：定价经理（PRICING_MANAGER）独立操作或在报价单内作为子流程
- **结果对内**：成本数据 + 7 项 metric（材料成本/加工费/模具费/设计成本/单位总成本/单位总成本(报价币种)/单件成本）
- **强调**：精度、版本、可追溯（"这个 0.123 是怎么算出来的"）

### 二者关系

```
核价单                          报价单
(成本视角)                      (商务视角)
   │                              │
   │   计算/锁定                  │   关联引用
   ▼                              ▼
costing_summary  ◄───── costing_summary_id ───── quotation_line_item
   ↑                              │
   │                              │
   └─ 引用全局基础数据             └─ 加上折扣/加价/税 → total_amount
      (元素/材料/汇率版本)
```

- **可独立**：核价单可以脱离报价单单独管理（在「核价单」菜单里直接建 → 计算 → 发布）
- **可关联**：`quotation_line_item.costing_summary_id` 软关联（ON DELETE SET NULL），删核价单不影响报价单
- **可嵌入**：报价单页面内嵌核价单 Tab，销售在出价时可直接看到/编辑核价

## 二、报价单功能

### 2.1 5 步向导（QuotationWizard）

| 步 | 名 | 主要动作 | 关键产出 |
|---|---|---|---|
| 1 | 选择客户 | 选客户、联系人、报价模板 / 核价模板 / 产品分类 | `quotation.customer_id / customer_template_id / costing_card_template_id` |
| 2 | **添加产品** | 添加产品行；填属性；自动加载组件 / 公式；可视化报价、核价、Excel 视图 | `quotation_line_item[]` |
| 3 | 优惠策略 | 应用客户级折扣 / 价格调整 / 阶梯折扣 | `system_discount_rate / pricing_strategy_id` |
| 4 | 交易条款 | 付款条件 / 交期 / 有效期 / 币种 / 备注 | `payment_terms / expiry_date / base_currency` |
| 5 | 提交审批 | 冻结快照 → 状态 DRAFT → SUBMITTED | snapshot 数据落 `*_snapshot` 列 |

### 2.2 第二步「添加产品」的三个视图

第二步页面（`QuotationStep2.tsx`）顶部有 mainTab + viewType 双轴切换：

|  | 产品卡片视图 | Excel 视图 | 比对视图 |
|---|---|---|---|
| **报价单 Tab** | 按报价模板的组件渲染卡片，每行表单填值 | 按 `customer_template_id` 反查 `costing_template` 渲染表格，每 lineItem 一行 | 同模板/同产品的多份报价历史并排比对 |
| **核价单 Tab** | 按核价模板的组件渲染卡片（V72） | 按 `costing_card_template_id` 反查 `costing_template` 渲染（V73/V74） | — |

> **关键**：报价单与核价单**共用同一份 `lineItems` 内存对象**，但渲染时按各自模板的 componentId 集合做白名单过滤。编辑回写走"updater 跑在视图态 → union-merge by componentId 回底层完整数据"sandwich 模式（避免 AP-19 双视图共享时的索引错位）。

### 2.3 报价单状态机

```
DRAFT  ──提交──►  SUBMITTED  ──销售经理审批──►  APPROVED
  ▲                  │                            │
  │                  │ 驳回                       │
  └──────────────────┘                            │
                                                  ▼
                                              EXPIRED / CLOSED
```

### 2.4 报价单的"快照机制"

提交时把这些字段冻结进 `quotation_line_item.*_snapshot`：
- `product_part_no_snapshot` / `product_name_snapshot`
- `unit_price_snapshot` / `discount_rate_snapshot`
- `formula_used_snapshot` / `data_source_snapshot`

**目的**：基础数据（料号、价格、汇率）后续可能变；快照保证"半年后看回这份报价依然能复现当时的算法"。

> **Phase2 脚注（2026-06-18 补，快照机制已扩展，本节上文写于 2026-05-05）**：当前快照分两代——
> - **v1 快照（`*_snapshot` 列）**：冻结基本字段（料号、品名、单价、折扣、公式、数据源），即本节上文所述原有内容。
> - **v2 快照（`quotation_line_item.quote_card_values` / `costing_card_values`，2026-06-01 起）**：冻结**完整行级数据**（整张产品卡片各 Tab 的逐行渲染结果），并在**渲染期脱钩**——优先读快照、不再实时调 `/batch-expand` 重查。
> - **冻结时机**：提交时冻结；自 **2026-06-18 起草稿（DRAFT）默认冻结**，打开报价单只读已冻快照、不自动重查。
>
> 最新细节以《三大核心模块基线》§4.2 Phase 2 段落为准。

## 三、核价单功能

### 3.1 三层数据架构

```
┌─────────────────────────────────────────────────────────────┐
│  全局基础数据 (3 类，每类独立版本)                          │
│  costing_price_version (kind=ELEMENT/MATERIAL/EXCHANGE)     │
│   ├─ costing_element_price                                  │
│   ├─ costing_material_price                                 │
│   └─ costing_exchange_rate                                  │
└─────────────────────────────────────────────────────────────┘
                              ▲
                              │ versionId 引用
                              │
┌─────────────────────────────────────────────────────────────┐
│  料号级数据 (8 张表，按料号绑定)                            │
│  costing_part_*                                             │
│   ├─ process_cost (8 种 cost_type discriminator)            │
│   ├─ tooling_cost                                           │
│   ├─ material_bom                                           │
│   ├─ element_bom                                            │
│   ├─ quality_check (INCOMING / SEMI_FINISHED stage)         │
│   ├─ plating                                                │
│   ├─ design_cost                                            │
│   └─ weight (1 row per part, unique)                        │
└─────────────────────────────────────────────────────────────┘
                              ▲
                              │ hf_part_no 关联
                              │
┌─────────────────────────────────────────────────────────────┐
│  核价单实例 (3 张表，1 个料号 1 份)                         │
│  costing_summary                                            │
│   ├─ costing_summary_override  (用户差量 - what-if)         │
│   └─ costing_summary_result    (7 metric 计算结果)          │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 核价单状态机

```
DRAFT  ──compute()──►  COMPUTED  ──publish()──►  PUBLISHED  ──archive──►  ARCHIVED
   ▲                       │                         │
   │ 用户改 override        │                         │
   └───────────────────────┘                         │
                                                     │
                                          (Excel 视图按 PUBLISHED 取数)
```

### 3.3 7 项 metric 计算（`CostingSummaryService.compute()`）

| metric_code | 中文 | 计算依赖 |
|---|---|---|
| MATERIAL_COST | 材料成本 | `Σ(material_bom × element_bom.composition_pct × element_price × (1+loss_rate))` |
| PROCESS_FEE | 加工费(工序合计) | `Σ unit_price` (8 种 cost_type 全部) |
| TOOLING_FEE | 模具工装费 | `Σ tooling_unit_cost / process_count / cycle_count` |
| DESIGN_COST | 设计成本 | `Σ (design_proc_fee + design_material_fee)` |
| UNIT_TOTAL_COST | 单位总成本 | `MATERIAL_COST + PROCESS_FEE + TOOLING_FEE + DESIGN_COST` |
| UNIT_TOTAL_QUOTE | 单位总成本(报价币种) | `UNIT_TOTAL_COST × 汇率(CNY→quoteCurrency)` |
| UNIT_PER_PCS | 单件成本 | `UNIT_TOTAL_QUOTE × weight_g_per_pcs / 1000` |

> 当前**未实现**的商务加价（管理费/财务费/利润/税费/电镀/其他外加工）— 在 `v_costing_summary_full` 视图里以 NULL 占位，等 compute() 升级后自动有值。

### 3.4 用户差量（override）—— Q3=B 决策

- 用户在核价单详情可填"差量"（`costing_summary_override.target_kind / target_key / field_name → override_value`）
- compute 入口先 load 差量进 Map（key=`{kind}:{target_key}:{field_name}`），求值时直接命中即用
- **不写回基础数据** —— 让 PRICING_MANAGER 试算 what-if，不污染团队共享数据
- override 修改后状态自动从 COMPUTED → DRAFT，强制重新 compute

## 四、模板体系（三层）

```
template (kind=QUOTATION/COSTING)         ◄── 报价/核价模板：组件 + 公式 + 产品属性
   │
   │ linked_template_id  反查
   ▼
costing_template (Excel 模板)             ◄── 列结构 (columns JSON) — Excel 视图按此渲染
```

| 模板类型 | 表 | 用途 | 绑定到报价单的列 |
|---|---|---|---|
| **报价模板** | `template` (template_kind='QUOTATION') | 报价单产品卡片视图 + Excel 视图源 | `quotation.customer_template_id` |
| **核价模板** | `template` (template_kind='COSTING') | 核价单产品卡片视图 + Excel 视图源 | `quotation.costing_card_template_id` |
| **Excel 模板** | `costing_template` | Excel 视图渲染时的列结构 | 按 `linked_template_id` 反查 |

### 4.1 模板派生新草稿（V72）

`PUBLISHED` / `ARCHIVED` 模板不可改 → 列表页「派生新草稿」复制一份，version 自增（v1.0 → v1.1）继续编辑。

### 4.2 同 series 多版本

`template_series_id` / `costing_template.series_id` 把"同一个业务用途的多个版本"串成一条 series；UI 默认只显示 series 内最新一份。

## 五、Excel 视图（核心交互之一）

> ⚠️ **2026-06-21 起报价侧 Excel 视图「列值计算」已反转为前端权威**（`buildExcelSnapshot` 复用卡片引擎，Excel 恒等卡片）：本节描述的「列结构来源 / 数据源类型 / 隐式 JOIN」仍准确，但**列值由前端计算、saveDraft 落 `quote_excel_values` 快照、导出/提交读快照**，后端不再主动重算报价 Excel 值。权威口径详见 `docs/三大核心模块基线.md §4.7` + 反模式 AP-58/AP-59。注：**列定义仍由后端 `getEffectiveColumns` 解析**（v2 `excel_component_id` 引用），只「值」走前端。核价侧暂仍后端计算。

### 5.1 渲染链

```
mainTab + viewType
   │
   ├─ quote+excel    →  customer_template_id      ┐
   └─ costing+excel  →  costing_card_template_id  │
                                                  ▼
                              costing_template.list({linkedTemplateId, status:'PUBLISHED'})
                                                  │
                                  优先 is_default=true，否则取最新一份
                                                  │
                                                  ▼
                                       按 columns JSON 渲染
                                       每个 lineItem 一行
```

### 5.2 列字段（详见 `Excel模板配置指南.md`）

每列：`{col_key, title, source_type, variable_path | formula, comparison_tag}`

- **source_type=VARIABLE**：取数列；`variable_path` 可填三种：① BNF 路径（`v_costing_summary_full.material_cost`）② lineItem 简写（`{hf_part_no}`）③ 显式谓词路径
- **source_type=FORMULA**：公式列；`formula` 用 `[X]` 引用同行其他列、`{code}` 引用变量

### 5.3 隐式 JOIN（自动按料号过滤）

后端 `ImplicitJoinRewriter` 渲染时自动注入：
- `hf_part_no = '<lineItem.productPartNo>'`（目标表/视图含 `hf_part_no` 列时）
- `customer_id = '<报价单.customerId>'`（目标表/视图含 `customer_id` 列时）
- `part_no` 同义自动注入

### 5.4 PIVOT 视图（V80 引入）

`v_costing_summary_full` — 把 7 个 metric × N 行 PIVOT 成 1 行 9 列（`material_cost / processing_cost / management_cost / ...`），让 Excel 模板能用单条 BNF 路径取每个成本项 — 而不是查 7 次 `costing_summary_result[metric_code='X']`。

## 八、已知限制（v1）

| # | 限制 | 影响 | 计划版本 |
|---|---|---|---|
| 1 | compute() 暂未实现商务加价 6 项（管理/财务/利润/税费/电镀/其他外加工）| `v_costing_summary_full` 中这 6 列恒 NULL，Excel 视图显示 "—" | v2 |
| 2 | LinkedExcelView 每个 lineItem 一行 | 同料号多 summary 不能在同一视图展开成多行 | v2 |
| 3 | FORMULA 不支持字符串字面量 / Excel 函数 | 显示固定文本 / SUM/IF/VLOOKUP 等 → 必须改用 VARIABLE 取视图硬编码字段 | v2（评估）|
| 4 | 同一 `linked_template_id` 内只能 1 份 PUBLISHED 默认 Excel 模板 | 多默认场景需要切换/归档 | 永久（设计取舍）|
| 5 | 核价单批量比较跨 summary 未实现 | 同料号在不同基础数据版本的成本差异要手动看 | v2 |
| 6 | Excel 视图无导出 / 复制为 Excel 文件功能 | 视图仅 web 渲染，不能"另存 .xlsx" | v2 |
| 7 | 公式可配置化（compute 7 项 metric 当前是 service 硬编码） | 业务调整需改代码 + 重启 | v2 |

## 九、术语速查

| 术语 | 含义 |
|---|---|
| **lineItem** | 报价单 / 核价单中的一条产品行（前端是 React state，后端是 `quotation_line_item` 一行）|
| **componentData** | lineItem 上挂载的"组件实例数据"（按 componentId 分组的 row + cell 值）|
| **driver expansion** | 组件按 `data_driver_path` 自动展开成 N 行（如：电镀方案有几行就展开几行）|
| **BNF 路径** | 后端能解析的"数据路径"语法：`<sheet>[谓词].<field>`，如 `mat_part[hf_part_no='X'].unit_weight` |
| **隐式 JOIN** | 后端 `ImplicitJoinRewriter` 自动按 lineItem.productPartNo / customerId 注入谓词 |
| **快照** | 报价提交后冻结到 `*_snapshot` 列的字段，确保历史可复现 |
| **override / 差量** | 核价单层面对全局/料号数据的用户级覆盖（不写回，仅本核价单生效） |
| **summary** | costing_summary 一行 = 某料号的一份核价计算实例 |
| **metric** | costing_summary_result 一行 = 一项成本指标的计算结果 |
| **PIVOT 视图** | `v_costing_summary_full` 把 N 行 metric 转成 1 行 9 列，方便 Excel 视图取数 |
| **template_kind** | template 表的 discriminator (QUOTATION/COSTING)；basic_data_config 上同名字段（V79）多了 BOTH |
