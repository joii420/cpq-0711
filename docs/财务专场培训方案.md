# CPQ 系统财务专场培训方案

> **版本**：v0.1 初稿 · 2026-07-16 · 待评审
> **听众**：财务 / 核价人员（PRICING_MANAGER）
> **时长**：约 2 小时 10 分（含休息）
> **形式**：讲解 + 现场演示 + 实操练习
> **范围**：不含 3D 选配；定价策略仅作预告（🚧 开发中）

面向财务（核价）人员的系统介绍与上手培训 —— 以核价工作台为主线，含导入、主数据与排查方法。

---

## 一、培训目标

培训结束后，参训人员应达到三个「会」：

1. **会用** —— 独立完成核价单的查看 → 切换数据版本 → 提交审批的完整闭环；
2. **会导** —— 掌握核价数据 Excel 导入的操作与版本升级规则，导入后能自行验证生效；
3. **会查** —— 数据不对时能按排查口诀自查：**先看版本 → 再看主数据 → 最后找管理员查模板**。

---

## 二、权限与范围说明（讲师必读）

- 财务账号**已关闭「报价单管理」入口**（菜单隐藏 + 路由守卫）。财务以**核价管理为主阵地**，从核价单联查报价详情。培训中不演示报价单列表。
- 财务实际可见菜单：工作台、客户列表、产品管理、**核价管理**、定价策略、模板管理、**主数据维护**、通知。培训内容框定在此范围内。
- 3D 选配相关内容本场全部移除。
- 「定价策略」模块**正在开发中**，本场不演示、不点击，仅以一页路线图预告（见第五节）。

## 四、分节内容明细

### ① 系统全景与财务的位置（15 分钟）

- 【讲】CPQ 整体链路一页图：**销售建报价单 → 提交 → 自动生成核价单 → 财务核价/审批 → 冻结归档**。
- 【讲】报价单 vs 核价单的边界（开场必讲透）：报价单 = 销售对外的价格清单，提交后冻结、财务不碰；核价单 = 内部成本计算，财务的工作对象、可重算。
- 【讲】财务的权限边界：能看什么、不能看什么、为什么（报价单管理已对财务关闭，核价单内可联查报价详情）。

### ② 核价工作台主线（45 分钟，重头戏）

按一张核价单的处理顺序，全程用**同一个料号**贯穿演示：

1. 【讲】**核价单从哪来**：销售提交报价后自动生成，财务不需要手工建；列表按状态筛选。
2. 【演示】**核价单结构**：BOM 树（物料 BOM 为主轴）+ 17 类页签（投料 / 元素 / 工序 / 能耗 / 辅料 / 人工 / 工装等），各页签数据来源。
3. 【演示】**7 项成本 metric**：各项成本如何累出总成本；小数口径 —— 计算列 4 位、对外总额 2 位。
4. 【演示】**版本选择（新能力，单独演示）**：同一料号多个数据版本时，核价侧可按页签独立切版本重算，报价侧保持冻结不变 —— 财务调价核价的核心工具。
5. 【讲】**用户差量（override）**：财务手工修正某项值时，系统记录差量而非覆盖原值。
6. 【讲】**状态机与冻结**：仅 PENDING 状态可切版本、可修改；审批通过后冻结，与后续版本变动完全隔离。
7. 【演示】从核价单**联查报价单详情**（财务查看报价上下文的正确入口）。

> ⚠️ **注意**：「7 项成本 metric」中如有依赖折扣/优惠的展示列，预演时确认当前是否有值；无值则演示时主动说明「该列依赖定价策略模块，上线后生效」，不要让听众自己发现一列空白。

### ③ 核价数据导入与版本维护（25 分钟）

财务是本功能的直接使用者，多留时间做一次完整导入演示：

1. 【讲】导入 Excel 的模板结构：每个 sheet 对应一类基础数据。
2. 【演示】**版本升级规则**：同料号再次导入 → 版本号自增、每个 sheet 独立升版、计算取最新版。
3. 【讲】常见失败与排查：料号对不上（**销售料号 vs 生产料号**的区别要讲清）、某 sheet 空导致页签显示「—」。
4. 【演示】导入后在核价单里验证生效：切到新版本，观察数值变化。

### ④ 主数据维护（15 分钟）

1. 【演示】材质库、元素主表（元素编号不可改、被引用即锁 —— 防定价断链）。
2. 【演示】工序 / 费率、能耗、工装等基础表的维护入口。
3. 【讲】料号版本管理：销售料号 / 生产料号双轨、客户料号映射。
4. 【讲】**因果关系**：主数据缺失 = 核价单页签空白 / 显示「—」；遇到「数不对」先查主数据和版本。

### ⑤ 价格构成解读 + 模板速览（10 分钟）

1. 【讲】报价价格的构成：明细行 → 行小计 → 产品小计 → 总额（含小数口径）—— 财务审核真正需要的能力，不依赖定价策略模块。
2. 【预告】**定价策略模块（🚧 开发中）**：一页 PPT，不演示，口径见第五节。
3. 【讲】核价模板 15 秒概念：核价单的页签结构由模板定义、按客户产品分类匹配 —— 让财务明白「为什么不同客户的核价单长得不一样」，出问题知道找管理员查模板。

### ⑥ 实操练习 + 常见问题 + Q&A（15 分钟）

- 【讲】排查口诀：**数不对 → 先看版本 → 再看主数据 → 最后找管理员查模板**
- 【实操】每人领一张 PENDING 核价单，完成练习（见第七节练习题）。

## 五、🚧 开发中功能的统一口径（定价策略）

> **纪律：培训全程不点击、不演示定价策略页面。** 菜单对财务可见但功能未落地，现场点开只会引发误解。

预告页（一页 PPT）的三句话口径：

1. 「客户级定价策略 / 优惠折扣模块**正在开发中**，上线后单独安排补充培训。」（不承诺具体日期，除非排期已确认）
2. 当前过渡口径：现阶段报价总价 = 各产品明细行价格汇总，客户级折扣暂由线下流程处理。*（此句需与业务现状核实后再定稿）*
3. 上线后形态一句话：「客户级折扣 / 返利规则将在报价流程中自动生效，财务审核时可看到折扣前后的价格构成。」

补充：

- 讲义 / 速查表中该功能标注 🚧「开发中」，不留空白 —— 留空会被当成漏讲。
- 可选：培训前请管理员对演示账号临时隐藏该菜单项，培训后恢复。

## 七、练习题（会后实操）

1. 找到分配给你的 PENDING 核价单，展开 BOM 树，说出任意 3 个页签的数据来源。
2. 为某一页签切换数据版本，记录切换前后的总成本变化。
3. 提交审批，观察状态流转与冻结效果。
4. （选做）从核价单联查对应的报价单详情，指出报价总额与核价总成本分别在哪里。
