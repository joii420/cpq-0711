## 1. 系统概览

### 1.1 系统是什么

CPQ（Configure 配置 / Price 定价 / Quote 报价）系统是专为制造业和工业元器件行业设计的报价管理平台。销售团队可以在系统内完成从客户基础数据管理、产品料号配置，到报价单生成、审批、输出的全流程操作，无需在多个 Excel 文件间人工维护和换算。

系统的核心价值在于：将复杂的元素 BOM、工艺费用、电镀方案等计算逻辑固化为公式模板，确保每次出价有据可查、版本可追溯、数据来源可还原。

### 1.2 解决的业务问题

| 痛点 | 系统如何解决 |
|------|------------|
| 多版本 Excel 混乱，不知哪份是最新 | 系统统一管理版本，DRAFT/SUBMITTED 状态清晰 |
| 元素价格（Ag/Au/Cu）人工查找易出错 | 元素价格中心统一录入，报价单内弹出参考价提示 |
| 客户基础资料更新后难以判断影响范围 | 漂移检测自动标注受影响报价单，黄色横幅提示 |
| 历史报价无法追溯当时用的是哪个数据版本 | 提交时自动冻结快照，数据来源 Tab 完整记录 |
| 新旧客户资料冲突处理耗时 | 导入时自动检测差异，UI 引导逐字段决策 |

### 1.3 五类核心用户

| 角色 | 英文标识 | 主要职责 |
|------|---------|---------|
| 销售代表 | SALES_REP | 创建报价单、导入基础数据、填写报价内容、提交审批 |
| 销售经理 | SALES_MANAGER | 审批报价单、查看历史版本、维护部分系统配置 |
| 系统管理员 | SYSTEM_ADMIN | 管理主数据结构、录入元素参考价、配置系统参数、监控锁状态 |
| 审计人员 | （只读访问） | 查询变更日志、查看历史快照、导出审计报告 |
| 业务管理员 | PRICING_MANAGER | 维护定价策略、查看产品与模板配置 |

## 2. 快速上手

### 2.1 登录系统

浏览器访问系统地址，输入账号和密码登录。首次登录会强制要求修改密码，修改完成后才能进入系统。如忘记密码，点击登录页"忘记密码"链接按提示操作。

[图：登录页截图]

### 2.2 管理员首次使用（5 步走）

| 步骤 | 操作 | 菜单位置 |
|------|------|---------|
| 1 | 创建用户账号，分配角色 | 系统管理 → 用户管理 |
| 2 | 创建客户，录入基本信息 | 客户管理 |
| 3 | 配置字段重要性（可选，有内置默认值） | 主数据维护 → 字段重要性 |
| 4 | 录入元素参考价（Ag/Au/Cu 等） | 系统管理 → 元素价格中心 |
| 5 | 确认系统配置阈值符合业务需求 | 系统管理 → 系统配置中心 |

### 2.3 销售首次使用（5 步走）

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 登录后进入"报价中心 → 报价单管理" | 查看已有报价单列表 |

### 3.2 每步说明

**步骤 1：创建报价单**
- 触发条件：销售收到客户询价，准备整理报价。
- 操作位置：报价中心 → 报价单管理 → 新建报价单（或"从基础数据导入"自动创建）。
- 期望结果：生成 DRAFT 状态报价单，编号自动分配。

**步骤 2：上传基础数据 Excel**
- 触发条件：客户提供料号基础资料 Excel 文件。
- 操作位置：报价单管理列表 → "从基础数据导入"按钮。
- 期望结果：系统解析 Excel，自动检测与数据库已有数据的差异和冲突。
- 常见问题：文件超过 2000 行会被拒绝，提示"请拆分导入"。

**步骤 3：处理差异与冲突**
- 触发条件：系统检测到基础资料差异（表结构级）或客户资料冲突（字段级）。
- 操作位置：导入向导步骤 5，自动弹出对应抽屉。
- 期望结果：用户确认后进入事务写入，所有变更有审计记录。
- 常见问题：备注字段为必填，不填无法确认基础资料差异。

**步骤 4：公式计算 + 漂移检测**
- 触发条件：数据写入后系统自动触发。
- 期望结果：报价单内所有公式字段计算完成。若引用的基础数据版本已变更，页面顶部出现黄色横幅"检测到数据漂移"。
- 常见问题：黄色横幅不会阻止保存，但会阻止提交。

**步骤 5：提交报价单**
- 触发条件：销售确认报价内容无误。
- 操作位置：报价单编辑页 → 提交按钮。
- 期望结果：状态由 DRAFT 变为 SUBMITTED，数据快照冻结，不可再编辑。
- 常见问题：存在 ERROR 单元格或漂移横幅时，提交按钮置灰。

## 4. 模块操作详解

### 4.1 销售代表操作手册

#### 场景 A：处理新询价（完整流程）

[图：报价单管理列表页截图]

**步骤 1：选择入口**

进入"报价中心 → 报价单管理"，点击页面右上角"从基础数据导入"按钮（如客户已提供 Excel 文件），或点击"新建报价单"手动创建。

**步骤 2：选择客户**

在导入向导第 1 步，通过客户选择器搜索或选择目标客户。若客户不存在，需先到"客户管理"页面录入。

**步骤 3：上传 Excel 文件**

向导第 2 步，点击上传区域选择 .xlsx 文件（仅支持 .xlsx 格式，不支持 .xls 或 .csv）。上传完成后系统自动解析，进度条显示解析进度。

**步骤 4：处理基础资料差异（UI-2 抽屉）**

若检测到基础资料（料号信息/BOM 结构/电镀方案）与数据库不同，系统弹出 UI-2 抽屉（宽度 960px），按数据表分组展示差异。

| 操作 | 说明 |
|------|------|
| 查看差异 | "旧值 → 新值"内联展示，字段多于 5 个时切换为左右双列对比 |
| 填写备注 | 必填，说明本次更新原因（唯一审计线索） |
| 全部确认 | 确认后写入待提交队列，继续后续步骤 |
| 取消 | 整次导入终止，不写入任何数据 |

**步骤 5：处理客户资料冲突（UI-1 抽屉）**

若检测到客户资料（组成件单价/各类费用/电镀费用）字段级差异，系统弹出 UI-1 抽屉（宽度 1200px），按"料号 × 数据表"分组折叠展示。

| 操作 | 说明 |
|------|------|
| 字段级决策 | 逐字段选择"采用导入值"或"保留现有值"，CRITICAL 字段（⭐⭐）排在顶部 |
| 批量操作 | 支持全局/分组两级批量选择 |
| 保存草稿 | 系统每 500ms 自动保存决策进度，关闭后可恢复 |
| 确认所有字段 | 所有字段均已决策后，确认按钮可用 |

**步骤 6：公式自动计算**

数据写入后系统自动计算所有公式字段，无需手动触发。公式结果实时刷新显示在报价单行项目中。

**步骤 7：填写元素单价**

对于包含元素（Ag/Au/Cu 等）的料号，系统不会自动填入元素单价（v1 版本），需销售手动在报价单元素 BOM 行填写实际单价。输入框旁会弹出参考价提示（由管理员录入的最新参考价）。

**步骤 8：检查漂移横幅**

若报价单顶部出现黄色横幅"检测到数据漂移"，说明本报价单引用的某个数据版本在导入后已被更新。点击横幅查看变更详情，可选择"刷新版本引用"按钮更新到最新版本。

**步骤 9：提交报价单**

点击右上角"提交"按钮，系统执行以下操作：
1. 校验所有公式字段无 ERROR
2. 校验无漂移横幅
3. 提交成功后状态变更为 SUBMITTED，数据快照冻结

---

#### 场景 B：审阅历史报价单

**查看数据来源**

在已提交（SUBMITTED）的报价单详情页，切换到"数据来源"Tab，可查看：
- 基础资料快照（提交时的版本）
- 客户资料引用版本（含版本号和导入时间）
- 元素单价（标注"销售手动填写"）
- 审计信息（提交人、提交时间、关联导入记录）

**字段溯源 Popover**

报价单每个字段右侧有 ⓘ 图标，悬停后显示：
- 数据来源（直接引用 / 公式计算 / 手动填写）
- 写入时间与操作人
- 对于公式计算字段，展开可查看完整计算链路
- 如果引用的基础资料快照与当前最新值不同，会显示 ⚠ 警告标识

点击"展开详情"可切换为抽屉视图，方便截图用于审计沟通。

**与最新数据对比**

在数据来源 Tab 中，客户资料引用版本行有"对比最新版本"按钮，点击打开双列对比抽屉，字段级差异高亮显示，数值变化同时展示百分比。

---

#### 场景 C：元素手填单价

1. 进入 DRAFT 报价单编辑页。
2. 找到包含元素 BOM 的产品行，展开元素明细。
3. 在"Ag 单价（¥/g）"等输入框中填写实际单价。
4. 输入框右侧参考价气泡显示：管理员参考价 / 本客户最近报价 / 30 天均价（供参考，不参与计算）。
5. 填写完成后公式字段自动重新计算。

### 4.2 销售经理操作手册

#### A. 审批报价单

1. 进入"报价中心 → 报价单管理"，筛选状态为"待审批（SUBMITTED）"。
2. 点击报价单编号进入详情页。
3. 查看报价内容、数据来源 Tab 及审批历史。
4. 点击顶部"审批"按钮，选择"通过"或"驳回"，填写审批意见。
5. 审批结果通知相关销售代表。

> v1 版本审批流程简化：销售经理直接审批，无多级流转。

#### B. 查看历史版本

1. 进入"主数据维护 → 历史版本"。
2. 按客户、料号或表名筛选。
3. 点击版本号查看详情（表格视图 / JSON 切换）。
4. 点击"对比"打开双列版本对比抽屉。
5. 历史版本为只读，不支持编辑。

#### C. 修改系统配置

销售经理可修改部分业务参数（由 SALES_MANAGER 权限限定）：
- 毛利率警告阈值（`business.gross_margin_warning_min`）
- 毛利率阻止提交阈值（`business.gross_margin_block_min`）

操作路径：系统管理 → 系统配置中心 → business 分类 → 修改对应值。

### 4.3 系统管理员操作手册

#### A. 主数据维护

[图：主数据维护总览页截图]

**数据总览页**（主数据维护 → 数据总览）

- 顶部显示各物理表的数据量与最近更新时间。
- 客户分组列表：料号数、当前版本、数据完整度、最近更新时间。
- 全局基础资料状态：mat_part / mat_bom / plating_plan 等覆盖情况。
- 底部展示最近 10 条导入记录。

**字段重要性配置**（主数据维护 → 字段重要性）

每个物理表的字段均可配置两个维度：

| 配置项 | 含义 | 说明 |
|-------|------|------|
| 重要性级别 | CRITICAL / IMPORTANT / NORMAL | 影响冲突决策时的排序和高亮 |
| 影响计算 | 是 / 否 | 若为"是"，字段变更后强制重新计算引用该字段的公式 |

系统内置了一套默认配置（参见附录），可在此页面按需覆盖。修改需二次确认。

**target_table 与 discriminator 配置**

在"配置中心 → 基础数据配置"的 Sheet 编辑抽屉中，可为每个 Sheet 配置：
- **映射物理表**：选择该 Sheet 导入时写入的目标物理表。
- **Discriminator 条件**：JSON 格式，用于多类型共用同一物理表时的区分条件（例如区分 bom_type='ELEMENT' 和 bom_type='INCOMING'）。

---

#### B. 元素价格中心

[图：元素价格中心截图]

操作路径：系统管理 → 元素价格中心

**录入参考价**

1. 点击"录入参考价"按钮（右上角）。
2. 选择元素名称（Ag / Au / Cu / Zn 等）。
3. 填写价格和单位（¥/g 等）。
4. 提交后该价格作为管理员参考价，向销售代表显示（不自动参与报价计算）。

**历史价格查询**

- 按元素名称筛选。
- 列表显示录入时间、录入人、价格、单位。
- 支持时间范围筛选查看价格走势。

> 注意：v1 版本元素价格仅供参考，不自动进入公式计算。销售代表需在报价单中手动填写实际单价。

#### C. 系统配置中心

#### E. DDL 扩列管理

操作路径：系统管理 → DDL 扩列管理

此功能用于在不发版的情况下向物理表添加新列，适用于业务方临时要求添加自定义字段的场景。

**4 步操作向导**

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 选择目标表 | 从可扩展表列表中选择目标表 | 系统只允许对指定的可扩展表操作 |
| 2 填写字段信息 | 字段名（小写字母+数字+下划线）/ 数据类型 / 注释 | 类型限于 VARCHAR/DECIMAL/INT/BOOLEAN/DATE/TIMESTAMP/JSONB |
| 3 配置重要性 | 选择字段重要性级别和是否影响计算 | 新字段默认 NORMAL / 不影响计算 |
| 4 预览 & 确认 | 查看生成的 SQL 语句，确认后执行 | 系统自动申请 DDL 锁，执行后释放 |

点击"复制 Migration SQL"可将 ALTER TABLE 语句复制到剪贴板，用于手动补录到版本控制。

**注意事项**
- 扩列期间所有新的导入请求将被暂时拒绝（DDL 锁互斥）。
- 新添加的列不可设置 NOT NULL，以避免大表写默认值的性能风险。
- 不允许添加与已有（含已删除）同名的列。

## 5. 公式配置详解

### 5.1 变量路径语法

公式中引用数据使用变量路径语法，基本格式为 `{表名.字段名}` 或 `{表名[过滤条件].字段名}`。

**基本规则**

- 路径用 `{}` 包裹
- **大小写敏感**：英文字段名必须与数据库字段名完全一致
- 表名使用中文 Sheet 名（系统会自动映射到物理表）
- 过滤条件支持多个，用 `AND` 或逗号分隔

**常用路径示例**

| 场景 | 路径写法 | 说明 |
|------|---------|------|
| 简单字段 | `{单重.unit_weight}` | 取 mat_part 表的单重 |
| 元素含量 | `{元素BOM[元素='Ag'].组成含量}` | 取银元素的 BOM 含量 |
| 多条件筛选 | `{来料BOM[料号='X1',bom_type='INCOMING'].毛用量}` | 多条件过滤 |
| IN 谓词 | `{组成件BOM[组成件料号 IN ['C001','C002']].单价}` | 匹配多个值 |
| LIKE 谓词 | `{来料BOM[料号 LIKE '%abc%'].净用量}` | 模糊匹配 |
| 嵌套查询 | `{组成件BOM[料号=元素BOM[元素='Ag'].input_material_no].单价}` | 以内层查询结果作为外层过滤条件 |

**注意事项**

- 嵌套层级最多 3 层
- 字符串字面量用单引号 `'` 包裹
- 不支持自动类型转换（字符串参与算术须用 `TO_NUMBER` 显式转换）

---

### 5.3 完整公式示例

**例 1：成品单价（含损耗）**

计算含损耗率的成品单价：

```
( {来料BOM.材料费} + {加工费.工艺费} )
  * ( 1 + {来料BOM[bom_type='INCOMING'].loss_rate} )
```

**例 2：含税价格**

```
TAX_INCLUDED(
  {组成件BOM.unit_price} * {来料BOM.net_qty},
  {报价.customer_id}
)
```

**例 3：跨币种汇兑**

将美元单价换算为人民币：

```
EXCHANGE(
  {来料固定加工费.fee_value},
  {来料固定加工费.currency},
  'CNY'
)
```

**例 4：复合聚合（含条件求和）**

统计某客户所有来料固定加工费合计：

```
SUM('来料固定加工费', 'customer_id=当前客户', 'fee_value')
```

**例 5：带默认值的查找**

查找料号对应的组成件单价，找不到时返回 0：

```
LOOKUP_OR(
  '组成件BOM',
  'component_part_no=C001',
  'unit_price',
  0
)
```

### 5.4 常见公式错误及处理

| 错误现象 | 可能原因 | 处理方法 |
|---------|---------|---------|
| 单元格显示红色 ERROR | 公式运行时错误（类型不匹配 / 路径解析失败 / 除数为零） | 点击 ⓘ 查看具体错误信息，修正路径或添加 IFNULL/IF 防空处理 |
| 报价单顶部黄色横幅 | 引用的基础数据版本已变更（漂移检测） | 点击横幅"刷新版本引用"，或点击查看变更详情后决定是否更新 |
| 路径解析失败 | 中文 Sheet 名拼写错误 / 英文字段名大小写不匹配 | 对照"配置中心 → 基础数据配置"中的 Sheet 名称和字段名重新检查 |
| 公式不生效（字段值不变） | 字段未配置为"公式类型（derived）" | 检查"配置中心 → 模板配置"中该字段的字段类型，改为"公式"并重新保存 |
| 含税/不含税函数无结果 | 客户税率未配置 | 联系管理员在客户信息中配置税率 |

### 6.2 上传 → 预览 → 决策 → 确认 4 步详解

**第 1 步：上传文件**

1. 点击导入向导"上传文件"区域，选择本地 .xlsx 文件。
2. 系统显示上传进度，解析完成后自动进入第 2 步。
3. 若文件格式错误或行数超 2,000，立即报错提示，文件不写入。

**第 2 步：预览与差异检测**

系统自动对比导入内容与数据库现有数据：
- **新增记录**：绿色标注
- **更新记录**：黄色标注（显示旧值 → 新值）
- **无变化记录**：灰色（仍可查看，不写入操作）

同时检测：
- 基础资料层差异（需走 UI-2 确认）
- 客户资料层冲突（需走 UI-1 决策）
- 业务校验问题（如含量之和不等于 100%、单重为 0 等）

**第 3 步：决策（UI-2 + UI-1）**

按顺序处理：
1. 若有基础资料差异 → 先弹出 UI-2 抽屉（整体确认/取消）
2. 若有客户资料冲突 → 再弹出 UI-1 抽屉（字段级决策）
3. 若两类均无差异 → 直接跳到第 4 步

**第 4 步：确认写入**

点击"确认导入"按钮，系统执行：
1. 单一事务写入所有 14 张物理表（全成功或全回滚）
2. 同步写入变更日志（change_log）
3. 释放产品导入锁
4. 如果从报价单页发起导入，自动创建或关联报价单草稿

## 8. 常见问题 FAQ

**Q1：上传 Excel 后报"未识别到任何 Sheet 数据"**

检查文件中的 Sheet 名称是否与系统要求的中文名完全一致（区分全角/半角空格）。例如"来料 BOM"（中间有空格）与"来料BOM"（无空格）是不同的 Sheet 名。

**Q2：公式单元格显示红色 ERROR**

点击该单元格右侧的 ⓘ 图标，查看具体错误描述。常见原因：
- 引用的 Sheet 名或字段名拼写错误（路径大小写敏感）
- 字符串参与算术运算，需用 `TO_NUMBER()` 转换
- 除数为零，在公式外层加 `IF(除数=0, 0, 原公式)`

**Q3：报价单页顶部出现黄色横幅**

这是"漂移检测"提示，说明本报价单引用的某个数据版本在创建后已被新导入的数据覆盖。点击横幅可查看具体变更内容，再决定是否更新引用版本。漂移不影响保存，但会阻止提交。

**Q4：提交按钮置灰，点击无响应**

提交前须满足以下所有条件：
- 报价单中无红色 ERROR 单元格
- 页面顶部无黄色漂移横幅
- 报价单状态为 DRAFT
- 当前用户有提交权限（SALES_REP 及以上）

**Q5：数据来源 Tab 不显示内容**

数据来源 Tab 在 SUBMITTED 状态的报价单中才显示完整快照。DRAFT 状态下，Tab 可见但内容为"当前跟随最新版本"的实时状态。

**Q6：如何取消已提交的报价单**

v1 版本不支持撤回已提交的报价单。若报价内容有误，需新建报价单重新填写。

**Q7：元素参考价在哪里录入**

由系统管理员在"系统管理 → 元素价格中心"录入。销售代表在填写报价单元素单价时，输入框旁会自动弹出最新参考价提示。

**Q8：如何给物理表添加新字段**

联系系统管理员，通过"系统管理 → DDL 扩列管理"的 4 步向导添加。操作前请确认无进行中的导入任务。

**Q9：导入卡住了，进度条不动**

可能是产品导入锁未正常释放。联系系统管理员进入"系统管理 → 锁监控"，查找活跃锁并强制释放。

**Q10：公式中配置了 ELEMENT_PRICE 函数，但计算结果为空**

`ELEMENT_PRICE` 和 `PREMIUM_PRICE` 函数在 v1 版本不可用，需等 v2 启用。v1 版本请改为销售手动填写元素单价，并在公式中通过变量路径引用报价单行数据（`{报价单.element_unit_price}`）。

**Q11：为什么某些字段在冲突决策时排在最前面**

字段重要性为 CRITICAL（⭐⭐）的字段（如单重、单价、含量等直接影响金额计算的字段）会排列在冲突决策列表顶部，并以醒目样式高亮，方便优先关注。

**Q12：变更日志中某条记录看不到字段详情**

鼠标悬停到"字段变更数量"列，会弹出气泡显示具体字段名。若需完整明细，点击该行展开或点击"查看详情"打开抽屉。

### 9.2 系统术语表

| 术语 | 含义 |
|------|------|
| HF 料号 / 宏丰料号 | 公司内部物料编号，全局唯一标识符 |
| 客户料号 | 客户侧对物料的编号，通过"客户料号与宏丰料号的关系"Sheet 建立映射 |
| DRAFT | 草稿状态，报价单可编辑 |
| SUBMITTED | 已提交状态，数据快照冻结，不可编辑 |
| 基础资料 | 物料本质属性，全局唯一，包括 mat_part / mat_bom / plating_plan / 客户料号映射 |
| 客户资料 | 客户特定的报价数据，包括组成件单价、各类费用、电镀费用等 |
| 版本 | 客户资料每次变更产生新版本号，基础资料不走版本（直接覆盖） |
| 漂移检测 | 报价单引用的数据版本与当前最新版本不一致时触发的检测机制 |
| 快照 | 报价单提交时将引用版本、元素单价等信息冻结保存，历史不可篡改 |
| UI-1 | 客户资料字段级冲突决策抽屉（1200px） |
| UI-2 | 基础资料整体差异确认抽屉（960px） |
| CRITICAL | 字段重要性最高级，直接影响金额计算的字段（⭐⭐ 标记） |
| IMPORTANT | 字段重要性中级，影响业务逻辑的字段（⭐ 标记） |
| NORMAL | 字段重要性默认级，一般字段 |
| fee_type | 费用类型枚举，区分来料固定加工费 / 组装加工费 / 电镀费用等 |
| change_source | 变更来源，标记某次变更由哪次导入或哪次人工操作触发 |
| importance | 字段重要性（CRITICAL / IMPORTANT / NORMAL），配置在字段重要性页面 |
| DDL 扩列 | 动态向物理表添加新字段，无需发版 |
| 产品导入锁 | 导入任务期间对涉及料号的悲观锁，防止并发冲突 |
| DDL 全局锁 | 执行 DDL 扩列时的全局互斥锁，锁活跃时所有导入请求暂停 |
| 参考价 | 管理员手动录入的元素价格，仅供销售参考，不自动参与计算 |
| 数据来源 Tab | 报价单详情页的溯源标签页，展示完整版本引用与快照信息 |
| ⓘ Popover | 报价单字段旁的追溯气泡，显示数据来源、写入时间等信息 |

### 9.3 v1 当前限制（v2 预计启用项）

| 功能 | v1 状态 | v2 方向 |
|------|---------|--------|
| 元素价格自动抓取 | 不可用，需手动填写 | 配置抓取规则，定时自动更新 |

| 批量版本软删除 | 仅单个版本操作 | 支持批量 |
| 变更日志精细度 | 当前字段级记录 | 增加操作意图分类与自动摘要 |
| 元素价格客户级配置 | 无（全局参考价） | 支持按客户配置取价规则 |
