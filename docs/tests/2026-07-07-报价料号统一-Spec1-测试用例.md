## 0. 运行方式与环境约定

**运行（在 worktree 或主仓的 `cpq-backend/`）**
```bash
# 单类
./mvnw -q -o test -Dtest=QuoteMaterialNoAllocatorTest
# 本次相关全部（v6 包）
./mvnw -q -o test -Dtest='com.cpq.basicdata.v6.**'
```

**环境约定（重要）**
- 全部是 `@QuarkusTest`，**连远程共享 PostgreSQL**（`10.177.152.12/cpq_db`），启动时 Flyway `migrate-at-start` 会应用 V308/V310。
- 每个测试类用**独特前缀**隔离数据（`QMNT-`/`MCMR-`/`QMNI-`/`TESTMNR-` 等），`@BeforeEach`+`@AfterEach` 双向清理，可反复跑、并发安全。
- 报价料号格式断言统一用正则 **`^\d{4}-\d{4}\d{6}$`**（`XXXX-YYMMNNNNNN`，四位客户码-年月+6位流水），**不写死具体号**（号随客户码/月份/流水变）。

**已知非本次失败（跑全量时会看到，非回归）**
- `VersionedV6MasterDetailTest`（2 个）：`CHILD_UQ=Map.of()` 空实现导致，**master 上先于本次就失败**（git stash 背靠背证实），见 BACKLOG **BL-0019**。
- 全量套件其它模块失败（`mat_bom` ON CONFLICT 漂移 / V5 导入事务嵌套）为共享 dev DB 既有 tech debt，与本次 0 关联。

**E2E（待办）**：`quotation-flow.spec.ts` 因 V308 清空共享 DB 基础数据，须**重导基础数据后**才能跑（协议级，改了 `MaterialBomMergeHandler`）。

## 3. `QuoteMaterialNoIntegrationTest` — 端到端集成（8 用例）

> 被测：`Q02CustomerMapHandler` / `MaterialBomMergeHandler` / `Q07IncomingOtherFeeHandler` / `P05CustomerMapHandler` 协同

| # | 用例 | 测试点 | 预期正确结果 |
|---|------|--------|------------|
| 1 | `q02_writesQuoteSystemType` | Q02 写 QUOTE | 导入 `(报价料号, 客户产品编号)` → `material_customer_map` 该行 `system_type='QUOTE'`、`material_no`=报价料号、`customer_product_no`=客户产品编号 |
| 2 | `q02_relabel_readsQuoteHeaderOrFallsBackToHongfengHeader` | 列 relabel + 回退 | 一行用「报价料号」表头、一行用「宏丰料号」表头 → **两行都被读到并登记**（各 1 行） |
| 3 | `q02_replace_onlyNarrowsQuoteCustomerProductRows` | 先删后插收窄 | 预置该客户 QUOTE+客户料号 / QUOTE+组件NULL / PRICING 三行，导入 1 新行后：旧 QUOTE+客户料号行**被删**、组件行**存活**、PRICING 行**存活**、新行**存在** |
| 4 | `q02_crossCustomerCollision_recordsErrorAndDoesNotOverwrite` | Q02 跨客户串号 | custA 导入报价料号 R 成功(`failedRows=0`)；custB 用同 R 导入 → `failedRows=1`、错误信息含"跨客户"；R 的 `customer_no` 仍 custA、`customer_product_no` 仍 CPN-A（**不被覆盖**） |
| 5 | `materialBomMerge_unnumberedComponent_mintsQuoteMaterialNo` | BOM 无号组件发报价料号 | 组成件料号空+有名 → `material_bom_item.component_no` 匹配 `^\d{4}-\d{4}\d{6}$`、**不以 9 开头**；该号已登记 `material_customer_map` QUOTE 行、`customer_no`=当前客户 |
| 6 | `q07_unnumberedIncomingMaterial_mintsQuoteMaterialNo` | 费用 handler 无号料件发号 | Q07 无号来料 → `unit_price.code` 匹配报价料号格式；该号登记为 QUOTE 行 |
| 7 | `p05_writesPricingSystemType` | P05 走 PRICING | P05 导入行 → `material_customer_map` 该行 `system_type='PRICING'` |
| 8 | `materialBomMerge_crossCustomerComponentNo_recordsErrorAndSkipsRowOnly` | **🔒 Blocker-1 守卫**：跨客户 per-row 降级 + sheet 不回滚 | custA 先铸报价料号 R；custB 一张 2 行 sheet（行1=R 跨客户、行2 无号正常）→ `totalRows=2`、`failedRows=1`、`successRows=1`、`errors[0].rowNo=1` 含"跨客户"；**行2 真落库**（`material_bom_item` 有 1 条当前行、≠R、格式正确）；R 归属仍 custA（**整表未回滚、好行未被静默丢失**） |

> ⚠️ 用例 8 是"跨客户异常触发 silent-rollback（`@Transactional` 拦截器 `setRollbackOnly`）"回归的**唯一守卫**（靠"行2 真落库"这条读已提交数据的断言证明）。**勿删或简化为单行场景**（单行测不到该层）。
