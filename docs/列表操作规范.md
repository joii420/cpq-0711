# CPQ 列表操作规范

> **作用**：所有"列表 + 操作列"型页面统一的设计/实现规范。
> **首次落地**：2026-05-05
> **配套实现**：`cpq-frontend/src/components/SelectableTable.tsx`
> **适用对象**：新功能开发、列表页重构、PR 评审。任何新增列表页面都必须按本规范实现，已有列表如果要改动也需顺手对齐。

---

## 0. 一句话总结

**行只承载数据 + 一个"主入口"链接。所有变更/状态切换/危险动作上提到顶部工具栏，按 selection 谓词决定启用/禁用。**

---

## 1. 设计原则

| 原则 | 落地 |
|---|---|
| **行不再装动作按钮** | 行内除"主入口"外无任何 `<Button>` / `<Popconfirm>` |
| **选择驱动启用** | 工具栏按钮 enable/disable 完全由 `selectedRows` 决定 |
| **永久可见 + 禁用态** | 工具栏一直在，按钮未选时变灰 + tooltip 给原因，比"选中后浮现"更易发现功能 |
| **统一多选** | 所有列表都是多选 checkbox。单选场景用 `enabledWhen` 限定 `length === 1` |
| **跨页保留选中** | 翻页 / 筛选切换不丢；离开页面才清 |
| **危险动作走 Modal** | 列出所选项让用户最后核对；不再用零散 `Popconfirm` |
| **部分失败可观察** | 批量调用用 `Promise.allSettled`，失败明细 `message.error` 列前 5 条 |

---

## 2. 何时用 / 何时不用

### ✅ 用 SelectableTable

满足以下**任意一条**：
- 有变更类动作（编辑、删除、发布、归档、状态切换…）
- 有危险动作（删除、停用、强制释放）
- 多选有真实价值（批量删除、批量审批）
- 状态依赖的动作（仅 DRAFT 可删除等）

### ❌ 不用 SelectableTable，保留行内链接

- **纯查看页**：行内只有"详情"`<a>` / "下载"按钮，且无副作用 → 强行选行多此一举（如 ImportHistoryList、ChangeLogCenterPage）
- **Master-Detail 双栏**：左侧导航列表用于切换右侧详情，行点击是导航语义不是选择语义（如 ImportConfigManagement）
- **树+编辑器**：根本不是列表（如 ComponentManagement）
- **Drawer 内部子表**：嵌套表格，不是页面级列表（如客户管理 Drawer 里的联系人 sub-table）

## 3. API 与数据结构

### 3.1 ToolbarAction\<T\>

```ts
interface ToolbarAction<T> {
  key: string;
  label: string;
  icon?: React.ReactNode;
  /** 危险动作（删除/停用）。会以红色按钮 + 红色确认按钮渲染 */
  danger?: boolean;
  /**
   * 启用判定：
   *   true            → 启用
   *   false           → 禁用，tooltip 显示通用提示「请先选择行」
   *   非空字符串       → 禁用，tooltip 显示该字符串作为禁用原因
   */
  enabledWhen: (selectedRows: T[]) => boolean | string;
  /** 危险/不可逆动作建议 true，弹 Modal 列出所选项二次确认 */
  needsConfirm?: boolean;
  /** Modal 标题，可包含 `{N}` 自动替换为所选数 */
  confirmTitle?: string;
  /** Modal 主描述（红色后果说明放这里） */
  confirmDescription?: string;
  /** 真正执行动作。允许 async；返回 Promise 期间按钮 loading */
  onClick: (selectedRows: T[]) => void | Promise<void>;
}
```

### 3.2 SelectableTable Props

```ts
<SelectableTable<T>
  rowKey="id"                     // 必填：唯一键
  columns={dataColumns}           // 列定义（不再含"操作"列）
  dataSource={list}               // 当前页数据
  loading={loading}
  pagination={...}                // 同 antd Table
  toolbar={<>新建按钮 + 筛选区</>}  // 顶部永久工具栏（永远可见，与选择无关）
  actions={[...]}                 // 工具栏动作列表
  rowLabel={(r) => r.name}        // 确认弹窗里展示的行标签
  getCheckboxProps={(r) => ({ disabled: r.status === 'INACTIVE' })}  // 可选：禁用某些行的选择
/>
```

### 3.3 runBatch helper（批量失败聚合）

```ts
import { runBatch } from '../../components/SelectableTable';

await runBatch(
  rows,
  (r) => fooService.delete(r.id).then(() => undefined),  // perRow，必须返回 Promise<void>
  {
    rowLabel: (r) => `${r.code} ${r.name}`,
    successMsg: `已删除 ${rows.length} 项`,
    concurrent: true,  // 默认 true；后端约束（如限流）改为 false 串行
  }
);
```

行为：
- 用 `Promise.allSettled` 聚合，**全部成功** 显示 `successMsg`
- **部分失败** 显示 `message.error`：「成功 N 项，失败 M 项：\n• 行A 错误描述\n• 行B 错误描述...」（前 5 条）
- 不一刀切回滚——成功的就生效，失败的列出原因让用户处理

## 4. 主入口列规则

每个列表保留一个"主入口"列（业务 ID / 名称），点击进详情或编辑：

```tsx
{
  title: '名称',
  dataIndex: 'name',
  key: 'name',
  render: (v: string, r: any) => (
    <a onClick={(e) => { e.stopPropagation(); navigate(`/foo/${r.id}`); }}>
      {v}
    </a>
  ),
}
```

**关键点**：
- 必须 `e.stopPropagation()`，否则会与"行点击切换选中"冲突
- 不强制选行——直接点 → 直接进入；行点击其他位置 → 切换选中

## 5. enabledWhen 写法范式

### 5.1 基础组合

```ts
// 单选
enabledWhen: (rows) => rows.length === 1 ? true : '编辑一次只能选一行',

// 多选 + 状态依赖
enabledWhen: (rows) => {
  if (rows.length === 0) return false;
  if (rows.some((r) => r.status !== 'DRAFT')) return '仅草稿状态可发布';
  return true;
},

// 跨字段组合（角色 + tab + 状态）
enabledWhen: (rows) => {
  if (!isPendingApprovalTab) return '请切到「待我审批」tab 后再审批';
  if (rows.length !== 1) return '审批一次只能选一行（需要填写意见）';
  if (rows[0].status !== 'SUBMITTED') return '仅审批中状态可通过';
  if (!canApprove) return '您不是该报价单的审批人';
  return true;
},
```

### 5.2 关键约定

| 返回 | 含义 |
|---|---|
| `true` | 启用 |
| `false` | 禁用 + 通用 tooltip "请先选择行" |
| 非空字符串 | 禁用 + tooltip 显示该字符串 **作为操作引导** |

**反模式**：
```ts
// ❌ 隐藏不可用按钮
{record.status === 'DRAFT' && <Button>删除</Button>}

// ✅ 始终显示，按状态禁用 + tooltip 教学
enabledWhen: (rows) => rows.every(r => r.status === 'DRAFT') ? true : '仅草稿状态可删除'
```

理由：用户看不到按钮时不知道有这个能力；看到灰按钮 + 解释才能学会"什么时候才能用"。

## 6. 危险动作的 Modal 模式

### 6.1 简单批量删除

```ts
{
  key: 'delete',
  label: '删除',
  icon: <DeleteOutlined />,
  danger: true,
  enabledWhen: (rows) => {
    if (rows.length === 0) return false;
    if (rows.some((r) => r.status !== 'DRAFT')) return '仅草稿状态可删除';
    return true;
  },
  needsConfirm: true,
  confirmTitle: '确认删除选中的 {N} 项？',
  confirmDescription: '⚠️ 此操作不可撤销。已发布过的版本请改用「归档」。',
  onClick: async (rows) => {
    await runBatch(rows, (r) => fooService.delete(r.id).then(() => undefined), {
      rowLabel: (r) => `${r.code} ${r.name}`,
      successMsg: `已删除 ${rows.length} 项`,
    });
    refresh();
  },
}
```

确认 Modal 自动列出所选项的 `rowLabel`，让用户最后核对。

### 6.2 需要文本输入的动作（如审批意见）

`needsConfirm: false`，在 `onClick` 里手动开自定义 Modal：

```ts
{
  key: 'approve',
  label: '审批通过',
  enabledWhen: (rows) => /* ... */,
  onClick: (rows) => {
    setActionTargets(rows);     // 把所选行暂存到 page state
    setApproveComment('');
    setApproveModalOpen(true);  // 开自定义 Modal
  },
}
```

自定义 Modal 关闭时记得清 `actionTargets` 状态。

## 7. 行为细节

| 场景 | 行为 |
|---|---|
| 单击行任意非链接区域 | 切换选中（点 a / button / checkbox / popover / modal 不触发） |
| 翻页 / 筛选切换 | 选中状态保留（preserveSelectedRowKeys） |
| 离开页面 | 选中状态丢弃（不持久化到 URL，避免分享链接歧义） |
| `Esc` | 取消所有选择（点工具栏「取消选择」按钮等价） |
| 已选 0 行 | 工具栏显示"未选择行" + 灰按钮；选中行 ≥ 1 显示"已选 N 项" + 蓝色高亮背景 |

## 8. 标准迁移 diff

迁移老页面（或参考新页面）的 diff 模式固定：

### 8.1 删除"操作"列

```diff
const columns = [
  { title: '名称', dataIndex: 'name', key: 'name', render: ... },
  // ... 数据列
- {
-   title: '操作', key: 'actions',
-   render: (_, record) => (
-     <Space>
-       <a onClick={() => openEdit(record)}>编辑</a>
-       <Popconfirm onConfirm={() => handleDelete(record.id)}>
-         <a style={{ color: 'red' }}>删除</a>
-       </Popconfirm>
-     </Space>
-   ),
- },
];
```

### 8.2 主入口列改为 `<a>` 链接

```diff
- { title: '名称', dataIndex: 'name', key: 'name' },
+ {
+   title: '名称',
+   dataIndex: 'name',
+   key: 'name',
+   render: (v, r) => (
+     <a onClick={(e) => { e.stopPropagation(); openEdit(r); }}>{v}</a>
+   ),
+ },
```

### 8.3 加 actions 配置

```diff
+ const actions: ToolbarAction<MyType>[] = [
+   {
+     key: 'edit', label: '编辑', icon: <EditOutlined />,
+     enabledWhen: (rows) => rows.length === 1 ? true : '编辑一次只能选一行',
+     onClick: (rows) => openEdit(rows[0]),
+   },
+   {
+     key: 'delete', label: '删除', icon: <DeleteOutlined />, danger: true,
+     enabledWhen: (rows) => rows.length > 0 ? true : false,
+     needsConfirm: true,
+     confirmTitle: '确认删除选中的 {N} 项？',
+     onClick: async (rows) => {
+       await runBatch(rows, (r) => myService.delete(r.id).then(() => undefined), {
+         rowLabel: (r) => `${r.code} ${r.name}`,
+         successMsg: `已删除 ${rows.length} 项`,
+       });
+       refresh();
+     },
+   },
+ ];
```

### 8.4 用 SelectableTable 包装

```diff
- <Table
+ <SelectableTable<MyType>
    rowKey="id"
    columns={columns}
    dataSource={data}
    loading={loading}
    pagination={...}
+   toolbar={<>{/* 新建按钮 + 筛选 */}</>}
+   actions={actions}
+   rowLabel={(r) => `${r.code} ${r.name}`}
  />
```

## 10. PR 自检清单

新增/改动列表页时按以下逐条核对：

### 数据层
- [ ] `rowKey` 设置为业务唯一 ID（不要用 index）
- [ ] 后端支持的所有筛选维度都暴露在 `toolbar` 里

### 列定义
- [ ] **没有**"操作"列
- [ ] 主入口列是 `<a onClick={(e) => { e.stopPropagation(); ... }}>`，而不是 `<Button>`
- [ ] 状态/类型/标签等显示用 `<Tag>` 而不是纯文字

### 动作工具栏
- [ ] 每个 action 有 `enabledWhen`，且失败分支返字符串而不是 false（除非真的就是"请先选择行"通用提示）
- [ ] 危险动作（删除/停用/强制操作）都设了 `danger: true` + `needsConfirm: true`
- [ ] `confirmTitle` 包含 `{N}` 占位符
- [ ] `confirmDescription` 写出后果（如"⚠️ 此操作不可撤销"）
- [ ] 批量场景用 `runBatch`，配 `rowLabel` 让失败明细可读
- [ ] action 的 `onClick` 在 `await` 完成后调 `refresh()` 重新拉数据

### 行交互
- [ ] 点行选中行为正常（不会因为列里嵌了 `<a>` `<button>` 而误触发）
- [ ] 翻页 / 筛选切换时选中保留
- [ ] 主入口链接点击后能跳转/打开 Drawer，不会同时切换选中

### 文案
- [ ] tooltip 文案是**操作引导**（"请切到「待我审批」tab"）而不是"无法操作"这种废话
- [ ] 确认 Modal 文案区分"操作目标"（标题）和"后果说明"（description）
- [ ] 危险后果用警告 emoji + 红色文案（"⚠️ 此操作不可撤销"）

### 角色权限
- [ ] 角色无操作权限时 → `actions: hasPerm ? [...] : []`（空数组让工具栏只显示"未选择行"提示，不再用 if-return-null 在 render 里判断）

### 表外动作
- [ ] 与"选中行"无关的纯导航动作（如「模板绑定」按钮永远跳同一页）→ 放 `toolbar` 而不是 `actions`
- [ ] 顶部"新建"永远独立放 `toolbar` 右侧

## 11. 反模式（PR 评审 Reject 理由）

| 反模式 | 为什么不行 |
|---|---|
| 行内放 `<Popconfirm>` | 多选删除时看不到具体在删谁，事故风险高 |
| 动作隐藏在 dropdown menu (⋮) | 隐藏 = 不可发现；用 disabled + tooltip 显式表达 |
| 用 `if (!canEdit) return null` 隐藏按钮 | "为什么没有这个按钮"对用户是黑盒，禁用 + tooltip 才能教学 |
| 危险动作走 `Popconfirm` | 多选场景看不到目标列表，无法核对 |
| 单选场景用 radio + 手动管选中状态 | SelectableTable 多选默认就够了，自己加 radio 是重复轮子 |
| 在工具栏放"模板绑定"这种**与所选无关**的导航按钮放进 `actions` | 永远不会变灰，但放在动作区会让用户以为它依赖选中 |
| 翻页时 `setSelectedKeys([])` 清选中 | 影响"50 条批量删除"等真实场景；用 preserveSelectedRowKeys 默认即可 |
| 用 `confirm()` (浏览器原生) 二次确认 | 形态不一致 + 无法列出所选项；统一走 SelectableTable 内置 Modal |

## 12. 例外白名单

下列页面被本规范明确豁免（理由见 `docs/RECORD.md` 同期记录）：

- `ImportHistoryList`、`ChangeLogCenterPage`、`FieldImportancePage` — 纯查看 / 单条编辑无批量场景
- `ImportConfigManagement` — Master-Detail 双栏布局
- `ComponentManagement` — 树+字段表+公式编辑器，非列表
- `BasicDataConfig` — Master-Detail 双栏 + 多 tab
- `SnapshotTab` — Drawer 内部子组件
- 任何 Drawer / Modal 内的嵌套子表（如客户管理的联系人 sub-table）

如果未来这些页面增加了批量动作，需要重新评估是否切到 SelectableTable。

## 维护

本规范由 `cpq-frontend/src/components/SelectableTable.tsx` 实现兜底——任何对规范的调整应**先改组件，再改文档，最后扩散到具体页面**，避免文档与实现脱节。

PR 中如需违反本规范的特例，必须在 PR 描述里写明理由 + 在 `docs/RECORD.md` 增加豁免说明。
