## 1. 背景与目标

### 1.1 核心理念
「导入」与「选配」只是同一份「料号 + 报价数据」的**两个入口**，落库结果（目标表 / 字段 / 版本 / 隔离维度）必须**完全一致**。本方案不为选配打补丁，而是建立一套**统一的 V6 入库规范**：

- **唯一蓝本**：`docs/table/报价系统Excel导入落库方案.md` 各 sheet 的落库逻辑即写入契约，选配照此对齐。
- **查询契约反向定义写入契约**：driver 视图读哪张 V6 表的哪些字段，入库就必须把对应数据按相同维度写进那张表。
- **V44 全系列禁用**（`mat_part`/`mat_bom`/`mat_process`/`mat_composite_process` …）：规范不展开 V44 写入，仅规划退场（详见 AP-53）。

### 1.2 产物
- 本设计方案（决策固化）。
- 后续据此产出《选配 V6 入库规范》正式文档 + 实现计划（writing-plans 阶段）。

### 1.3 范围边界
- **覆盖**：选配（SIMPLE / COMPOSITE）全部落库 → 全部压 V6 基础资料层，不碰 `quotation_line_*` per-quote 报价表、不碰 V44。
- **暂不改导入代码**，但所有规则写成通用形式，导入将来可复用。

## 3. 现状摸底结论（决定工作量）

经代码 + 迁移 + 表结构调研，确认与 `选配落库字段清单.md` 描述的差异：

### 3.1 各页签 driver 现状（关键纠偏）
| 页签 | 组件 CODE | 当前 driver 视图 | FROM 的表 | 选配现状 |
|------|-----------|-----------------|----------|---------|
| 材质 | COMP-CFG-MATERIAL-RECIPE | `composite_child_materials_mirror` | **V6 `material_bom_item`** | ✅ 已写（自指行 characteristic=NULL） |
| 元素 | COMP-CFG-ELEMENT-BOM | `composite_child_elements_mirror` | **V6 `element_bom_item`** | ✅ 已写（characteristic=2000） |
| 子配件 | COMP-CFG-CHILD-PARTS | 跨组件引用 materials_mirror | **V6 `material_bom_item`** | ✅ 已写（characteristic=ASSEMBLY） |
| **工序** | COMP-CFG-PROCESS | `composite_child_processes_mirror` | **per-quote `quotation_line_process`** + BOM fallback | ❌ 未写 `unit_price` |
| **组合工艺** | COMP-CFG-COMPOSITE-PROC | `composite_process_mirror` | **per-quote `quotation_line_composite_process`** | ❌ 未写 `capacity` |

> **结论**：「材质/元素/子配件」三页早已是 V6（视图虽名为 `_mirror`，FROM 的是 V6 基础表），选配也早已写 V6 → 本期仅**文档化**。真正要改的只有「工序」「组合工艺」两页。

### 3.2 选配后端写入现状（`ConfigureProductService`）
- V44 双写：`mat_part`/`mat_bom`/`mat_process`/`mat_composite_process`（**待删**）。
- V6 已写：`material_master`、`material_bom_item`（自指 + ASSEMBLY）、`element_bom_item`；**未写主表** `material_bom`/`element_bom`（子项1 待补）。
- per-quote 已写：`quotation_line_process`、`quotation_line_composite_process`（**待停**）。
- 指纹复用 `lookupHfByFingerprint` 仍查 V44 `mat_part`（**待切 V6**）。
- `FingerprintCalculator` 已支持 2 参（不含工序）/ 3 参（含工序）；选配调 2 参，符合 D4。

### 3.3 表结构关键事实
- `unit_price`：`version_no VARCHAR(20) NOT NULL`，**无 is_current**；`pricing_price` V276 起允许 NULL。唯一键 13 维含 `version_no`。
- `capacity`：`calc_version VARCHAR(20)`，**已有 `is_effective`**，**无 `customer_no`**（仅按料号×工序×资源组×版本）。唯一键 `(material_no, process_no, resource_group_no, COALESCE(calc_version,''))`。
- `element_bom`/`element_bom_item`：版本就是 `characteristic VARCHAR(100) NOT NULL`（'2000'+1）。
- `material_bom`：主表有 `bom_version VARCHAR(20) NOT NULL`；`material_bom_item` 子表**无独立版本列**，`characteristic` 是语义标签（NULL=材质自指 / ASSEMBLY=子配件），非版本号。
- `material_master`：`config_fingerprint`（V266）、`material_recipe_id`（V265）已就绪，一料一行无版本。

## 4. 各类数据 → V6 落库规范

> 固定值通用规则（对齐导入「三、通用落库规则」）：`system_type=QUOTE`；`customer_no` 由系统按报价单客户提供；布尔字段是→1/否→0；导入前清洗空行。

### 4.1 料号身份 → `material_master`（对标导入 §2/§3/§18）
- upsert by `material_no`：`material_type`、`material_name`、`unit_weight`、`material_recipe_id`、`config_fingerprint`。
- 一料一行，**不版本化、不加 is_current**。
- 指纹复用：`SELECT material_no FROM material_master WHERE config_fingerprint=:fp`（替代 V44 `mat_part`）。

### 4.2 材质自指行 → `material_bom_item`（对标导入 §3）
- `characteristic=NULL`、`material_no=component_no=料号`（自指）、`component_usage_type=recipe.symbol`、`seq_no=1`。
- 主表 `material_bom`（子项1 补写）：`bom_type=MATERIAL`、`bom_version`、`is_current`。

### 4.3 元素 → `element_bom` + `element_bom_item`（对标导入 §4）
- 主表 `element_bom`（子项1 补写）：`characteristic`（版本，'2000'+1）+ `is_current`。
- 子表 `element_bom_item`：每元素一行，`component_no`=元素代码、`content`=含量%、`characteristic`=对齐主表版本。

### 4.4 子配件（COMPOSITE）→ `material_bom_item`（对标导入 §12）
- `characteristic='ASSEMBLY'`、`material_no`=父料号、`component_no`=子料号、`composition_qty`、`issue_unit`、`seq_no`。
- 主表 `material_bom`（子项1 补写）：`bom_type=ASSEMBLY`、`bom_version`、`is_current`。

### 4.5 工序 → `unit_price`（对标导入 §10 自制加工费）★ 本期补写
- 固定：`system_type=QUOTE`、`price_type=MATERIAL`、`cost_type=自制加工费`。
- 字段：`finished_material_no`=料号、`code`=投入料号、`operation_no`=工序编号、`seq_no`、`currency`、`unit`。
- `pricing_price` = **NULL**（子项3：单价由报价单 INPUT 层维护）。
- 版本化（§5）：分组键 `(system_type, price_type, cost_type, code, finished_material_no, operation_no)`；内容/行数不同 → `version_no` max+1 + `is_current` 翻转。

### 4.6 组合工艺 → `capacity`（对标导入 §14 组装加工费）★ 本期补写
- 字段：`material_no`=父料号、`process_no`=`def_code`、`resource_group_no=QUOTE_ASSEMBLY`、`seq_no`、`production_type`（沿用导入默认）、`fixed_cost`=NULL（同子项3 语义，单价 INPUT 维护）。
- 版本化：分组键 `(material_no, process_no, resource_group_no)`；同父料号组合工艺不同 → `calc_version` max+1 + `is_current` 翻转。
- **不再写** `participating_parts`/`param_values`（D9 取消）。

## 5. 统一版本化机制 + `is_current` 全局加列

### 5.1 版本化写入语义（统一）
1. 分组键 = 各表唯一键**去掉版本维度**后的字段组合。
2. 写入时按分组键查现有：内容/行数与最新版本**完全一致** → 复用（不新增）；**不一致** → 取该组 `max(数字版本)+1` 写新行。
3. 起始版本 `'2000'`（字符串存储，数字递增语义）。
4. 新行 `is_current=true`；同分组键旧行批量 `UPDATE … SET is_current=false`。
5. 渲染视图统一 `WHERE is_current=true`。

### 5.2 `is_current` 加列清单（16 张带版本 V6 表，统一新增）
| 表 | 版本字段 | 已有 is_effective | 本期选配涉及 |
|----|---------|:---:|:---:|
| `unit_price` | `version_no` | — | ✅ 工序 |
| `capacity` | `calc_version` | ✅ | ✅ 组合工艺 |
| `material_bom` | `bom_version` | — | ✅（子项1 主表） |
| `element_bom` | `characteristic` | — | ✅（子项1 主表） |
| `element_bom_item` | `characteristic` | — | ✅ 元素 |
| `plating_scheme` | `scheme_version` | — | ❌ |
| `production_energy` | `calc_version` | — | ❌（核价） |
| `auxiliary_energy` | `calc_version` | — | ❌（核价） |
| `tooling_cost` | （内含 is_effective） | ✅ | ❌ |
| `production_consumable` | `consumable_version` | — | ❌ |
| `packaging_consumable` | `packaging_version` | — | ❌ |
| `electricity_price` | `version_no` | — | ❌（核价） |
| `labor_rate` | `version_no` | — | ❌（核价） |
| `exchange_rate_v6` | `version_no` | — | ❌ |
| `fee_config` | `pricing_version_no` | — | ❌ |
| `material_version_mgmt` | `pricing_version_no` | ✅ | ❌ |

- 统一 `ALTER TABLE … ADD COLUMN is_current BOOLEAN NOT NULL DEFAULT TRUE`（含已有 `is_effective` 的表）。
- `is_effective` **保留不改名**；**生效判定统一以 `is_current` 为权威**。
- ⚠️ 实现期必排查：`is_effective` 现有引用（尤其**核价侧**视图 / 代码），确保切到 `is_current` 不破坏核价主线（详见 §9 风险）。

## 11. 风险与自检

### 11.1 风险
- **R1（高）**：`is_effective` → `is_current` 生效判定切换可能影响**核价系统**（核价大量用 V6 表 + `is_effective`）。须全工程 grep `is_effective` 引用，逐一评估。
- **R2（中）**：工序/组合工艺 driver 退场 mirror，COMPOSITE 父级聚合子件路径（V273/V274 的 UNION + BOM fallback）须在新 V6 视图重新覆盖，否则父级工序页回归空。
- **R3（中）**：删 V44 写入后，存量依赖 V44 的指纹复用须确认已全部迁到 V6（历史选配料号指纹是否回填 `material_master.config_fingerprint`）。
- **R4（低）**：`capacity` 无 `customer_no` → 组合工艺全客户共享（符合 D8，但需确认业务可接受）。

### 11.2 强制自检（按 CLAUDE.md）
- 后端：touch Java 重启 → `/q/health` 200；Flyway `success=t`。
- 视图 DDL 后必重启清进程级缓存。
- 协议级改动（`ComponentDriverService`/`ConfigureProductService`/模板 snapshot 迁移）→ 跑 E2E `quotation-flow.spec.ts`（SIMPLE）+ `composite-product-flow.spec.ts`（COMPOSITE），`'加载中' final count = 0`。
- 工序页 / 组合工艺页：选配后渲染**有数据**（修复前空 → 修复后有值），附截图。
