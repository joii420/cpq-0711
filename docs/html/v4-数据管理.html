📝 设计要点
命名空间分离
「配置中心」放业务建模/计算类配置（组件、模板、数据源、BOM、公式）；「数据管理」放维护类基础数据（材质、工序、客户、料号、供应商）。角色清晰，权限矩阵不会两边重叠。
为何"基础数据配置"改名"主数据管理"
原名包含"配置"二字易与"配置中心"混淆；行业通用术语是 Master Data Management（MDM = 主数据管理），更准确地描述其"权威数据源"职能。
历史版本 / 字段重要性 去哪了
这两项不再独立菜单：「历史版本」放在"主数据管理 → 某个数据集 → 详情页 → Tab：历史版本"；「字段重要性」放在"主数据管理 → 数据集设置 → 字段配置"。下沉至详情后用户在做主数据维护时随手能用，不必跨菜单跳转。
权限
「数据管理」目录默认对 SALES_MANAGER / SYSTEM_ADMIN 可见；销售代表 / 定价经理仅在需要时通过具体子菜单获取权限。
