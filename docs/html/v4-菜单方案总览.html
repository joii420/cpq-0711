🎯 本次改版的 4 条设计原则

① 场景聚合 把"做同一件事要跳的菜单"折叠到同一页面。例：组件配置时随手开全局变量；模板配置时随手对比版本 / 改标签 / 绑产品。
② Tab 替代独立菜单 当 N 个菜单的字段结构相近、用户在工作时会来回切换时，合并为 Tab：产品管理（3 Tab）、BOM 管理（3 Tab）、数据管理子页。
③ 按钮 + 抽屉替代独立菜单 当某功能是"主页面的附属配置"时，下沉为按钮 + Drawer（不离开主页面上下文）。例：全局变量、业务标签、版本比对、产品绑定、导入历史。
④ 目录按角色分层 报价中心（销售看）、定价管理（定价经理看）、配置中心（业务建模角色）、数据管理（数据维护角色）、系统管理（管理员）。每个目录的常驻用户角色清晰。
