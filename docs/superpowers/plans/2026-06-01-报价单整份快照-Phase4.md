**Architecture:** Phase 2 已落地"快照→driverExpansions 停 batch-expand"（Task 8a）。Phase 4 在此之上：① 先做两块低风险（autosave 瞬态自愈 + 组合产品夹具/spec）；② 再做耦合高风险核心块（渲染读 `formulaResults`/`editRows` ⟺ 编辑写 `editRows`，必须一起做否则双源错配）；③ 最后结构脱钩 + 退役旧存储 + 对账收口。每块独立 E2E 守护，绝不混提。

**Tech Stack:** React + AntD（前端渲染/编辑）；Quarkus + Panache（`CardSnapshotService`/`QuotationResource`）；Playwright（`e2e/task8-snapshot-render.spec.ts`，QT-20260601-1482 苏州西门子，系统 google-chrome）。
