# 核价单表与报价单/核价单状态机重构 实现计划（第一期）

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development 逐任务实现。步骤用 `- [ ]` 复选框跟踪。
> 隔离 worktree 在执行期由 superpowers:using-git-worktrees 创建（**起手先 `ls` 确认含最新依赖特性**：`ProductDetailViews.tsx`/`ReadonlyExcelView.tsx`/`CostingOrderListPage.tsx`/`CostingReviewPage.tsx`/V304，避免 Phase1 stale-base 事故）。

**Goal:** 把第一期精简 `costing_order` 升级为完整核价单实体（单号/显式状态/驳回原因/冻结 DTO/按提交累积），理顺报价单×核价单状态机，核价工作台读冻结副本只读渲染，核价管理列表增强。

**Architecture:** 后端 `costing_order` 表演进（V305）+ `CostingFreezeService` 提交时组装 frozen_dto；状态同步挂在 submit/costingApprove/costingReject/withdraw/beginEdit；前端工作台 frozen 模式渲染 + 列表身份键迁移 + 老审批退役。务实版真冻结：冻结构(enrich 后 componentData + gvDefs) + 依赖既有 costing 卡片/Excel 零计算快照冻值，无独立 path-cache。

**Tech Stack:** Java 17 + Quarkus + Hibernate Panache + Flyway + PostgreSQL JSONB；React + Ant Design + TS + Playwright。

**权威 spec:** `docs/superpowers/specs/2026-06-29-核价单表与报价单核价单状态机重构-design.md`（v3）。每个任务的细节以该 spec 对应章节为准。
