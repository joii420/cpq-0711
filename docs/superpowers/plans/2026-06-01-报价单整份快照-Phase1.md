# 报价单整份快照（渲染脱钩）· Phase 1 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改变任何渲染行为的前提下，建立"报价单级 4 份结构快照 + 产品行级 4 份值快照"的数据模型与写入链路，并给组件管理加 `rowKeyFields` 行键配置（含强制校验）。

**Architecture:** 加性改动（only write, never read）。新增独立表 `quotation_view_structure`（报价单级 4 份结构）+ `quotation_line_item` 新增 4 个值列；新建 `CardSnapshotService` 在所有"加产品"入口生成四份结构+四份初始值；组件新增 `row_key_fields` 列 + 保存校验。渲染端完全不动，Phase 1 末尾用"快照值 vs 实时值"对账验证正确性。

**Tech Stack:** Java 17 / Quarkus 3.23.3 / Hibernate Panache / Flyway / PostgreSQL 16 JSONB；前端 React + Ant Design + TypeScript。

**Spec:** `docs/superpowers/specs/2026-06-01-报价单产品行整份快照-design.md`

**范围外（本 Phase 不做）:** 渲染读快照（Phase 2）、草稿态打开重刷（Phase 2）、编辑回写（Phase 2）、核价/报价 Excel 读快照（Phase 3）、ComparisonView + 旧链路降级 + 删存量（Phase 4）、`CostingSheet` 提交单据流程（始终范围外）。

## Task 3: 组件行键校验（TDD）

**Files:**
- Modify: `cpq-backend/src/main/java/com/cpq/component/service/ComponentService.java`（校验入口约 `validateFields()` / `create()` / `update()`）

**校验规则（软硬分级，防卡死存量组件）:** 组件 `dataDriverPath` 非空（driver 驱动、可能多行）且 `fields` 含至少一个可编辑字段（`field_type` ∈ 可录入集合，以现有 `VALID_FIELD_TYPES` "用户可录入"类为准）时，需声明 `rowKeyFields`，且每个名字存在于 `fields[].name`。
- **新建组件（`create`）→ 硬拦**（不满足抛业务异常）。
- **存量组件更新（`update`）→ 软校验**（不满足仅 `LOG.warn`，不阻断，避免现有组件保存回归）。
- **哨兵豁免**：`rowKeyFields == ["__seq_no__"]` 视为已显式声明（按行号对齐、自担风险），直接通过。
- 无 `dataDriverPath` 或无可编辑字段 → 豁免。

- [ ] **Step 3: 实现 validateRowKeyConfig 并接入 create/update**

在 `ComponentService` 加方法（用项目已有的 Jackson `ObjectMapper`；可编辑字段集合参照 `docs/组件管理字段配置指南.md §十一` 的"用户可录入"类型）:

```java
private static final java.util.Set<String> EDITABLE_FIELD_TYPES =
    java.util.Set.of("INPUT_NUMBER", "INPUT_TEXT", "LIST_FORMULA"); // 以指南矩阵的可录入类为准

private static final org.jboss.logging.Logger LOG = org.jboss.logging.Logger.getLogger(ComponentService.class);

/** hard=true: 新建路径, 违规抛异常; hard=false: 更新路径, 违规仅告警(不阻断存量组件保存). */
public void validateRowKeyConfig(String dataDriverPath, String fieldsJson, String rowKeyFieldsJson, boolean hard) {
    if (dataDriverPath == null || dataDriverPath.isBlank()) return; // 单行/固定 → 豁免
    com.fasterxml.jackson.databind.JsonNode fields = readJson(fieldsJson);
    boolean hasEditable = false;
    java.util.Set<String> fieldNames = new java.util.HashSet<>();
    for (var f : fields) {
        fieldNames.add(f.path("name").asText());
        if (EDITABLE_FIELD_TYPES.contains(f.path("field_type").asText())) hasEditable = true;
    }
    if (!hasEditable) return; // 纯只读 driver 组件 → 豁免
    com.fasterxml.jackson.databind.JsonNode keys = (rowKeyFieldsJson == null || rowKeyFieldsJson.isBlank())
        ? null : readJson(rowKeyFieldsJson);
    if (keys == null || !keys.isArray() || keys.isEmpty()) {
        fail(hard, "含可编辑字段的多行组件必须声明 rowKeyFields"); return;
    }
    // 哨兵: ["__seq_no__"] = 显式豁免(按行号对齐)
    if (keys.size() == 1 && "__seq_no__".equals(keys.get(0).asText())) return;
    for (var k : keys) {
        if (!fieldNames.contains(k.asText())) {
            fail(hard, "rowKeyFields 引用了不存在的字段: " + k.asText()); return;
        }
    }
}

private void fail(boolean hard, String msg) {
    if (hard) throw new IllegalArgumentException(msg);
    LOG.warnf("[rowKeyFields soft-validation] %s", msg);
}

private com.fasterxml.jackson.databind.JsonNode readJson(String s) {
    try { return new com.fasterxml.jackson.databind.ObjectMapper().readTree(s == null ? "[]" : s); }
    catch (Exception e) { throw new IllegalArgumentException("invalid json", e); }
}
```
接入: `create()` 持久化前调 `validateRowKeyConfig(c.dataDriverPath, c.fields, c.rowKeyFields, true)`（硬拦）；`update()` 调同方法但 `hard=false`（软校验）。

@Transactional
public void snapshotLineValues(com.cpq.quotation.entity.QuotationLineItem li) {
    var q = com.cpq.quotation.entity.Quotation.findById(li.quotationId);
    if (q == null) return;
    li.quoteCardValues   = safe(() -> buildCardValues(li, q.customerTemplateId));
    li.quoteExcelValues  = safe(() -> buildExcelValues(li, q.customerTemplateId));
    if (q.costingCardTemplateId != null) {
        li.costingCardValues  = safe(() -> buildCardValues(li, q.costingCardTemplateId));
        li.costingExcelValues = safe(() -> buildExcelValues(li, q.costingCardTemplateId));
    }
    li.cardSnapshotAt = java.time.OffsetDateTime.now();
    li.quoteValuesAt  = li.cardSnapshotAt;
    li.persist();
}

private String safe(java.util.concurrent.Callable<String> c) {
    try { return c.call(); } catch (Exception e) { return null; } // 降级，单份失败不连坐
}

> 实现 note（**不双写**，见 spec §1.4）：`buildCardValues` 必须**复用 `ConfigureSnapshotService.snapshotLines` 同一次 expand 展开结果**组装新列，**不另跑一遍 expand**（否则加产品耗时翻倍 + 两套值漂移）。落地方式：让 `snapshotLines` 把它已算出的每组件 `ExpandDriverResponse.rows` 传给 `CardSnapshotService` 组装 `quote_card_values`/`costing_card_values`（baseRows 直接取这批 rows）。展开逻辑保持现状纪律（COMPOSITE 子件聚合 AP-45 / 行数纪律 AP-51：`rowCount>0?rowCount:baseRows.length`，禁 `Math.max` / V6 表 AP-53 / DATA_SOURCE binding 不丢 AP-39）。对每行按组件 `rowKeyFields` 从 `driverRow` 拼 `rowKey` 写进 baseRows/formulaResults 索引。`buildExcelValues` 调 `ExcelViewService.buildRowData(li, columns, templateId, formulaByName, customerId)`（现有签名，执行时核对）取每行 `{colKey:value}`。
