# 核价 BOM 递归展开 组件级开关 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把核价 BOM 递归展开从 P1「核价侧一律生效」改为组件级开关 `bom_recursive_expand`（默认开）：勾选才按 BOM 闭包递归展开（树+系统列），不勾选按根料号单料号普通渲染。

**Architecture:** Component 新增独立布尔列（默认 true，与语义不同的 `tree_config` 区分）。后端 `CardSnapshotService.expandTemplateDriverBaseRows(...,closure)` 按组件 `bomRecursiveExpand` 分流（true→`expandForPartSet`+`buildSpineBaseRows`；false→`expand`+`buildBaseRowsFromRows`）。前端 `QuotationStep2` 引入 `activeComponentBomTree` 数据驱动标志（按该组件行是否含 `__sys.nodeId`），替换裸 `cardSide === 'COSTING'` 的系统列/建树渲染。组件管理加 Checkbox。

**Tech Stack:** Java 17 + Quarkus 3.23.3、Hibernate/Panache、PostgreSQL 16、Flyway、JUnit5 `@QuarkusTest`、React + Ant Design + TypeScript、Playwright E2E。

**Spec:** `docs/superpowers/specs/2026-06-06-核价BOM递归展开-组件级开关-design.md`

**前置现状（已核对）：**
- 当前最大 Flyway 版本 `V294` → 本计划用 `V295`（实施时若被并发会话占用，取当时最大+1）。
- `Component` 实体（`com.cpq.component.entity.Component`）已有 `tree_config`/`data_driver_path` 等；本期加 `bom_recursive_expand`。
- create/update 共用 `CreateComponentRequest`（无 `UpdateComponentRequest`）。`ComponentService.create`=line 70、`update`=line 128。
- `ComponentDTO.from` 映射在 line 37-50；`treeConfig` 映射 line 50。
- `CardSnapshotService`：普通展开 `expandTemplateDriverBaseRows(tid,li,cid,qid)`=line 889（用 `expand`+`buildBaseRowsFromRows`）；闭包重载 `(...,closure)`=line 945（用 `expandForPartSet`+`buildSpineBaseRows`，for 循环 line 962-981）。
- `QuotationStep2.tsx`：系统列/建树裸 `cardSide === 'COSTING'` 在 line 1618(表头)、1779(`isBomTree`)、1815(`bomSys`)、1819(单元格)、1971(tfoot 占位)；`__sys` per-row 透传 line 1745；活动 Tab 展开数据 `activeDriverExpansion`。

```java
        String partNo = li.productPartNoSnapshot;   // false 分支单料号 expand 用
        QuotationIdContext.set(quotationId);
        try {
            for (Object[] dc : driverComps) {
                if (dc == null || dc[0] == null) continue;
                String cidStr = dc[0].toString();
                boolean recursive = !(dc[1] instanceof Boolean) || (Boolean) dc[1]; // 默认/非布尔 → true
                UUID compId = UUID.fromString(cidStr);
                try {
                    if (recursive) {
                        // 勾选：BOM 闭包递归 → spine 全节点行 + __* 系统列
                        Map<String, ExpandDriverResponse> byPart = componentDriverService.expandForPartSet(
                            compId, customerId, closure.partSet, li.id, compositeType);
                        baseRowsByComp.put(cidStr, buildSpineBaseRows(closure, byPart));
                    } else {
                        // 未勾选：按根料号单料号普通展开（无系统列），等同报价侧取数
                        ExpandDriverResponse exp = componentDriverService.expand(
                            compId, customerId, partNo, null, null, null, li.id, compositeType);
                        List<ExpandDriverResponse.Row> rows =
                            (exp != null && exp.rows != null) ? exp.rows : new ArrayList<>();
                        baseRowsByComp.put(cidStr, buildBaseRowsFromRows(rows));
                    }
                } catch (Exception e) {
                    LOG.warnf("[card-snapshot] expand(recursive=%b) comp=%s li=%s: %s",
                              recursive, compId, li.id, e.getMessage());
                    baseRowsByComp.put(cidStr, MAPPER.createArrayNode());
                }
            }
        } finally {
            QuotationIdContext.clear();
        }
        return baseRowsByComp;
```

> 注：`compositeType` 变量在原 line 960 已声明（`String compositeType = li.compositeType;`），保留不动；本替换段从原 `QuotationIdContext.set` 开始。确认 `java.util.ArrayList` 已 import（line 889 普通重载已用 `new ArrayList<>()`，同文件已 import）。
