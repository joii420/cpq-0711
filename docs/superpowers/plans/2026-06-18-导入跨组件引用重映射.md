# 组件导入跨组件引用重映射(Bug2)Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development。每个 Task 派全新子代理,TDD,Task 间两阶段评审。

**Goal:** 修复"导入组件时不重映射跨组件引用",使导入副本的 `cross_tab_ref.source` / `targetExpr[].source`(UUID)和 `component_subtotal.component_code`(code)指向导入后的新副本,而非源目录的原组件;并修复已导入到罗克韦尔目录的断裂引用,让 `罗克韦尔模板0617` 能发布。

**Architecture:** 抽一个纯函数 `FormulaRefRemapper.remap(formulasJson, idMap, codeMap)` 统一重写三类引用(幂等:只替换 map 命中的);导出端给 bundle Item 记录原组件 id;导入 commit 两遍(先建全部新组件收集 老id→新id / 老code→新code,再重写 formulas);存量走目录级一次性补救端点(从目录内 `<base>__impN` 反推映射)。

**Tech Stack:** Quarkus + Hibernate Panache;JUnit;Jackson JsonNode。

**根因(已确诊):** `ComponentImportService.commit` 第 227-228 行 `c.formulas = nodeToJson(it.formulas)` 逐字拷贝公式,不改其中的跨组件引用 → 新副本仍引用源目录原组件 UUID/code。证据:罗克韦尔「产品」`bd3a6c14` 的 cross_tab_ref.source 仍 = `af0841d0`(报价模板V2 的「其他费用」),发布时 `validateCrossTabRefs` 报"跨页签引用的源组件不在本卡片: af0841d0"。

**⚠️ 工作区纪律(所有子代理):** 工作目录 `/home/joii/project/cpq/.claude/worktrees/component-import-ref-remap`;**绝不 cd 主仓**;提交前 `git branch --show-current` 须是 `worktree-component-import-ref-remap`;后端测试 `cd .../component-import-ref-remap/cpq-backend && ./mvnw ...`;dev server(8081)跑主工作区代码不反映 worktree 改动,本特性靠 worktree 内 `./mvnw` 单测验证,存量补救(G5)在合并后跑。

## Task G2: FormulaRefRemapper(纯函数 + 单测)

**Files:** Create `FormulaRefRemapper.java` + `FormulaRefRemapperTest.java`

- [ ] **Step 1: 写失败测试**(TDD,先写 `FormulaRefRemapperTest`):
  - 输入 formulas JSON(数组,元素含 `expression:[token...]`),idMap `{oldUuid:newUuid}`,codeMap `{oldCode:newCode}`。
  - 断言:① `cross_tab_ref.source` 命中 idMap → 替换为新 id;② `cross_tab_ref.targetExpr[].source` 命中 idMap → 替换;③ `component_subtotal.component_code` 命中 codeMap → 替换;④ **幂等/不误伤**:不在 map 里的引用(如已指向新副本的)原样保留;⑤ 非相关 token 不动;⑥ 空/畸形 formulas 不抛异常(返回原值)。
  ```java
  // 关键用例骨架
  String out = FormulaRefRemapper.remap(formulasJson,
      Map.of("af0841d0-...","7661cb7a-..."), Map.of("COMP-0028","COMP-0028__imp1"));
  // 断言 out 中 source 已变 7661cb7a;targetExpr.source 已变;component_subtotal.component_code 已变;
  // 已是 1b2d1bdb 的 source(不在 idMap)不变
  ```
- [ ] **Step 2: 跑失败** — 类不存在 → 编译失败。
- [ ] **Step 3: 实现** `FormulaRefRemapper.remap(String formulasJson, Map<String,String> idMap, Map<String,String> codeMap) -> String`:
  - 用 Jackson 解析为 JsonNode;遍历每个 formula 的 `expression` 数组每个 token:
    - `type=cross_tab_ref`:`source` 若 idMap 含则替换;`targetExpr` 数组里每个元素的 `source` 若 idMap 含则替换。
    - `type=component_subtotal`:`component_code` 若 codeMap 含则替换。
  - 只在 map 命中时改(幂等);解析失败/空 → 返回原 formulasJson。返回序列化后的 JSON 字符串。
  - 纯静态方法,无依赖注入。
- [ ] **Step 4: 跑通过**(`./mvnw test -Dtest=FormulaRefRemapperTest -q`)+ **Step 5: 提交** `feat(import-remap): FormulaRefRemapper 纯函数重写三类跨组件引用` + RECORD。
