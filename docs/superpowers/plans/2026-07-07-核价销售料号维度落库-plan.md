**Goal:** 把核价基础数据 Excel 新增的「销售料号」(=报价料号 `XXXX-YYMMNNNNNN`) 作为料号级隔离维度落进 11 张核价表（唯一键 + 升版 groupKey），实现"同生产料号、不同销售料号各自独立升版/取价/渲染"，下游核价视图按销售料号分组不扇出。

**Architecture:** 生产料号原落库列（`material_no`/`finished_material_no`/`code`）不动，旁加 `sales_part_no VARCHAR(32) NULL`；进各表 uq 末尾（`COALESCE(sales_part_no,'')`）+ `VersionedV6Writer` 调用方 groupKey；`material_master` 不变（生产料号主表）；销售↔生产↔客户三码由 `material_customer_map` 承载（报价 Spec1 已建 `system_type`/`production_no`，本期加 `sales_part_no`）。全局/元素级 5 表不加维度。

**Tech Stack:** Java 17 + Quarkus 3.34 + Hibernate Panache + Flyway + PostgreSQL 16；JUnit5 + `@QuarkusTest`（连远程 DB `10.177.152.12/cpq_db`，用独特测试料号 + `@BeforeEach/@AfterEach` 清理）；Playwright E2E。

**硬前置（已满足）：** 报价 Spec1 已合 master（V308/V310，`material_customer_map` 有 `system_type`/`production_no`）——Task 0 校验。

**关键决策（spec §11，已定）：** D4 前置已满足；D5 砍掉 P24 map 补登；D6 N>1 确认→下游视图全改无豁免；D7 labor_rate 必须加维。
