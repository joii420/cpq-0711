# cross_tab_ref 目标公式（第一期）实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** 让 cross_tab_ref 的目标从"选单列"扩展为"可写公式"（`targetExpr`），公式可引用 A 匹配行列(`field`) + B 当前行列(`b_field`) + 四则运算 + 全局变量；逐行先算再聚合。不含函数（函数=第二期）。

**Architecture:** 新增可选 token 字段 `targetExpr`（ExpressionToken[]，非空时优先于 `target`）+ 新 token 类型 `b_field`。前后端两个卡片引擎对等：evalCrossTab 对每个匹配 A 行，用现有 token 引擎递归求值 targetExpr（`field`→A 行值、`b_field`→B 当前行、global_variable→B 上下文），再按 agg 聚合。配置走 CrossTabRefDrawer 的"改用公式"模式。

**Tech Stack:** Java(Quarkus/JUnit) + React/TS(Vitest) + Ant Design Drawer。

**设计依据:** `docs/superpowers/specs/2026-06-05-跨页签目标公式-design.md`。

- [ ] **Step 4: evalCrossTab 接入 targetExpr**

在 `evalCrossTab` 内新增 helper（放在该方法内或类内私有方法），并把取值处改为调它：

```java
    /** 取某匹配 A 行的目标值: 有 targetExpr → 在 (A行 field + B行 b_field + B 上下文 gvar) 求值; 否则取 arow[target]。 */
    private Object targetRowValue(Map<String, Object> arow, JsonNode token, RowContext ctx) {
        JsonNode te = token.path("targetExpr");
        if (te.isArray() && te.size() > 0) {
            RowContext sub = new RowContext();
            for (Map.Entry<String, Object> e : arow.entrySet()) {
                Double n = toNumber(e.getValue());
                if (n != null) sub.fieldValues.put(e.getKey(), n);
            }
            sub.currentRowRaw = ctx.currentRowRaw;       // b_field 取 B 当前行
            sub.basicDataValues = ctx.basicDataValues;   // global_variable 取 B 上下文
            sub.crossTabRows = ctx.crossTabRows;
            return evaluateExpression(te, sub);          // BigDecimal（内部异常→ZERO4）
        }
        return arow.get(token.path("target").asText(""));
    }
```

修改 evalCrossTab 取值两处：
- NONE 单匹配：`return hits.get(0).get(target);` → `return targetRowValue(hits.get(0), token, ctx);`
- 聚合循环：`Double n = toNumber(h.get(target));` → `Double n = toNumber(targetRowValue(h, token, ctx));`

（`target` 局部变量保留供无 targetExpr 时回退；targetRowValue 内部自行读 token。）
