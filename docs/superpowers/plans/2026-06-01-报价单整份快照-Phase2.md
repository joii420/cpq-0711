# 报价单整份快照（渲染脱钩）· Phase 2 主体实施计划

> **For agentic workers:** 逐 Task 执行。鉴于 Phase 1 期间实现 Agent 两轮虚报"完成/测试通过"，**本 Phase 每个 Task 的测试/对账/E2E 门禁必须主线亲跑亲验**（跑测试看真实 Tests run、查 DB 看真实值、E2E 看截图），不接受 Agent 自检声明。

**Goal:** 报价卡片渲染完全读「报价单级结构快照 + 行级值快照」，与模板/基础数据彻底脱钩；草稿态打开重刷报价侧值（按 driverRow 行键保编辑）；编辑回写（写 editRows + 重算公式 + 报价 Excel 回写）；强制 E2E。

**Architecture:** 三子阶段。2a=后端暴露快照 DTO + 公式引擎搬后端算 formulaResults 落快照 + 前端读快照 hook（旁路）。2b=草稿重刷 + 编辑回写端点 + 渲染切换读快照。2c=强制 E2E + 对账收口。**核心风险=公式引擎从前端搬后端的金额漂移**，用对账门禁（后端算==前端 formulaEngine 算）守。

**Spec:** `docs/superpowers/specs/2026-06-01-报价单产品行整份快照-design.md`

**前置（已完成）:** Phase 1（数据模型+四份快照写入+对账13/13）；行键命名空间方案A修正（V279，driverRow 英文 key）。

**关键纪律:** 不双写；行键对齐用 driverRow key（已修）；公式重算口径加产品/草稿重刷/编辑三处调同一 FormulaCalculator 对账防漂移；AP-44/31/50/54/39。

**范围外:** Phase 3（Excel视图+核价卡片读快照）、Phase 4（ComparisonView+旧链路退役+回写基线+删存量）。

> 架构评审：2a 开工前过一次 cpq-architect（公式引擎搬后端 + 渲染权威迁移是基线 §4.2 破坏性改动）。
