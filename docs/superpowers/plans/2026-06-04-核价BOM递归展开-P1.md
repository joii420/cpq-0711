# 核价 BOM 递归展开 P1（默认最新 + 树 + 固定列）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: 用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实现。步骤用 `- [x]` 复选框跟踪。

**Goal:** 核价侧渲染产品卡片时，由 `material_bom_item` 算出根料号的整棵 BOM 子料号闭包，所有核价组件按整棵树取数；每个核价组件渲染层注入 3 个系统固定列（料号/父料号/版本下拉占位）+ 节点元数据，前端按 `parent_id→node_id` 建树。**P1 只做"默认最新（`is_current`）渲染 + 树 + 固定列"**；版本切换重查、Excel 树状、累乘均后置。报价侧零改动。

**Architecture:** 中央闭包注入 —— 新增 `BomClosureService` 算一次闭包（料号集合 + spine 骨架 + 环清单），把料号集合喂给现有 `:hfPartNos`，复用 `SqlViewExecutor.executeAllRows` 的 `WHERE hf_part_no = ANY(:hfPartNos)` 外包机制，业务视图零改动。固定列走**渲染期系统列（路线 b，绕开 AP-44）**，不进 `component.fields`。核价卡片数据**加产品时算好存快照**（`buildCostingCardValues → expandTemplateDriverBaseRows`），故闭包注入在快照生成处。

**Tech Stack:** Java 17 + Quarkus 3.23.3、Hibernate/Panache、PostgreSQL 16（含 `WITH RECURSIVE ... CYCLE`）、Flyway、JUnit5 `@QuarkusTest`、React + Ant Design、Playwright E2E。

**Spec:** `docs/superpowers/specs/2026-06-04-核价BOM递归展开-design.md`

## 范围说明（重要）

- **触发**：核价一律生效（走 `costingCardTemplateId` 的核价渲染全部递归），**无开关、无 schema 字段**（用户确认口径，方案 1）。
- **行为变化提示**：核价卡片快照在**加产品/重算快照时**会一次性展开整棵 BOM 树并落 `costing_card_values`。存量核价单在下次快照重算时也会变成整棵树（用户已确认核价一律生效）。
- **P1 不做**：① 节点级版本切换重查（版本下拉 P1 仅渲染 + 显示当前版，不可真正切换取数）；② `SqlViewExecutor` (料号,版本) 配对过滤；③ Excel 树状多行渲染（P1 Excel 保持现状不破）；④ 累乘用量列；⑤ 性能/并发优化（本期顺序执行，实测驱动后议）。
- **业务视图硬前提（P1）**：核价组件 `$view` 只需带 `hf_part_no` 列（同现状，零改动）。
- **闭包口径（已锁）**：`customer_no='_GLOBAL_'`、`system_type='PRICING'`、`is_current=true`、不约束 `characteristic`、PG16 `CYCLE` 防环并回传成环料号。

- [x] **Step 1: DTO**
  - `BomClosureResult { List<String> partSet; List<SpineNode> spine; List<String> cyclePartNos; }`
  - `SpineNode { String nodeId; String parentId; int lvl; String hfPartNo; String parentNo; String bomVersion; boolean isCycle; }`
  - `nodeId` = 边 id 路径拼成字符串（如 `"e1/e3/e7"`，根的子节点 `parentId=""`）。

- [x] **Step 2: 闭包查询**（裸 JDBC，参照 `SqlViewExecutor.executeJdbc` 取连接方式）
  - SQL 见 spec §3（`WITH RECURSIVE bom_tree ... CYCLE node_no SET is_cycle USING cyc_path`，节点 `LEFT JOIN LATERAL` 取当前 `bom_version`，`edge_path` 累加 `material_bom_item.id`）。
  - 签名预留 P2：`compute(String rootPartNo, Map<String,String> versionOverrides)`；**P1 内部恒按 `versionOverrides` 空处理，走纯 CTE（is_current）**。
  - `partSet` = 去环后 `DISTINCT node_no`；`cyclePartNos` = `is_cycle=true` 的 `node_no`。

- [x] **Step 3: 进程级缓存**
  - `Caffeine`，key = `rootPartNo`（P1 口径固定，无需进 key；预留 overrides 指纹位）；TTL 与 `ComponentDriverService.expandCache` 一致（30s），`maximumSize` 合理上限。
  - 暴露 `evictAll()`（基础数据导入后调用，复用现有 evict 时机）。

- [x] **Step 4: 单测** `@QuarkusTest`（命名空间化测试数据，参照 `VersionedV6WriterTest`）
  - 多层树（P→A、P→B、B→C）→ partSet 含 {P,A,B,C}，spine lvl/parentId 正确；
  - 重复子件 DAG（P→A、P→B、B→A）→ A 出现 2 行、`nodeId` 不同、各自 `parentId` 指向正确父；
  - 环（A→B→A）→ `cyclePartNos` 含环料号、不死循环；
  - 单层 / 空（无子件）→ partSet 仅根、spine 仅根行。

- [x] **Step 5: 自检**：`./mvnw -q -Dtest=BomClosureServiceTest test` 全绿。

## 验收标准（P1 Definition of Done）

1. 打开/新建核价报价单，核价产品卡片每个 driver 组件按整棵 BOM 树（默认 `is_current` 最新）多行展开。
2. 每个核价组件多出 3 系统固定列：料号 / 父料号 / 版本（下拉占位，显示当前版）。
3. 卡片按 `parent_id→node_id` 呈树（缩进），重复子件正确分裂为独立节点。
4. BOM 成环 → 截断 + 前端告警列出成环料号。
5. 报价侧（报价卡片 / 报价 Excel / 核价 Excel）行为与改动前一致（隔离）。
6. 全量自检通过（§Task 5 Step 3）。
