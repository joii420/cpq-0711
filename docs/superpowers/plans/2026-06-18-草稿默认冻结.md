# 报价单草稿默认冻结 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 DRAFT 报价单首次 bake 后默认冻结（结构+值），打开不再自动重查；仅在用户显式点「刷新基础数据」按钮时重算值；并修复触发本问题的 Bug1 字段路径不一致 + 迁移存量脏快照。

**Architecture:** 复用现有 `QuotationLineItem.cardSnapshotAt`（非空=已首次冻结）作冻结判据，零新增列。后端 `refreshQuoteCardValues`/`snapshotLineValues` 加 `force` 短路；`refreshDraftQuoteCards` 移除 `rebuildStructureForDraft`（R1 结构永冻）。前端删 on-open 自动重刷，改 Step2 工具栏显式按钮。Bug1 走全库审计 + 配置端点改 path。存量草稿走一次性 admin 端点干净重烤。

**Tech Stack:** Quarkus 3.23 + Hibernate Panache（后端）；React + Ant Design（前端）；JUnit（后端单测）；Playwright（E2E）。

**强时序约束**：Phase C（Bug1 修复）必须先于 Phase D（存量重烤），否则重烤把同样的 `#ERROR` 冻回去。

**设计依据**：`docs/superpowers/specs/2026-06-18-草稿默认冻结-design.md`

**自检纪律**：每次后端改动 touch java 强制 Quarkus 重启 + curl health；前端改动 `npx tsc --noEmit` + Vite 200；协议级（CardSnapshotService）改动强制跑 `quotation-flow.spec.ts` + `composite-product-flow.spec.ts`。详见 CLAUDE.md「修改后强制自检」。

```java
public int refreshDraftQuoteCards(UUID quotationId) {
    if (quotationId == null) return 0;
    Quotation q = Quotation.findById(quotationId);
    if (q == null || !"DRAFT".equals(q.status)) return 0; // 非 DRAFT no-op
    // R1（2026-06-18 草稿默认冻结）：显式刷新只刷"值"，不重建结构。
    //   原 self.rebuildStructureForDraft(quotationId) 调用已移除——结构创建即冻、永不变。
    List<QuotationLineItem> lines = QuotationLineItem.list("quotationId", quotationId);
    int n = 0;
    for (QuotationLineItem li : lines) {
        try {
            if (li.id != null) componentDriverService.evictForLineItem(li.id);
            self.refreshQuoteCardValues(li, true); // 显式刷新 = force 重算值
            n++;
        } catch (Exception e) {
            LOG.warnf("[card-snapshot] refreshDraftQuoteCards line=%s failed: %s", li.id, e.getMessage());
        }
    }
    return n;
}
```
