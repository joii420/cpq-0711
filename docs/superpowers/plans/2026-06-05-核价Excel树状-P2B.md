# 核价 Excel 树状视图（P2-B）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: 用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实现。步骤用 `- [ ]` 复选框跟踪。

**Goal:** 核价 Excel 视图从「1 行/产品」改为「每个 BOM spine 节点 1 行」——注入 料号/父料号/版本 三列、按 lvl 缩进成树，CARD_FORMULA 列按本节点有效行聚合；报价 Excel 零改动。

**Architecture:** 方案 A（spine 同源）——Excel 构建时复算 `BomClosureService.compute` 拿权威 spine，按 `__nodeId` 把 `costingCardValues` 的有效行过滤到本节点，复用现有 `cardFormulaEvaluator` 逐节点逐列求值。前端 `useExcelSnapshotRows` 多行解析 + `LinkedExcelView` 注入列 + 缩进。

**Tech Stack:** Java 17 + Quarkus、Jackson、JUnit5 `@QuarkusTest`、React + Ant Design Table、Playwright E2E。

**Spec:** `docs/superpowers/specs/2026-06-05-核价Excel树状-P2B-design.md`

## 范围（重要）
- **做**：核价 Excel **渲染（快照 `costingExcelValues`）路径**树化。
- **不做**：POI xlsx 导出树化（`exportExcelView` 走默认/报价模板，follow-up）；节点级版本切换（P2-A）；累乘；Excel 折叠交互。

- [ ] **Step 3: 新增 `filterByNodeId` 静态方法**（`CardEffectiveRows`）
  ```java
  /**
   * P2-B 核价 Excel 树：把每个 tab 的有效行过滤到指定 spine 节点（按行内 __nodeId）。
   * 返回新 Map（不改原 eff）；某 tab 无匹配行 → 该 tab 行列表为空（节点该组件无数据 → 列空）。
   */
  public static Map<String, TabRows> filterByNodeId(Map<String, TabRows> eff, String nodeId) {
      Map<String, TabRows> out = new LinkedHashMap<>();
      if (eff == null) return out;
      for (Map.Entry<String, TabRows> e : eff.entrySet()) {
          List<Map<String, Object>> kept = new ArrayList<>();
          for (Map<String, Object> r : e.getValue().rows) {
              Object n = r.get("__nodeId");
              if (n != null && n.toString().equals(nodeId)) kept.add(r);
          }
          out.put(e.getKey(), new TabRows(kept, e.getValue().subtotal));
      }
      return out;
  }
  ```

  ```java
  /**
   * P2-B 核价 Excel 树：按根料号 BOM 闭包 spine 逐节点出行；每行注入 料号/父料号/版本 + 树元数据，
   * 各列用「过滤到本节点 __nodeId」的有效行求值。空节点列 → null（对齐卡片"—"）。
   * 仅核价侧调用（cardValuesJson = costingCardValues，含 spine __nodeId）。
   */
  public List<Map<String, Object>> buildLineTreeRows(QuotationLineItem li, UUID templateId,
                                                     UUID customerId, String cardValuesJson) {
      List<Map<String, Object>> rows = new ArrayList<>();
      if (li == null || templateId == null) return rows;
      try {
          Template template = Template.findById(templateId);
          if (template == null || template.excelViewConfig == null || template.excelViewConfig.isBlank())
              return rows;
          List<Map<String, Object>> columns = parseJsonArray(template.excelViewConfig);
          if (columns.isEmpty()) return rows;
          List<TemplateFormulaDTO> templateFormulas = templateFormulaService.listByTemplate(templateId);
          Map<String, TemplateFormulaDTO> formulaByName = new LinkedHashMap<>();
          for (TemplateFormulaDTO f : templateFormulas) formulaByName.put(f.name, f);

          var eff = parseEffectiveRows(cardValuesJson, templateId);
          com.cpq.component.dto.BomClosureResult closure =
              bomClosureService.compute(li.productPartNoSnapshot, java.util.Map.of());

          for (com.cpq.component.dto.BomClosureResult.SpineNode node : closure.spine) {
              var effN = com.cpq.quotation.service.card.CardEffectiveRows.filterByNodeId(eff, node.nodeId);
              Map<String, Object> row = buildRowData(li, columns, templateId, formulaByName, customerId, effN);
              row.put("__hfPartNo", node.hfPartNo);
              row.put("__parentNo", node.parentNo);
              row.put("__bomVersion", node.bomVersion);
              row.put("__nodeId", node.nodeId == null ? "" : node.nodeId);
              row.put("__parentId", node.parentId);
              row.put("__lvl", node.lvl);
              row.put("_lineItemId", li.id != null ? li.id.toString() : null);
              rows.add(row);
          }
      } catch (Exception e) {
          LOG.warnf("[ExcelView] buildLineTreeRows failed li=%s tmpl=%s: %s",
              li != null ? li.id : null, templateId, e.getMessage());
      }
      return rows;
  }
  ```
  > 注：`parseEffectiveRows` 现为 private —— 同类内调用无需改可见性。`buildRowData(...,eff)` 传 `effN`，CARD_FORMULA 走 `CardDataProvider.fromEffectiveRows(effN)` 只聚合本节点行。

## 验收标准（P2-B Definition of Done）
1. 核价单 Excel 视图按整棵 BOM 树逐节点出行（= spine 节点数），前置 料号/父料号/版本 列 + lvl 缩进。
2. 各列按本节点聚合：有数据节点 CARD_FORMULA 列非空、空节点空白；DAG 重复子件出多行。
3. 叶子料号版本带出边版本（如 1630010773 = 2000）。
