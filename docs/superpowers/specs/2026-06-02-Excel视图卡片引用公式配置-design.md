# Excel 视图「卡片引用 + 公式」内容配置 — 设计方案

- 日期：2026-06-02
- 状态：已与用户确认需求，待写实现计划
- 范围：报价单/核价单 Excel 视图的**列内容配置与求值**（本轮聚焦"逐料号逐列公式"，跨料号合计行暂缓）

---

## 0. 背景与动机

当前 Excel 视图列用 BNF 路径 `$view.col` 直接查"模板 SQL 视图"取数（`SqlViewExecutor`），并按 `hf_part_no` 匹配料号。该方式存在两类痛点：

1. **配置门槛高 / 易错**：用户要写 SQL 视图 + ASCII 列名 + BNF 路径；中文列别名会被路径文法 `[a-z_][a-z0-9_]*`（`SqlViewExecutor.PATH_PATTERN` / `SQL_IDENT`）拒绝，直接渲染「—」（2026-06-02 实测根因）。
2. **数据来源割裂**：Excel 列绕过产品卡片、各自查底层资料，用户难以表达"用卡片里已经算好的结果再算"。

**新方向**：Excel 视图每一列 = 一条**公式表达式**，只引用**该料号自己产品卡片各页签已算好/填好的结果**（页签单元格值 + 页签小计），支持四则运算、聚合（含条件求和）、IF、ROUND、ABS、百分比。每个料号用同一套列公式，各算各的行。

## 2. 数据来源（数据流 A）

- 输入 = 产品卡片每个**页签(组件实例)** 的结果：
  - 行级：`QuotationLineComponentData.rowData`（JSON 行数组，每行 = `{字段key: 最终值}`，含该页签自身输入/DATA_SOURCE/公式算出的结果）
  - 页签级：`QuotationLineComponentData.subtotal`
- Excel 层**不碰底层 SQL 视图**；卡片内部怎么取数仍走它自己的 driver / DATA_SOURCE / 公式机制。
- **前置依赖**：用户须先把产品卡片填好/算好，Excel 视图才有结果可消费。卡片结果变更应触发 Excel 重算（§6）。

## 3. 引用与寻址（可视化选择器，C）

配置时在 Excel 列编辑界面用选择器生成引用，公式里显示友好占位，底层存稳定结构：

| 引用类型 | 选择器交互 | 友好占位示例 | 底层存储（ref 对象） |
|---|---|---|---|
| 页签小计 | 选页签 → 小计 | `[加工.小计]` | `{ tab, field: "__subtotal__" }` |
| 单值-首行 | 选页签 → 字段 → 取首行 | `[加工.加工费]` | `{ tab, field, mode: "FIRST_ROW" }` |
| 单值-按条件取行 | 选页签 → 字段 → 条件 | `[加工.加工费(工序=电镀)]` | `{ tab, field, mode: "ROW_WHERE", cond }` |
| 跨行聚合 | 用聚合函数包裹（§4） | `SUM_OVER(加工 WHERE …, …)` | 聚合源 `tab` = 页签实例 |
| **本表列引用** | 选"引用本表其他列" → 点选已配置列 | `[A]`、`[单位成本]` | 裸 `col_key`（无 `.`） |

**占位区分（卡片引用 vs 本表列引用）**：
- 卡片引用 = **含 `.`** 的占位 `[页签.字段]`（如 `[加工.加工费]`）。
- 本表列引用 = **裸 col_key**（无 `.`）`[A]` / `[B]`。
- 求值器按"占位是否含 `.`"区分两类，互不冲突。
- 本表列引用让 B 列公式可复用 A 列已算结果（如 A=`SUM_OVER(...)`，B=`[A] * 1.13`）。

**稳定标识纪律**：
- `tab` = **模板组件实例 ID**（`template_component.id` 或 `componentId + sortOrder`），**不是 `componentId`** —— 同一组件在模板里可多次出现（参见反模式 AP-37/AP-40），按 componentId 会串实例。
- `field` = 组件字段定义的稳定 `field_key`；选择器按显示名展示、存 key。改显示名不断公式。
- 配置时选择器的"页签清单"来自**模板已绑定的组件实例**，"字段清单"来自**组件字段定义**（配置期无运行时数据即可选）。

## 4. 公式能力

- 运算：`+ - * /`、括号
- 聚合（对某页签的行集，先按 WHERE 过滤，再对"行内表达式"逐行求值后聚合）：
  - `SUM_OVER( 页签 WHERE 条件 , 行内表达式 )`
  - `COUNT_OVER / AVG_OVER / MIN_OVER / MAX_OVER` 同形
  - 行内表达式支持逐行计算：`SUM_OVER(加工 WHERE 工序='电镀' AND 数量>0, (单价+辅料)*数量)`
  - WHERE/行内表达式里的列名 = 该页签的字段
- 条件分支：`IF(条件, 算式1, 算式2)`
- 函数：`ROUND(x, n)`、`ABS(x)`
- 百分比：公式内字面量 `12%` 解析为 `0.12`；列级可勾选"按百分比显示"（如 `0.12`→`12%`）
- WHERE 条件（C 档）：`= != > < >= <=`、`IN(...)`/包含、`AND` / `OR` + 括号组合
- **本表列引用**：公式可引用本 Excel 表已配置的其他列结果 `[col_key]`（裸标识，无 `.`），如 B 列 = `[A] * 1.13`。
- **求值顺序 + 环检测**：列间按**依赖拓扑排序**求值（不再只靠从左到右），保证被引用列先算；检测循环引用（A↔B 互引）并在保存校验/求值时报错。
- 综合示例：
  ```
  A: SUM_OVER(加工 WHERE 工序 IN ('电镀','酸洗'), 加工费)
  B: ROUND( ([投料.小计] + [A]) * 12%, 2 )
  ```

## 5. 空值规则（规则一）

1. **单值引用**为空、或引用页签 0 行 → 该引用记为空；纯单值列 → 显示「—」。
2. **聚合**命中 0 行 → 结果按 **0**。
3. **公式含多引用**：部分引用为空 → 该空引用当 **0** 继续算；但公式涉及的引用**全部为空** → 整列显示「—」。

## 6. 求值位置与落地（后端求值，已确认）

**后端求值**。理由：① 卡片结果已持久化于 `QuotationLineComponentData`（行值 + 小计），后端可直接取；② 与"公式引擎搬后端"既定方向一致；③ 可落 `quotation_line_item.excel_view_snapshot`，前端只渲染。

- 复用并扩展现有 `TemplateFormulaService`（已实现 `SUM_OVER` 等聚合 + `[col_key]` / `@变量` 解析）。
- 新增 **卡片数据提供器（CardDataProvider）**：把聚合/引用的数据源从 `DataLoader`(SQL 视图) 切到"指定页签实例的 `rowData` 行集 + subtotal"。
- 求值上下文 = 单个 `lineItem`（料号）：以其 `QuotationLineComponentData` 列表为数据域，逐列算出值。
- **列间求值顺序**：先解析各列公式的 `[col_key]` 依赖建有向图 → **拓扑排序**后逐列求值，被引用列结果缓存供后续列复用；存在环则报错（保存校验 + 运行时双重防护）。
- 触发：进入/刷新 Excel 视图、卡片数据保存后，重算该料号行（沿用 `regenerateAllSnapshots` 思路）。

## 7. 配置 / 渲染落点（新增列来源类型，与旧并存）

- **列来源类型**：新增 `source_type = "CARD_FORMULA"`，存于 `template.excel_view_config` 列对象；字段含 `formula`(含友好占位的表达式) + `refs`(占位→ref 对象映射) + `display_format`(可选百分比/小数位)。
- **并存**：旧 `$view.col`(VARIABLE) 与 `FORMULA` 渲染路径**保留**给老模板；新模板列只用 `CARD_FORMULA`。两套并存、互不影响（Q7 A + Q9 A）。
- **前端**：
  - `ExcelViewConfigTab.tsx` 增加"卡片引用选择器（页签→字段→模式→条件）+ 公式编辑器（占位 chip + 函数按钮 + 校验）"。
  - `LinkedExcelView.tsx` 渲染改读后端算好的 `CARD_FORMULA` 列值；按 `display_format` 做百分比/小数显示；空值按规则一显示「—」。
- **后端**：
  - `ExcelViewService.buildRowData` 增加 `CARD_FORMULA` 分支 → 调 `TemplateFormulaService`(扩展版) + `CardDataProvider`。
  - 列公式校验端点：保存模板时校验占位 ref 指向的页签实例/字段存在、聚合源合法、表达式语法正确。

## 8. 本轮不做 / 明确边界

- 跨料号"合计行 / 分组小计"（Q6，暂缓，后续单独立项）。
- 不批量迁移老模板，不下线旧 `$view.col` 渲染（Q9 A）。
- 不改产品卡片内部取数机制（卡片照旧）。
- 不引入文本/映射类函数（Q10 未选 B）。

## 9. 关键风险 / 待实现计划细化的点

1. **字段类型联动（AP-44）**：新增 `CARD_FORMULA` 列来源属"渲染层协议改动"，实现计划需按 17 检查点矩阵核对，并跑 E2E（`quotation-flow.spec.ts` + 组合产品 spec）。
2. **同组件多实例 cache（AP-37/AP-40）**：ref 必须按模板组件实例 ID 定位；后端聚合按实例取 `QuotationLineComponentData`（同 lineItem 下按 componentId+sortOrder 精确匹配，禁用 `firstResult()`）。
3. **行数权威（AP-51）**：聚合按页签实际行集，不得用历史持久化行数累加。
4. **占位 ↔ ref 一致性**：友好占位仅显示用；求值与校验一律走底层 ref 对象，避免改显示名/页签名导致公式断裂。
5. **百分比字面量解析**：`%` 需在公式 tokenizer 阶段处理（`12%`→`0.12`），勿与取模混淆（本语法不提供取模）。
6. **列引列拓扑求值 + 环检测**：保存模板时即校验列依赖图无环、被引列存在；运行时拓扑序求值。占位 `[X]` 含 `.` 走卡片引用、不含 `.` 走本表列引用，二者解析路径必须严格区分。
7. **空值随列引传播**：被引用列为「—」/空时，按规则一在引用方当 0 续算（全空则「—」），避免空值沿列链放大成整列「—」。
