# BL-0045 核价导入自动补 `process_master` — 设计

> 日期：2026-07-12 ｜ 来源：BACKLOG BL-0045（P1）｜ 状态：设计已确认，待写实现计划
> 前置：task-0712 主数据维护-核价基础数据维护（已交付）；tesk-0709 核价导入版本化（已交付）

## 1. 问题

核价导入（tesk-0709「导入即版本化」）落库的工序码是一套 **Z 码体系**（`Z002 / Z008 / Z053 / Z490 / Z611`），而 `process_master` 主表当初未按这套码播种——现有 41 行只有 `K1~K14`（中文名真实工序）、`MRO-XX-NNNN`（另一套结构化码）、外加零星 `Z029 / Z350`。结果：料号核价维护页（task-0712）的工序名称列（`operation_name`，靠 `readRows` 对 `process_master` LEFT JOIN 带出）对所有导入 Z 码返回 **null**。

**决定性事实**：源 Excel 每个工序 sheet 都在「工序编号」旁**自带「工序名称」列**（`Z053→铣割`、`Z008→成品清洗`、`Z611→无缝焊接`…）。名称一直在源文件里，只是导入 handler 只读了编号、没读名称、没落主表。

这把 BACKLOG 原列的「3 选 1」淘汰了两个：
- ❌ **Z→MRO 映射表**：两套码无业务对应，且根本不需要——名称现成。
- ❌ **导入时码值归一**：无归一目标，同样淘汰。
- ✅ **真正的解**：导入时读「工序名称」列 → upsert `process_master`。一次改导入，之后每次重导自动保持主表同步，名称带出立即解析。

## 2. 已确认决策（逐条与需求方澄清，2026-07-12）

| # | 决策点 | 结论 |
|---|--------|------|
| D1 | 治理方向 | **方案 A：导入时从源「工序名称」列自动 upsert 主表**（非映射表、非码值归一、非一次性回填脚本） |
| D2 | 覆盖哪些主表 | **仅 `process_master`**。元素无源名称、来料/材料品名触及核心主档 `material_master`，均另议 |
| D3 | upsert 写入语义 | **insert-if-missing（缺则插、有则不动）**。尊重任何人工改过的名称；`Z029/Z350` 已存在即不碰 |
| D4 | 挂哪个导入 / 是否回填 | **仅核价（P 系列）导入 upsert；不写一次性回填脚本**。核价马上重导即覆盖；报价（Q）侧本期不动 |
| D5 | 唯一索引 | **加 `process_master(process_no)` 唯一索引**（现无重复，已核），启用 `ON CONFLICT` + 强制工序码唯一 |

**非目标（本期不做）**：
- 报价（Q 系列）导入不写 `process_master`（Q 侧工序名仍可能 null，日后单独处理）。
- 不对现有已入库数据做回填（现有数据只存码没存名，回填只能造占位脏名；重导会带真名，优于回填）。
- 元素 / 来料 / 材料主表名称补全（D2 排除）。
- `process_master` 的其他列（category / is_outsource / currency / unit / defect_rate）不由导入写入，保持 null。

## 3. 设计

### 3.1 组件与改动

| # | 改动 | 说明 |
|---|------|------|
| C1 | **Flyway 迁移**（新增） | `CREATE UNIQUE INDEX <ux_process_master_process_no> ON process_master(process_no);`。现无重复（已核）。⚠️ 落地时取**下一个空闲版本号（≥V328，实现时复核）**——共享 `cpq_db` 有多会话 flyway 并发，勿撞号（见记忆 `cpq-shared-flyway-history-churn`） |
| C2 | **`ProcessMasterRepository.ensureByProcessNo(processNo, processName, importedBy)`** | 新增方法（该仓储已存在，Panache）。native SQL：`INSERT INTO process_master(id, process_no, process_name, created_by, updated_by) VALUES (gen_random_uuid(), ?, ?, ?, ?) ON CONFLICT (process_no) DO NOTHING`。`created_at/updated_at` 走列默认 `now()`。**有则不动**（D3） |
| C3 | **10 个 P-handler** 读名称 + 注册 | `P06 / P08 / P09 / P10 / P11 / P12 / P13 / P14 / P18 / P23`（即所有含「工序编号」列的核价 sheet）。行循环里增读 `row.getStr("工序名称")`，连同已读工序编号调 ensure |
| C4 | **`ImportContext.sharedCache` 去重** | key 约定 `"ensuredProcessNos"`（`Set<String>`）。本次导入已 ensure 过的码跳过，避免重复 DB 调用——照既有 `writtenMaterialNos` 惯例（`ImportContext` 注释已立此模式） |
| C5 | **维护页零改动** | `PricingMaintenanceService.readRows` 已对 `process_master` LEFT JOIN 带出 `operation_name`；码补进主表后名称自动解析，无需改 task-0712 任何代码 |

> 架构一致性：导入侧本就在 upsert 主表（P05 `masterRepo.upsertByMaterialNo` 同步 `material_master`；`ImportContext` 注释明示「已写 material_master 的料号集」缓存模式）。新增 `process_master` upsert 是**同一既有架构的延伸，非破例**。

### 3.2 数据流

```
核价 Excel 上传
  → P-handler 解析每行
    → 读 (工序编号, 工序名称)
    → ctx.sharedCache["ensuredProcessNos"] 命中? 命中则跳过
    → 否则 ProcessMasterRepository.ensureByProcessNo(no, name, ctx.importedBy)
         INSERT ... ON CONFLICT (process_no) DO NOTHING
    → 记入 sharedCache
  → （既有）版本化写入各核价表
维护页 readRows → LEFT JOIN process_master → operation_name 带出（非 null）
```

### 3.3 边界与错误处理

- **工序名称为空**的行：`process_name` NOT NULL → **跳过 upsert**（不造「名称=码」占位脏名，契合 D3/D4「不回填占位」口径）。该码名称仍 null，与现状一致、**不恶化**。
- **工序编号为空**：本就是现有各 handler 的必填校验（`recordError`），维持不变，不进 ensure。
- **curated 码保护**：`K*/MRO*` 不出现在核价源 → 永不被碰；`Z029/Z350` 已存在 → `ON CONFLICT DO NOTHING` 不覆盖。
- **仅核价**：只改 P-handler → 报价（Q）导入不写 `process_master`（契合 D4）。
- **同码多 sheet / 多行同名**：首次 ensure 后 `sharedCache` 跳过；跨导入用唯一索引 + `ON CONFLICT` 兜底（**幂等**）。
- **并发导入**：唯一索引 + `ON CONFLICT DO NOTHING` 保证并发插同码不产生重复行、不抛错。
- ensure 在 handler 既有 `@Transactional(REQUIRES_NEW)` 内执行，与版本化写入同事务边界。

## 4. 测试

**单元**
- `ensureByProcessNo`：不存在码 → 插入并带名；已存在码（如 Z029）→ 行数不变、名称不被覆盖；名称为空 → 不插入。
- handler：读「工序名称」列 + 注册；`sharedCache` 命中跳过重复 DB 调用。

**集成 / 端到端**
- 重导核价样例（`docs/table/核价系统功能基础数据功能结构所需字段（6.0版） .xlsx` 等）→ `process_master` 新增 `Z002 / Z008 / Z053 / Z490 / Z611`，名称为 `铣割 / 成品清洗 / … / 无缝焊接`。
- 维护页 `readRows` 各含工序码的 tab：`operation_name` 非 null。
- **幂等**：连续重导两次 → `process_master` 无重复行、无新增噪声。
- **回归**：核价导入其余落库（版本号、行数、精度）不受影响；报价导入不写 `process_master`。

**自检口径**（对齐 CLAUDE.md「修改后强制自检」）
- 后端：`touch` 触发 Quarkus 重启 → 目标 endpoint 200/401；Flyway 新迁移 `success=t`；唯一索引存在（`\di` / `information_schema`）。
- 迁移前复核 `process_master` 无重复 `process_no`（现已核，实现时再确认一次）。

## 5. 验收标准

1. 加 `process_master(process_no)` 唯一索引，Flyway 迁移 `success=t`，无重复冲突。
2. 核价导入后，源 Excel 出现过的工序 Z 码全部落入 `process_master` 且名称正确（铣割 / 成品清洗 / 无缝焊接…）。
3. 料号核价维护页含工序码的各 tab，工序名称列带出非 null。
4. `Z029 / Z350` 及 `K*/MRO*` 既有行不被覆盖（名称/其他列保持原值）。
5. 工序名称为空的行不产生占位脏行；报价导入不写 `process_master`。
6. 连续重导两次无重复 `process_master` 行（幂等）。
7. 核价导入其余落库口径（版本化、行数、精度）无回归。
