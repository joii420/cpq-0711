- **日期**: 2026-06-02

- **关联记录**: `docs/RECORD.md`（2026-06-02 选配链路 V44 排查系列）、`docs/反模式.md` AP-53（V44 表已废弃核心规则）

## 1. 背景与问题

报价单「选配添加」链路在**渲染/读取侧已迁 V6**（driver 路径、SQL 视图读 `material_master`/`element_bom_item`/`unit_price`），但**指纹判重 + 落库写入侧仍在 V44 双写**（`mat_part`/`mat_bom`/`mat_process`/`mat_composite_process`）。

实证（2026-06-02）：`mat_process` 当天仍有选配写入（`insertProcesses` 无条件执行，无开关）。代码注释明示这是**故意的过渡期双写**，原因是指纹去重强约束链：`uq_mat_part_fingerprint`（在 mat_part 上）+ `mat_process.hf_part_no → mat_part.part_no` FK。过早切 V6 → 指纹漏判 → 重复建料号 → FK 断裂（409）。

本方案把选配写路从「V44+V6 双写」切到「**V6 单写**」，为将来彻底退役 V44（删表）铺路。

## 2. 目标与边界（已与用户确认）

| 维度 | 决定 |
|---|---|
| **范围** | **只改选配写路**（ConfigureProductService）切 V6 单写 + 顺带 3 个下游读路加 V6。**删表是后续独立事，本次不删。** |
| **历史数据** | **不迁移**。切点后在 V6 重新积累。只存在于 V44 的历史选配料号，切后不再能指纹复用（接受重新生成）。 |
| **切换方式** | **干净大切换**，一次到位。无特性开关、无双写灰度、无回滚阶段。 |
| **料号身份** | **全局唯一 + 全局指纹去重**，延续 V44 语义。 |
| **下游耦合** | 选配停写 V44 后，3 个仍读 V44 的下游（提交快照 / 漂移检测 / loadLineItems）**本方案顺带改读 V6**，保证选配产品端到端可用。 |

**明确不在范围**（声明，避免误解）：
- 导入侧（`BasicDataImportServiceV5` / `StagingWriter` / `StagingMerger` / `DiffDetector`）仍读写 V44 → **本次后 V44 表仍不能删**。
- `PartVersionService` / `ElementPriceService` / `ValveDemoSeedResource` 的 V44 使用。
- 删 V44 表本身（DROP）。

## 3. 当前状态盘点（关键：V6 基建大半已就位）

### 3.1 已就位的 V6 基建
- `material_master` 是**全局表**（无 customer_no，PK=material_no）且**已有 `config_fingerprint` 列**；`insertMaterialMasterV6` **已在写 fingerprint** → 全局指纹去重的 V6 落点现成。
- 选配 `configure()` 主流程**已双写** V6：
  - 材质 → `insertMaterialMasterV6`（含 config_fingerprint）
  - 元素 → `insertElementBomV6`（→ `element_bom_item`，按客户）
  - 材质自指 BOM → `insertMaterialBomItemV6`（→ `material_bom_item`）
  - 组合 ASSEMBLY → `insertMaterialBomAssemblyV6`
  - 组合工艺 → `insertCompositeProcessCapacityV6`（→ `capacity`）
  - 工序单价（**仅组合**）→ `insertProcessUnitPriceV6`（→ `unit_price`）
- **工序渲染已全读 V6 `unit_price`**（`v12_labor_cost`/`v12_incoming_proc`/`v12_finished_proc` 等 v12_* 视图），与 `mat_process` 无关。

### 3.2 唯一实质性缺口
**简单料号的工序只写 V44 `mat_process`，没有 V6 `unit_price` 写入**（`insertProcessUnitPriceV6` 是组合专用，简单路径 :301 调 `insertProcesses(mat_process)`）。
→ 推论：简单配置料号的工序当前可能**根本没在 V6 渲染**（既有隐性缺口）。本方案必须补「简单料号工序 → V6 unit_price」写入。

## 4. 改动设计（按依赖顺序）

### A. 指纹权威切 V6
- Flyway 迁移：`material_master.config_fingerprint` 加 **UNIQUE 索引**（全局去重约束，对应替代 `uq_mat_part_fingerprint`）。
  - **前置**：建索引前先扫存量 `material_master` 是否已有重复 `config_fingerprint`（双写期可能有脏数据），有则先清理，否则建索引失败。
- `lookupHfByFingerprint`：`SELECT part_no FROM mat_part WHERE config_fingerprint=:fp` → `SELECT material_no FROM material_master WHERE config_fingerprint=:fp`。

### B. 补简单料号工序 V6 写入（缺口）
- 新增 `insertProcessSimpleUnitPriceV6(hfPartNo, processIds, customerCode)`：把简单料号选中的工序写入 V6 `unit_price`，列口径对照 v12_* 视图读取方式（process_code / assembly_process / 加工费维度）。
- 简单路径（resolvePart :301）从 `insertProcesses(mat_process)` 改调 `insertProcessSimpleUnitPriceV6`。
- 验证：简单料号配工序后，v12 工序视图 / 报价工序 Tab 正确显示。

### C. 停所有 V44 写入（选配内）
- 删调用：`insertMatPart` / `insertElementBom` / `insertProcesses` / `insertProcessesWithLineItemId` / `insertAssemblyBom` / `insertCompositeProcesses` + resolvePart 的 `DELETE mat_process`。
- 删双写桥：`backfillV44FromV6`、`backfillV6FromV44`（V6 单写后不再需要）。
- 删死代码：`copyElementBom` / `readElementsFromMatBom`。
- 对应方法体一并删除（避免留死代码）。

### D. 选配内 V44 读改 V6
- `buildSnapshot`（复用回显）：mat_part→material_master（单重）、mat_process→unit_price（工序）、mat_composite_process→capacity（组合工艺）。
- `backfillProcessesForNewCustomer`：mat_process→unit_price（跨客户复制工序；对齐 unit_price 的 customer 维度）。

### E. 3 个下游读路加 V6
- 见 §3.4。改读 material_master / element_bom_item，字段按 V6 列映射对齐。

## 5. 风险点

1. **指纹唯一索引落地**：建 UNIQUE 前必须清存量重复 `config_fingerprint`，否则迁移失败（启动 Flyway 报错）。
2. **简单料号工序 V6 模型**（最大的真·新开发）：`unit_price` 列结构能否承载 `mat_process` 的 process_code/assembly_process 语义，需对照 v12_* 视图读取口径设计；若 unit_price 维度不够，可能需扩列或建中间映射。
3. **FK 解耦**：`mat_process.hf_part_no → mat_part.part_no` —— 停写后该 FK 不再触发，但要确认无残留写入路径撞它。
4. **跨客户复用 unit_price 维度**：`backfillProcessesForNewCustomer` 改 V6 后，unit_price 的客户维度复制要对齐。
5. **V44 表本次删不掉**：导入侧仍读写 mat_process/mat_part/mat_bom → 本次只停选配写入，**表保留**。方案显式声明，不在交付承诺里含 DROP。
6. **提交快照口径漂移**：SnapshotCollector 改读 V6 后，快照字段（part_name/material/unit/quantity）要与 V6 列映射对齐，避免提交快照内容变化。
7. **大切换无回滚**：干净大切换 = 一旦上线即生效，无开关。需在合并前用 E2E 充分验证简单 + 组合两条选配路径。

## 6. 验证策略

- **写入对账**：选配简单料号 + 组合料号各跑「确认产品」，验 V6 表（material_master/element_bom_item/material_bom_item/unit_price/capacity）写入齐全、V44 表（mat_part/mat_bom/mat_process/mat_composite_process）**零新增**（对比 created_at）。
- **指纹复用**：相同配置再选一次 → 复用同一全局料号（查 material_master.config_fingerprint 命中）。
- **工序渲染（验缺口 B）**：简单料号配工序 → v12 工序视图 / 报价工序 Tab 显示正确。
- **提交快照（验 E）**：提交一张选配产品报价单 → submission_snapshot 非空且 part_name/material/元素 字段正确。
- **漂移检测（验 E）**：选配产品跑漂移检测不报错、元素对账正确。
- **E2E**：`composite-product-flow.spec.ts`（组合）+ 选配简单路径 E2E（夹具需就绪）。

## 7. 交付边界声明

本方案**交付**：选配写路 V6 单写（停 V44 写）+ 简单料号工序 V6 写入 + 3 下游读路 V6 + 指纹权威切 V6。
本方案**不交付**：删 V44 表、导入侧迁移、其它子系统（PartVersion/ElementPrice/Drift 写入侧）迁移。这些属「V44 整体退役路线图」后续阶段。
