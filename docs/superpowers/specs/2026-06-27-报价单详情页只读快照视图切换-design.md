## 1. 背景与目标

报价单**详情页**（`QuotationDetail.tsx`，只读路由）当前**只展示报价单一侧的产品卡片**（`ReadonlyProductCard`），用户要看 Excel 视图、核价单、比对视图，必须进入**编辑向导**（`QuotationStep2.tsx`）。

编辑向导有两级视图切换：
- 上层 `mainTab`（Segmented）：报价单 / 核价单 / 比对视图
- 下层 `viewType`（Segmented）：📋 产品卡片 / 📑 Excel 视图

**目标**：把这套两级切换以**只读、读快照**的形式搬到详情页，让用户**不进入编辑态**也能查看报价单的全部视图。详情页打开**零计算、零 driver 展开、零后端 ensure**，所有展示一律读已持久化的快照 JSON——符合"提交后全部数据冻结"的业务语义。

## 3. 现状盘点（实现期需复核的事实基线）

> 以下为 brainstorming 阶段读码确认的事实，作为实现起点；实现期对每条做一次现码复核（记忆/快照形状可能演进）。

### 3.1 详情页与三套视图的取数现状
- **详情页** `cpq-frontend/src/pages/quotation/QuotationDetail.tsx`：在"产品明细"卡片内渲染 `ReadonlyProductCard` 列表（`filter(li => li.compositeType !== 'PART')`）。
- **报价·卡片**：`ReadonlyProductCard.tsx` 已脱钩读快照——`useSnap = !!lineItem.quoteCardValues && components.length>0`；FORMULA 优先读 `formulaResults[rowKey]`（真零计算），`buildSnapshotExpansions` 构造 driverExpansions（不 batch-expand），无快照才回退实时；依赖 `quoteCardStructure`（提供 rowKeyFields）。**这一格已具备。**
- **报价·Excel**：`LinkedExcelView.tsx` QUOTE 侧走 `frontendRows = buildExcelSnapshot(...)`，**前端现算**（非读 `quoteExcelValues`）。`quoteExcelValues` 已落库但当前仅供导出消费。
- **核价·Excel**：`LinkedExcelView` COSTING 侧走后端 `getExcelView` / `ensureExcelValues`（懒算）。
- **比对**：`ComparisonView.tsx` 用 `useLinkedExcelRows` 对两侧逐格求值（有 `__loading__`、"正在求值…"）。`buildComparisonModel`（`comparisonModel.ts`）本身是**纯函数**：吃已算好的行值 → 按 `comparison_tag` 映射 + 差异比对，不跑公式。

### 3.2 后端快照落库现状
- `CardSnapshotService.ensureExcelValues`（`CardSnapshotService.java:582`）：对整单一次 `IN` 预取 componentData（`ExcelCompDataContext`，免逐行查），逐行 `buildExcelValues`，**`== null` 才算**，把 `quoteExcelValues` + `costingExcelValues` 落库；核价 Excel 基于已冻结的 `costingCardValues` 做**公式再投影**（非 driver 展开）。
- 调用点：①独立端点 `POST /{id}/ensure-excel-values`（`QuotationResource.java:304`）；②**submit 内** `QuotationResource.java:365`：`cardSnapshotService.ensureExcelValues(id)`（尽力，不阻断提交）。
- saveDraft：P3 lazy，`computeExcel=false` → Excel 值留 NULL（`CardSnapshotService.java:520/560`）。**故草稿态 Excel 快照为空 → 与 D6 一致。**
- 结论：**提交后** `quoteCardValues / quoteExcelValues / costingCardValues / costingExcelValues` 全部齐备 → 详情页可读全部 5 视图。

## 4. 架构（选项 A：薄只读渲染器 + 复用只读卡片）

**原则**：详情页一次 `getById` 拿全 5 视图所需的全部快照（值 + 结构 + Excel 列定义）；各只读渲染器是**纯函数**，输入快照 JSON、输出 UI，内部**不发起任何计算 / 展开 / 后端值求值请求**。

| 视图 | 渲染器 | 吃的快照字段 | 工作量 |
|---|---|---|---|
| 报价·卡片 | 复用 `ReadonlyProductCard` | `quoteCardValues` + `quoteCardStructure` | 已具备 |
| 核价·卡片 | `ReadonlyProductCard` 加 COSTING 分支 | `costingCardValues` + `costingCardStructure` | 加分支 |
| 报价·Excel | 新建 `ReadonlyExcelView` | `quoteExcelValues` + 报价 Excel 列定义 | 新增 |
| 核价·Excel | 同 `ReadonlyExcelView` | `costingExcelValues` + 核价 Excel 列定义 | 新增 |
| 比对 | 新建 `ReadonlyComparison` | `quoteExcelValues` + `costingExcelValues` + `comparison_tag` | 新增 |

**比对无需新增持久化**：`ReadonlyComparison` 吃两份已落库的 Excel 快照 + `comparisonTagService.list('ACTIVE')`，喂 `buildComparisonModel`（纯 join/diff）→ 渲染。join/diff 不是"公式计算"，符合 D1 零计算。

### 反分叉纪律（AP-41 / AP-50）
把以下**纯展示/纯数据逻辑抽成共享模块**，让只读组件与编辑态组件**共用同一套逻辑、只换取数源**（快照 vs 重算），杜绝双源分叉：
- `renderCellValue` / `formatNumber`（从 `LinkedExcelView.tsx` 抽出）→ 共享给 `ReadonlyExcelView`。
- `buildComparisonModel`（已在 `comparisonModel.ts`，本就纯）→ `ReadonlyComparison` 直接复用，仅改"喂它的 rows 来源"为快照。
- 比对列构造（`ComparisonView` 内的 columns/dataSource 组装）→ 抽为共享或复用。

## 5. 数据流

```
详情页 QuotationDetail 挂载
  └─ quotationService.getById(id)   ← 唯一数据请求（无 batch-expand / 无模板值重算 / 无 ensure）
       └─ 返回: lineItems[] {
              quoteCardValues, quoteCardStructure,
              costingCardValues, costingCardStructure,
              quoteExcelValues, costingExcelValues,
              报价/核价 Excel 列定义
          } + comparison_tag(单独一次轻查, 无计算)
  └─ 两级 Segmented(mainTab × viewType) 选择
       ├─ 报价·卡片  → ReadonlyProductCard(quoteCardValues)
       ├─ 核价·卡片  → ReadonlyProductCard(costingCardValues, side=COSTING)
       ├─ 报价·Excel → ReadonlyExcelView(quoteExcelValues, 列定义)
       ├─ 核价·Excel → ReadonlyExcelView(costingExcelValues, 列定义)
       └─ 比对       → ReadonlyComparison(quote/costingExcelValues, tags)
  └─ 任一视图对应快照为空 → <Empty description="暂无数据" />
```

## 7. 前端改动

### F1. `QuotationDetail.tsx`
- 把"产品明细"卡片内的 `ReadonlyProductCard` 列表，替换为**两级 Segmented + 按选择渲染对应只读渲染器**的容器。
- 状态：`mainTab: 'quote'|'costing'|'comparison'`、`viewType: 'card'|'excel'`（与编辑态枚举对齐）。
- 每个视图按"对应快照是否为空"决定渲染内容或 `<Empty description="暂无数据" />`。
- 基本信息 / 合计行 / 审批历史 / 一级 Tabs / 各 Drawer **均不动**。
- 组合产品 PART 子件过滤维持现状（`compositeType !== 'PART'`）。

### F2. `ReadonlyProductCard.tsx`
- 加 COSTING 分支：`side === 'COSTING'` 时读 `costingCardValues` + `costingCardStructure`，逻辑与现有 QUOTE 快照路径对称（`useSnap`、`buildSnapshotExpansions`、`formulaResults[rowKey]`）。
- 保持"无快照回退实时"仅作兜底；详情页严格期望读到快照。

### F3. 新建 `ReadonlyExcelView.tsx`（薄）
- 输入：`excelValues`（已算好的行值快照）+ 列定义（标题 / `display_format` / `source_type`）。
- 渲染：纯 HTML/AntD 表格；单元格走共享 `renderCellValue` / `formatNumber`。
- **零计算**：不调 `buildExcelSnapshot`、不调后端、不 `ensureExcelValues`、不 `useDriverExpansions`。
- 空 → "暂无数据"。

### F4. 新建 `ReadonlyComparison.tsx`
- 输入：`quoteExcelValues` + `costingExcelValues`（两份快照行值）+ `comparison_tag`（`comparisonTagService.list('ACTIVE')`）。
- 复用 `buildComparisonModel` + 比对列/行组装（抽共享）；渲染比对表格 + "仅看差异"开关 + 导出按钮（导出走现有 `comparisonExportService`，只读不重算）。
- **零计算**：不走 `useLinkedExcelRows`。
- 任一侧 Excel 快照为空（如草稿）→ "暂无数据"。

### F5. 抽共享纯函数（反分叉）
- 从 `LinkedExcelView.tsx` 抽 `renderCellValue` / `formatNumber` 到共享模块，`LinkedExcelView` 与 `ReadonlyExcelView` 共用。
- `comparisonModel.ts` 的 `buildComparisonModel` 共用；比对列/行组装抽共享供 `ComparisonView` 与 `ReadonlyComparison` 共用。

## 12. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 双源分叉（AP-41/50）：只读 Excel/比对与编辑态算出不同结果 | 抽共享纯函数（`renderCellValue`/`formatNumber`/`buildComparisonModel`），只读组件与编辑组件**共用逻辑只换取数源**；E2E 对比两侧一致 |
| 旧报价单（feature 前提交）缺某快照字段 | 空 → "暂无数据"（D5）；不回退实时、不 backfill |
| `getById` 增大响应体（多捎 4 份值 + 列定义） | 仅只读路径需要；如体积过大，评估按 viewType 懒取（保持"零值计算"前提，仅分页取已存 JSON） |
| Excel 值 JSON 形状与 `ReadonlyExcelView` 假设不符 | §9 实现期先读 `buildExcelValues` 定契约，再写渲染器 |
| 协议级遗漏（AP-44 类静默失败） | 强制双 spec E2E + 5 视图截图作渲染证据 |
