## 1. 背景与目标

报价单**详情页**（`QuotationDetail.tsx`，只读路由）当前**只展示报价单一侧的产品卡片**（`ReadonlyProductCard`）。用户要看 Excel 视图、核价单、比对视图，必须进入**编辑向导**（`QuotationStep2.tsx`），它有两级视图切换：上层 `mainTab`（报价单 / 核价单 / 比对视图）+ 下层 `viewType`（📋 产品卡片 / 📑 Excel 视图）。

**目标**：把这套两级切换以**只读、读快照**的形式搬到详情页，让用户**不进入编辑态**也能查看报价单全部视图，符合"提交后全部数据冻结"的业务语义。

**"零计算"的精确定义（评审 M-3 修订）**：本设计承诺的不是"零 HTTP 请求"，而是 **零 driver batch-expand、零 `ensureExcelValues`、零阻塞渲染的公式重算**。
- 新增的 Excel / 比对只读渲染器：读已落库快照 + getById 捎回的列定义 → **真正零后端值计算**。
- 复用的报价 `ReadonlyProductCard`：FORMULA 读 `formulaResults` 不重算（`ReadonlyProductCard.tsx:240-250`），driver 用 `buildSnapshotExpansions` 不发 batch-expand（`:217-233`）；但它**既有**逐行 `enrichComponentData`（`:183`）+ `usePathFormulaCache`→`POST /formulas/batch-evaluate`（`:256`，`QuotationDetail.tsx:139` 顶层亦预热一次）。这是既有行为，**本次不引入新计算、也不要求消除**（可选优化见 §11）。

## 3. 现状盘点（已对当前代码核验，含评审纠错）

### 3.1 详情页与三套视图的取数现状
- **详情页** `QuotationDetail.tsx`：在"产品明细"卡片内渲染 `ReadonlyProductCard` 列表，过滤组合子件 `filter(li => li.compositeType !== 'PART')`（`:441`）。
- **报价·卡片**：`ReadonlyProductCard.tsx` 已脱钩读快照——`useSnap = !!lineItem.quoteCardValues && components.length>0`；FORMULA 优先读 `formulaResults[rowKey]`（不重算，`:240-250`）；`buildSnapshotExpansions` 构造 driverExpansions 不发 batch-expand（`:217-233`）；依赖**报价单级** `quoteCardStructure`。**但仍跑 `enrichComponentData`（`:183`）+ `usePathFormulaCache`(→batch-evaluate，`:256`)**——故"已具备"但**非零后端请求**（评审 M-3 纠错）。
- **报价·Excel**：编辑态 `LinkedExcelView` QUOTE 侧**恒走前端 `frontendRows = buildExcelSnapshot(...)` 实时算**（`LinkedExcelView.tsx:153-205`），不读 `quoteExcelValues`。`quoteExcelValues` 已落库，且通用快照 hook `useExcelSnapshotRows` 也消费它（**非"仅供导出"**，评审纠错）。
- **核价·Excel**：编辑态 COSTING 侧走后端 `getExcelView` / `ensureExcelValues`（懒算）。**核价 Excel 快照是 BOM 树**（`treeMode:true`，`__nodeId/__hfPartNo/__parentNo/__bomVersion/__lvl`）。
- **比对**：编辑态 `ComparisonView` 用 `useLinkedExcelRows` 对两侧逐格求值。`buildComparisonModel`（`comparisonModel.ts:91-168`）**是纯函数**：吃已算行值 → 按 `comparison_tag` 映射 + 差异比对，不跑公式（属实）。**注意**：对 v2「引用 EXCEL 组件」模板，`useLinkedExcelRows` 命中 v2 后退化（空列/空行），故编辑态比对本身在 v2 下可能为空——**不能作为只读比对的对账基准**（评审 B-1）。

### 3.2 后端快照落库 / 暴露现状
- **四份值快照已全部在 `LineItemDTO.from` 暴露**：`quoteCardValues / quoteExcelValues / costingCardValues / costingExcelValues`（`QuotationDTO.java:184-187` 声明、`:229-232` 赋值）。
- **四份结构已全部在报价单级顶层暴露**，含 `costingCardStructure`：`QuotationService#populateViewStructures` 的 `case "COSTING_CARD"`（`QuotationService.java:162-178`，COSTING_CARD 在 `:173`）。
- Excel 值快照形状（`CardSnapshotService.buildExcelValues`）**已知**：报价/平表 `{rows:[{colKey:value}, ...]}`（`CardSnapshotService.java:1073`）；核价/树 `{rows:[...], treeMode:true}` 带 `__lvl/__parentNo` 等（`:1106-1121`）。**现成消费方**：`useExcelSnapshotRows.ts:23-61`（含树识别）。
- `ensureExcelValues`（`CardSnapshotService.java:582`）整单一次 `IN` 预取 componentData，逐行 `buildExcelValues`，**`== null` 才算**，落 `quoteExcelValues` + `costingExcelValues`（两侧分别 `:607-616`）。
- 调用点：①独立端点 `POST /{id}/ensure-excel-values`；②**submit 内** `QuotationResource.java:365`（覆盖两侧，已核验属实）。saveDraft：P3 lazy，Excel 留 NULL → 草稿态 Excel 快照空，与 D6 一致。
- **列定义现状（真正缺口）**：getById 现仅捎带 `quoteExcelStructure/costingExcelStructure.columns`，但**camelCase（`colKey`/`comparisonTag`）且无 `display_format`**（`quotationService.ts:77-92`）。而渲染/比对消费的是 `CostingTemplateColumn`：**snake_case `col_key`/`comparison_tag`/`display_format.decimals`**（`costingTemplateService.ts:3-19`）。→ **列定义是唯一需要后端补的东西**。

## 4. 架构（选项 A：薄只读渲染器 + 复用只读卡片）

**原则**：详情页一次 `getById` 拿全 5 视图所需快照（值 + 结构 + **带 `display_format` 的有效 Excel 列定义**）；各只读渲染器尽量纯函数，**不发起 driver 展开 / ensure / 后端逐行值求值**。

| 视图 | 渲染器 | 吃的快照字段 | 工作量 |
|---|---|---|---|
| 报价·卡片 | 复用 `ReadonlyProductCard`（现状） | `quoteCardValues` + `quoteCardStructure` | 已具备（含既有 enrich/batch-evaluate） |
| 核价·卡片 | `ReadonlyProductCard` 加 **结构驱动 COSTING 路径** | `costingCardValues` + `costingCardStructure` + 核价模板 id | **Medium**（移植编辑态核价快照渲染） |
| 报价·Excel | 新建 `ReadonlyExcelView`（复用 `useExcelSnapshotRows`） | `quoteExcelValues` + 报价有效列 | 新增（平表） |
| 核价·Excel | 同 `ReadonlyExcelView`（树形分支） | `costingExcelValues`（**BOM 树**） + 核价有效列 | 新增（**树形布局**） |
| 比对 | 新建 `ReadonlyComparison`（复用 `buildComparisonModel`） | `quoteExcelValues` + `costingExcelValues` + `comparison_tag` | 新增 |

**值/结构无需任何后端暴露改动**（§3.2 已全暴露）。**唯一后端改动 = 列定义捎回**（§6 B1）。

### 反分叉纪律（AP-41 / AP-50）+ 其边界（评审 B-2）
抽共享**纯展示/纯数据逻辑**，让只读与编辑态组件共用、只换取数源：
- `renderCellValue` / `formatNumber`（从 `LinkedExcelView.tsx` 抽出）→ 共享给 `ReadonlyExcelView`。
- `buildComparisonModel`（已纯）+ 比对列/行组装 → `ReadonlyComparison` 复用。
- **边界（必须在计划中言明）**：共享仅统一**显示格式**，不统一**值来源**。报价 Excel 存在"编辑态前端 `buildExcelSnapshot` 实时引擎"与"后端 `buildExcelValues` 落库引擎"两套（统一引擎项目核价侧未做）。**详情页只读 Excel 的事实源 = 后端冻结快照值**，不与编辑态实时值对账（见 §10）。

## 5. 数据流

```
详情页 QuotationDetail 挂载
  └─ quotationService.getById(id)            ← 主数据请求
       └─ 返回:
            · 报价单级顶层: 四份结构(quote/costing × card/excel 结构) + 【新增】带 display_format 的报价/核价有效 Excel 列定义
            · lineItems[].{ quoteCardValues, costingCardValues, quoteExcelValues, costingExcelValues }
  └─ comparisonTagService.list('ACTIVE')      ← 轻查 tag(无值计算)
  └─ 两级 Segmented(mainTab × viewType) 选择
       ├─ 报价·卡片  → ReadonlyProductCard(quoteCardValues, quoteCardStructure)         ※既有 enrich/batch-evaluate
       ├─ 核价·卡片  → ReadonlyProductCard(costingCardValues, costingCardStructure, side=COSTING, 核价模板id)
       ├─ 报价·Excel → ReadonlyExcelView(useExcelSnapshotRows(quoteExcelValues) + 报价有效列)
       ├─ 核价·Excel → ReadonlyExcelView(useExcelSnapshotRows(costingExcelValues, 树) + 核价有效列)
       └─ 比对       → ReadonlyComparison(quote/costingExcelValues + tags → buildComparisonModel)
  └─ 任一视图对应快照为空 → <Empty description="暂无数据" />
```
> 注：四份**结构**在响应**顶层**（报价单级），四份**值**在 `lineItems[]` 上（评审 Minor 修正）。

## 6. 后端改动（评审 M-1 修订：缩成单一改动）

### B1.（唯一后端改动）getById 捎回带 `display_format` 的有效 Excel 列定义
- 在 `QuotationService#getById` 装配处，用 `ExcelViewService.getEffectiveColumns(template)`（`ExcelViewService.java:67`）取**报价模板 + 核价模板**两份**有效列**（含 `col_key`/`comparison_tag`/`display_format`/`source_type`，零值计算），随响应捎回。
- 目的：详情页 `ReadonlyExcelView`/`ReadonlyComparison` 拿到 snake_case + display_format 的权威列定义，**不必**调 `getExcelView`（会逐行算值，违反零计算），也避免 camelCase 结构列缺 display_format 导致 PERCENT/小数位静默丢。

### B2. submit 冻结回归守卫（核验，预计不改逻辑）
`QuotationResource.java:365` submit 内 `ensureExcelValues` 已覆盖两侧（已核验）。补回归测试：submit 后该报价单四值列均非 NULL。

### B3. 不动项（明确）
- **不**暴露值/结构（已全暴露）。**不**在 saveDraft 热路径补 Excel 计算。**不**退役 `row_data`/`snapshot_rows`。**不**把核价侧改前端单引擎。

## 7. 前端改动

### F1. `QuotationDetail.tsx`
- "产品明细"卡片内 `ReadonlyProductCard` 列表 → 替换为**两级 Segmented + 按选择渲染对应只读渲染器**的容器。
- 状态 `mainTab: 'quote'|'costing'|'comparison'`、`viewType: 'card'|'excel'`（与编辑态枚举对齐）。
- 每视图按"对应快照是否为空"渲染内容或 `<Empty description="暂无数据" />`。
- 基本信息 / 合计 / 审批历史 / 一级 Tabs / 各 Drawer **不动**；PART 过滤维持。

### F2. `ReadonlyProductCard.tsx` —— 加**结构驱动 COSTING 路径**（评审 M-4）
- 不是"对称加分支"。核价侧无核价 componentData，需：
  - 用 `buildComponentDataFromStructure(costingCardStructure, ...)`（`enrichComponentData.ts`，已被 `QuotationWizard.tsx:29/426` 使用）组装核价 componentData；
  - 核价模板 id 取 `quotation.costingCardTemplateId`（**不在 lineItem 上**，需从详情页线程化传入）；
  - `useSnap` 读 `costingCardValues`，`buildSnapshotExpansions(..., 'COSTING', ...)`。
- **1:1 参照**编辑态 `QuotationStep2` 已有的核价快照渲染（`:3070` `useSnapCosting` 读 `costingCardValues`；`:3324/:3370` `side="COSTING"` + `cardStructure={costingCardStructure}`）。
- 属协议级（改 ReadonlyProductCard 渲染分支，AP-50）→ 强制双 spec E2E。

### F3. 新建 `ReadonlyExcelView.tsx`（复用现成 hook）
- **取值复用 `useExcelSnapshotRows`**（`useExcelSnapshotRows.ts:23-61`）：解析 `*ExcelValues` 的 `{rows:[...]}`、按 `col_key` 铺平、识别核价树。
- **列定义**来自 getById 捎回的有效列（B1），走共享 `renderCellValue`/`formatNumber`。
- **核价侧树形布局**：`costingExcelValues` 是 BOM 树（`treeMode:true`/`__lvl/__parentNo`），需树渲染（参照编辑态 LinkedExcelView 树渲染 / `treeTable.ts`），**非平表**。
- 零后端值计算；空 → "暂无数据"。

### F4. 新建 `ReadonlyComparison.tsx`
- 输入：`quoteExcelValues` + `costingExcelValues`（两份冻结快照行值，经 `useExcelSnapshotRows` 解析）+ `comparison_tag`（`comparisonTagService.list('ACTIVE')`）+ B1 的有效列（取 `comparison_tag`）。
- 复用 `buildComparisonModel` + 比对列/行组装（抽共享）；渲染比对表 + "仅看差异"开关 + 导出（`comparisonExportService`，吃模型不重算）。
- 零计算；任一侧 Excel 快照空（草稿）→ "暂无数据"。

### F5. 抽共享纯函数（反分叉，注意 §4 边界）
- `LinkedExcelView.tsx` 抽 `renderCellValue`/`formatNumber` → 共享给 `ReadonlyExcelView`。
- `buildComparisonModel` 共用；比对列/行组装抽共享供 `ComparisonView` 与 `ReadonlyComparison` 共用。
- **抽函数后必跑编辑态 E2E**，确认不破坏 `LinkedExcelView`/`ComparisonView`。

## 12. 风险与缓解（据评审更新）

| 风险 | 缓解 |
|---|---|
| **跨引擎对账陷阱（B-1/B-2）**：报价 Excel/比对 编辑态与详情页由两套引擎产出，硬要"一致"会卡死 E2E | §10 改口径：详情页只读 Excel/比对**只与冻结快照/导出对账**，冻结快照为唯一事实源 |
| 双源分叉（AP-41/50）：只读 Excel/比对与编辑态显示分叉 | 抽共享 `renderCellValue`/`formatNumber`/`buildComparisonModel`（仅统一**格式**）；抽函数后跑编辑态 E2E 防回归 |
