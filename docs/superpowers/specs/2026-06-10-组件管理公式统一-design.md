## 1. 背景与目标

Excel 模板的「页签连表公式」配置页与公式配置功能已完成。本次做一次收敛重构，把**所有公式配置能力集中到组件管理**，并统一三类组件的配置体验。

**核心目标**

1. Excel 模板**收编**为组件管理中的一种组件类型，模板配置页只需"拖入 Excel 视图即用"。
2. 三类组件（页签 / EXCEL / 小计）的公式配置**统一复用**现有 `TabJoinFormulaDrawer`（页签连表公式抽屉）。
3. 组件管理列表风格与模板管理对齐，配置风格统一。
4. 清理：移除冗余的右侧 `FieldPanel`；路径来源收敛为 SQL 视图；隐藏/合并若干配置项。

---

## 2. 决策清单（已与用户逐项确认）

| # | 议题 | 决策 |
|---|------|------|
| 1 | Excel 模板与组件的关系 | **A 完全收编**：废弃独立 Excel 模板实体，迁移为 `component_type = EXCEL` 的组件。模板配置页 Excel 视图拖入 EXCEL 组件即用。 |
| 1b | EXCEL 组件粒度 | 一个 EXCEL 组件 = **一整张 Excel 表**（含全部列），与现有 Excel 模板 1:1。模板 Excel 视图通常拖 1 个。 |
| 2 | 公式抽屉形态 | **复用 `TabJoinFormulaDrawer`**（页签连表公式抽屉），三类组件公用同一个。聚合为可选项（直接取值 / SUM / AVG / MIN / MAX / COUNT）。 |
| 3 | 组件管理布局 | **左列表 + 右内嵌详情（Master-Detail 双栏）**。左：搜索/新建/批量动作 + 手风琴目录 → 三段固定分类（页签组件 / EXCEL 组件 / 小计组件）→ 纵向渐变卡片。右：选中组件后内嵌详情（非抽屉）。卡片复用现有 `tm-component-item` 渐变样式（页签紫 / EXCEL 青 / 小计橙）。命中列表规范「例外白名单 · Master-Detail 双栏」。 |
| 4 | 右侧 `FieldPanel` 去留 | **B 移除 FieldPanel，并砍掉"全局变量 / 其他数据源插入公式"能力**。公式编辑统一走 `TabJoinFormulaDrawer`（本页签字段引用 + 跨页签引用已足够）。全局变量插入暂不做，未来如需再并入该抽屉。**字段级**全局变量取值（FieldConfigTable 🌐 按钮）/ 数据源绑定（DataSourceModal）独立存在，**不受影响**。 |
| 5 | 详情区配置项精简 | 隐藏「树表（纯展示）」；「数据驱动路径（可选）」与「核价 BOM 递归展开」**合并为一行**；公式列表**移除「结果类型」列**（结果恒为数字）。 |
| 6 | 路径来源收敛 | **A 只清 UI 入口，保留底层路径引擎**：UI 仅保留"选 SQL 视图"作为路径来源 → 产出 `$view.列`；移除 `PathPickerDrawer` 的 manual 手输原始路径 Tab、DATA_SOURCE 的 `BNF_PATH` 子类型、老表直引警告/lint。底层 `basic_data_path` / `data_driver_path` 路径字段与 BNF 求值引擎（`BnfPathResolver` / `PathToSqlGenerator`）**保留**，值统一为 `$view.列`，用户永不手写裸路径。历史老表直引数据可直接删除（均为测试数据）。 |
| A | 组件管理里公式抽屉的"可引用页签集" | **同目录组件即页签集**：编辑某组件公式时，矩阵列出**同目录下其他页签/EXCEL/小计组件**（组件名作页签别名）。语义：一个目录 = 一个产品的页签集合。试算样本 = 选一张"用到该组件的报价/核价卡片"；无样本则禁用试算，仅保存表达式。 |

**字段配置不变项**：字段属性 `字段名 / 字段类型 / 内容(取值来源) / 金额 / 小计 / 行键 / 备注 / 排序` 全部沿用现有 `FieldConfigTable`，保持可编辑。本次对字段配置的唯一改动是路径来源收敛（点 6）。

## 3. 数据模型改动

### 3.1 Excel 模板 → EXCEL 组件（点 1）

- 新增/复用 `component_type = 'EXCEL'` 枚举（现有为 NORMAL / SUBTOTAL，新增 EXCEL）。
- Excel 模板的列配置（列名 / 来源 / 公式 / **显示·隐藏** / 排序）迁移进 EXCEL 组件的存储结构（JSONB）。
  - **「显示/隐藏」列属性必须保留**：隐藏列仍参与公式计算，但不在 Excel 视图/导出中呈现。
- 模板与 EXCEL 组件的关联走现有"模板—组件关联表"，Excel 视图引用 EXCEL 组件（只读引用，列与公式在组件侧维护）。
- 旧独立 Excel 模板管理入口删除。
- **迁移**：现有 Excel 模板数据 → EXCEL 组件；历史均为测试数据，迁移策略可"重建 + 删旧"，不必严格保数。

### 3.2 路径来源收敛（点 6）

- 移除 UI：`PathPickerDrawer` 的 manual Tab、`DATA_SOURCE` 的 `BNF_PATH` 子类型选项、老表直引黄色警告/`BnfPathLinter` 面向用户的提示入口。
- 保留底层：`basic_data_path` / `data_driver_path` 字段、`$view.列` 文法、`BnfPathResolver` / `PathToSqlGenerator` 求值。
- 删除历史含裸表路径（`mat_*` / `costing_part_*` 等）的组件/模板测试配置。

## 4. UI 设计（详见原型）

### 4.1 组件管理页（Master-Detail）

```
┌──────────────────┬───────────────────────────────────┐
│ 搜索 / ＋新建      │  〔右：选中组件的内嵌详情〕         │
│ 发布 停用 删除     │  投料  页签组件·COMP-0027   复制 保存│
│ ─────────────     │  ┌─────────────────────────────┐  │
│ ▼ 标准螺丝产品     │  │数据驱动路径 ＋ 核价BOM递归 一行│  │
│   页签组件         │  └─────────────────────────────┘  │
│    投料/回料/…     │  字段配置 | 公式 | SQL视图          │
│   EXCEL组件        │  〔字段表：名/类型/来源/金额/小计/  │
│    核价单Excel     │    行键/备注 可编辑〕               │
│   小计组件         │                                    │
│    产品小计        │                                    │
│ ▶ 精密弹簧产品     │                                    │
└──────────────────┴───────────────────────────────────┘
```

- 左栏手风琴目录，每目录下三段固定分类，纵向卡片；卡片左侧勾选框驱动顶部批量工具栏。
- 点卡片 → 右栏内嵌详情；空态提示"从左侧选择组件"。
- 三类组件详情差异：
  - **页签组件**：合并配置行（数据驱动路径 + 核价BOM递归）+ Tabs(字段配置 / 公式 / SQL视图)。
  - **EXCEL 组件**：Excel 列配置（列名 / 来源·公式 / **显示开关** / 操作）。
  - **小计组件**：仅公式（与页签组件公式区一致）。

### 4.2 公式编辑 = `TabJoinFormulaDrawer`

- 在详情「公式」列表点"编辑/添加" → 弹出该抽屉（唯一保留的抽屉，聚焦子任务）。
- 抽屉结构（与现有 1:1）：试算条 → 公式表达式 TextArea → 运算符 + 函数(SUM/AVG/MIN/MAX/COUNT) → 规则提示 → 页签字段矩阵（明细字段 token + 页签总计 token + 行键锁定置灰）。
- 本次差异：**去「结果类型」列、去「全局变量插入」**；矩阵的"可引用页签集 = 同目录组件"（决策 A）。

### 4.3 模板配置页（点 1 模板侧）

- 调色板分三组：页签组件 / EXCEL 组件 / 小计组件（渐变卡片，可拖）。
- Excel 视图为「单组件容器」，拖入 EXCEL 组件即用；列与公式只读引用自组件，模板侧不再配公式。

## 5. 影响面与风险

- **AP-44 字段类型联动协议**：新增 `component_type = EXCEL`、移除 `BNF_PATH` 子类型属于字段/组件类型协议改动，需对照 AP-44 的 17 个检查点核查，并跑 E2E（`quotation-flow.spec.ts` + 必要时 `composite-product-flow.spec.ts`）。
- **AP-53 路径规则**：收敛后所有路径必须 `$view.列`；删除历史裸表路径数据时确认无残留引用。
- **`TabJoinFormulaDrawer` 重定位**：原为模板级（`templateId` + SampleCardPicker + 模板 tabDefs）。搬到组件管理需新增"以目录为页签集"的上下文来源 + 组件级试算样本来源（选用到该组件的卡片）。这是后端/前端契约新增点，须在实现计划中拆任务。
- **快照/渲染链路**：Excel 组件参与报价单/核价单渲染时，沿用现有 component snapshot 机制；迁移后验证 Excel 视图渲染与导出（显示/隐藏列正确）。

## 6. 不在本次范围（YAGNI）

- 公式抽屉内"全局变量插入" —— 暂不做，未来按需并入 `TabJoinFormulaDrawer`。
- 产品卡视图（detail 模式）拖拽组装逻辑 —— 沿用现状，本次不动。
- 历史 Excel 模板/裸表路径数据的"保数迁移" —— 测试数据，重建即可。

## 7. 验收要点

1. 组件管理：左列表 + 右详情，三段分类卡片，点卡进详情；三类组件详情形态正确。
2. 三类组件配公式都弹 `TabJoinFormulaDrawer`，无结果类型列、无全局变量插入，矩阵为同目录组件。
3. 字段配置保留 金额/小计/行键/备注 可编辑；路径来源只剩 SQL 视图。
4. EXCEL 组件列保留显示/隐藏；模板 Excel 视图拖入 EXCEL 组件可渲染、可导出。
5. 详情区：树表隐藏、数据驱动路径与核价BOM递归合并一行。
6. E2E 通过 + 报价单/核价单/详情三视图渲染正确（按 AP-44 SOP）。
